## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

| Variable group | Method | Parameters | When applied |
|---|---|---|---|
| CRSP variables (`StockRet`, `MarketRet`, `Volatility`, `amihud_illiq`) | Per-year 1%/99% | lower=0.01, upper=0.99, min_obs=10 | Inside `CRSPEngine.get_data()` before caching |
| Compustat variables (`Size`, `Lev`, `ROA`, `TobinsQ`) | Per-year 1%/99% | lower=0.01, upper=0.99 | Inside `CompustatEngine` |
| Linguistic variables (all `_pct` columns) | Per-year 0%/99% (upper-only) | lower=0.0, upper=0.99, min_obs=10 | Inside `LinguisticEngine.get_data()` |
| Clarity residuals (B-specs) | None | `_skip_winsorization=True` | `ceo_clarity_residual.py:36` |

**RESOLVED (L4):** Inconsistent winsorization regimes — different lower bounds for linguistic (0%) vs CRSP/Compustat (1%) — intentionally retained. Economic rationale: linguistic pct variables are naturally zero-bounded; true zero-uncertainty calls are valid observations and should not be trimmed. Table notes corrected to explicitly disclose both regimes: "CRSP and Compustat financial variables are winsorized at the 1st/99th percentile per year. Linguistic variables are winsorized at the 0th/99th percentile per year (upper-tail only, given their natural lower bound of zero)."

### Missing Data Policy

- Listwise deletion: `df_sample.replace([np.inf, -np.inf], np.nan).dropna(subset=required)` before each regression
- Inf → NaN coercion before dropna (correct)
