## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H7 — Speech Vagueness and Stock Illiquidity |
| Research question | Does managerial speech vagueness (uncertainty language) in earnings calls increase post-call stock illiquidity (Amihud 2002)? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `call_quarter_int` (time index, integer-encoded fiscal quarter) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |

### A3) Outcome / Target Definition

| Field | Value |
|-------|-------|
| Dependent variable | `delta_amihud` — Change in Amihud (2002) illiquidity around the earnings call |
| Construction | `PostAmihud - PreAmihud`, where PreAmihud = `mean(daily_illiq)` over trading days [-3,-1] before call and PostAmihud = `mean(daily_illiq)` over trading days [+1,+3] after call. `daily_illiq = |RET| / (VOL * |PRC|) * 1e6`. Uses trading-day positions (rank-based), not calendar days. |
| Source builder | `AmihudChangeBuilder` (`src/f1d/shared/variables/amihud_change.py`) |
| Timing convention | Event-window change: post-call [+1,+3] minus pre-call [-3,-1] trading days relative to the last trading date on or before the call |
| Minimum trading days | 2 per window (calls with <2 valid trading days in either pre or post window get NaN) |
| Missing | Drop (listwise deletion at regression stage) |
| Winsorization | **NOT WINSORIZED.** `delta_amihud` enters OLS raw. Range [-99.6, 2678.2], SD=12.62. See Critical issue L1. |

## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

| Variable group | Method | Bounds | When applied | Code reference |
|---------------|--------|--------|-------------|----------------|
| Linguistic `_pct` columns | Per-year | 0% / 99% (upper-only) | At engine load time | `_linguistic_engine.py` L255-258, `winsorization.py:winsorize_by_year()` |
| CRSP variables (StockRet, MarketRet, Volatility, amihud_illiq) | Per-year | 1% / 99% | At engine load time | `_crsp_engine.py` L444-447, `winsorization.py:winsorize_by_year()` |
| Compustat controls (Size, Lev, ROA, TobinsQ, etc.) | Per-fyearq | 1% / 99% | At engine compute time | `_compustat_engine.py` L1138-1160, `_winsorize_by_year()` |
| Clarity residuals | Not winsorized | N/A | N/A | `ceo_clarity_residual.py` L37: `self._skip_winsorization = True` |
| **`delta_amihud` (DV)** | **NOT WINSORIZED** | N/A | N/A | `amihud_change.py`: zero matches for "winsor". Range [-99.6, 2678.2], SD=12.62. **CRITICAL: extreme leverage points enter OLS raw.** |
| **`pre_call_amihud` (control)** | **NOT WINSORIZED** | N/A | N/A | `amihud_change.py`: same builder, no winsorization step. Max=267.8. |

**VERIFIED IMPLEMENTATION ISSUE:** The DV `delta_amihud` and its lagged-level control `pre_call_amihud` both bypass winsorization entirely. In contrast, the inter-call Amihud level (`amihud_illiq`) from `CRSPEngine` IS winsorized at 1%/99% per year — but `amihud_illiq` is not used as the DV. The `AmihudChangeBuilder` is a separate builder that does not invoke any winsorization. This is the most serious specification issue in the suite.

### Missing data policy

- **Listwise deletion** at regression stage: `dropna(subset=required)` on DV + IV + all controls (`run_h7_illiquidity.py` L190).
- Inf values replaced with NaN before dropna (L190).
- Missing `ff12_code` classified as "Main" (not dropped).

### Denominator protections

- `dollar_volume = 0` -> replaced with NaN to avoid inf (`_crsp_engine.py` L220).
- `atq <= 0` -> Size = NaN (`_compustat_engine.py` L1027).
- Inf values replaced with NaN after all ratio computations (`_compustat_engine.py` L1132-1133).

### Scaling

- Amihud daily illiquidity scaled by 1e6 (standard in literature).
- Linguistic variables are percentages (word count / total words * 100).
- Size is log-transformed.

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line referee verdict

H7 posits that managerial speech vagueness increases post-call stock illiquidity (measured as the change in Amihud illiquidity around the call). The current run (2026-03-13) shows **marginal evidence at the 10% level** for Manager QA Uncertainty (A3: beta=0.3285, p_one=0.069) and Manager Clarity Residual (B2: beta=0.0072, p_one=0.080). CEO specs and Manager Presentation specs are insignificant. The H7-C Wald test is marginally significant (p=0.097).

However, these results are obtained from regressions where the DV (`delta_amihud`) is **completely unwinsorized** with extreme outliers (max/SD = 212x). Until the DV is winsorized and results re-examined, the marginal significance cannot be trusted — it may be driven by a handful of extreme leverage points.

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

The model specification is reasonable: event-window change DV with pre-call level control, appropriate FE structure, entity-clustered SEs. The critical deficiency is that the DV and `pre_call_amihud` control enter OLS unwinsorized with extreme values. The suite currently supports **descriptive association only**, and even that is unreliable without winsorization.
