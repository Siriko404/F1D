## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H7 — Speech Vagueness and Stock Illiquidity |
| Research question | Does managerial speech vagueness (uncertainty language) in earnings calls increase post-call stock illiquidity (Amihud 2002)? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `call_quarter_int` (time index, integer-encoded fiscal quarter) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |

### A3) Outcome / Target Definition

| Field | Value |
|-------|-------|
| Dependent variable | `amihud_illiq` — Amihud (2002) illiquidity measure |
| Construction | `mean(|daily_ret| / dollar_volume) * 1e6` over post-call window `[call_date + 1 calendar day, next_call_date - 5 calendar days]` (note: `pd.Timedelta(days=N)` operates in calendar days; effective end buffer ≈ 3-4 trading days) |
| Timing convention | Contemporaneous (t): the Amihud measure for the window following the call that generated the linguistic measures |
| Minimum trading days | 10 (calls with <10 valid trading days in window get NaN) |
| Missing → | Drop (listwise deletion at regression stage) |

### A4) Main RHS and Controls

**Key independent variables (one per spec):**

| Spec | Variable | Description |

| B2 | `Manager_Clarity_Residual` | Manager idiosyncratic uncertainty (residual from H0.3 regression) |

## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

| Variable group | Method | Bounds | When applied | Code reference |
|---------------|--------|--------|-------------|----------------|
| Linguistic `_pct` columns | Per-year | 0% / 99% (upper-only) | At engine load time | `_linguistic_engine.py` L255-258, `winsorization.py:winsorize_by_year()` |
| CRSP variables (StockRet, MarketRet, Volatility, amihud_illiq) | Per-year | 1% / 99% | At engine load time | `_crsp_engine.py` L444-447, `winsorization.py:winsorize_by_year()` |
| Compustat controls (Size, Lev, ROA, TobinsQ, etc.) | Per-fyearq | 1% / 99% | At engine compute time | `_compustat_engine.py` L1138-1160, `_winsorize_by_year()` |
| Clarity residuals | Not winsorized | N/A | N/A | `ceo_clarity_residual.py` L37: `self._skip_winsorization = True` |

### Missing data policy

- **Listwise deletion** at regression stage: `dropna(subset=required)` on DV + IV + all controls (`run_h7_illiquidity.py` L196).
- Inf values replaced with NaN before dropna (L196).
- Missing `ff12_code` classified as "Main" (not dropped).

### Denominator protections

- `dollar_volume = 0` → replaced with NaN to avoid inf (`_crsp_engine.py` L220).
- `atq <= 0` → Size = NaN (`_compustat_engine.py` L1027).
- Inf values replaced with NaN after all ratio computations (`_compustat_engine.py` L1132-1133).

### Scaling

- Amihud illiquidity scaled by 1e6 (standard in literature).
- Linguistic variables are percentages (word count / total words * 100).
- Size is log-transformed.
- Volatility and StockRet scaled to percentage units (* 100).

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line referee verdict

H7 posits that managerial speech vagueness increases post-call stock illiquidity. As implemented, **no specification produces a statistically significant positive coefficient on any uncertainty measure**. Several coefficients are negative (wrong sign). The Wald test for H7-C (QA > Pres) is also insignificant. The hypothesis is clearly not supported by the data.

**Overall status: THESIS-STANDARD AS IMPLEMENTED (for a null result)**

The implementation is technically sound: appropriate estimator, proper FE, clustered SEs, winsorization, robustness checks. The null result is credible and honestly reported. The suite supports **descriptive association only** (and that association is essentially zero). There is no causal claim to evaluate.

If the thesis write-up treats this as a null finding and discusses it as such, the implementation is adequate. If it attempts to salvage causal language ("vagueness may increase illiquidity"), that would overstate the evidence.

### K2) Identification and research design threats

| Concern | Evidence from code/data | Severity | Why it matters | Recommended fix |
|---------|----------------------|----------|---------------|----------------|
| Reverse causality | Amihud is post-call ([call+1 calendar day, next_call-5 calendar days]) while uncertainty is measured at call time → timing is correct for forward direction | Low | Properly handled by construction | None needed |
| Omitted-variable bias | Firm FE + quarter FE absorb time-invariant firm traits and common time shocks; linguistic controls (negativity, analyst uncertainty) included | Medium | Within-firm variation in uncertainty could still be driven by unobserved firm-quarter shocks (e.g., disclosure events, M&A) | Cannot fully resolve without instrument |
| Contemporaneous confounders | Volatility and StockRet are computed over the SAME window as amihud_illiq — they share the same CRSP daily returns | High | Including Volatility (std of returns in window) and StockRet (sum of returns in window) as controls mechanically absorbs variation in amihud_illiq (which is |ret|/volume). This creates a bad-control problem: Volatility is essentially an input to Amihud | Remove Volatility and StockRet from controls or use pre-call values |
| Look-ahead bias | Compustat controls use merge_asof backward → no future data leakage; Amihud uses post-call window → timing is correct | Low | Properly implemented | None needed |
| Endogenous sample selection | Min 5 calls filter after listwise deletion; CEO specs lose ~25% of sample | Medium | Firms with fewer calls may be systematically different (smaller, less followed) | Report sample characteristics by inclusion/exclusion status |
