### 5b. Aggregation (call-level to firm-year)

| Variable type | Aggregation | Method | Code |
|---------------|-------------|--------|------|
| IVs (4 uncertainty measures) | Arithmetic mean across all calls in (gvkey, fyearq) | `groupby.mean()` | Builder L157-158 |
| Financial controls + DV + ff12_code | Last non-null value per (gvkey, fyearq) | `groupby.last()` | Builder L151-153 |

**Semantic note on `groupby.last()`:** Pandas `groupby.last()` returns the last *non-null* value per group, not strictly the last row's value. Because financial variables are constant within a firm-fiscal-year (all sourced from the same Compustat quarter via `_compute_annual_q4_variable`), this distinction is almost always immaterial. The only scenario where it matters is if a firm has multiple calls in one fiscal year and the last call's Compustat match produced a NaN while an earlier call did not -- in that case the earlier call's value would silently be used. This edge case is expected to be extremely rare.

## 6. Variable Dictionary

### 6a. Dependent variables

#### `PayoutRatio`

| Property | Detail |
|----------|--------|
| **Formula** | `dvy_annual / iby_annual` |
| **Interpretation** | Common dividends as a fraction of income before extraordinary items (Attig et al. 2013) |
| **Code** | Engine `_compute_and_winsorize()` L1105-1114 |

**Complete construction chain:**

1. **Raw Compustat fields:** `dvy` (common dividends paid, YTD cumulative within fiscal year) and `iby` (income before extraordinary items, YTD cumulative within fiscal year). Both loaded from `inputs/comp_na_daily_all/comp_na_daily_all.parquet` via `CompustatEngine.get_data()` (Engine L1232-1252). All numeric columns coerced to `float64` (Engine L1257-1258).
2. **Deduplication:** `(gvkey, datadate)` duplicates dropped, keeping last row (most recent restatement). Engine L1265.
3. **Annualization via `_compute_annual_q4_variable()`** (Engine L236-276): For each of `dvy` and `iby`, filter to fiscal-quarter-4 rows (`fqtr == 4`), which hold the full-year cumulative total. Drop rows with NaN `fyearq`. Deduplicate `(gvkey, fyearq)` keeping last. Join back to all quarterly rows via `(gvkey, fyearq)` using a left merge (`validate="m:1"`), so every call in the same fiscal year receives the same annual value.
   - `dvy_annual`: Engine L1091: `_compute_annual_q4_variable(comp, "dvy", "_dvy_annual")`
   - `iby_annual`: Engine L1059: `_compute_annual_q4_variable(comp, "iby", "_iby_annual")`
4. **Missing-dvy treatment:** `dvy_for_payout = pd.Series(dvy_annual, index=comp.index).fillna(0)` (Engine L1108). Firms with positive earnings but no reported dividends are treated as zero-payout, not dropped.
5. **Division with iby guard:** `np.where(iby_for_payout > 0, dvy_for_payout / iby_for_payout, np.nan)` (Engine L1110-1114). When `iby_annual <= 0` (losses or zero earnings), PayoutRatio is NaN and the observation is excluded from regressions.
6. **No upper-bound censoring:** PayoutRatio can exceed 1.0 when dividends exceed income (e.g., drawing on retained earnings). This is a known limitation (L1 in Section 12).
7. **Inf cleanup:** PayoutRatio is in the `ratio_cols` list (Engine L1176). All `inf` / `-inf` values replaced with NaN (Engine L1185-1186).
8. **Per-year winsorization:** PayoutRatio is in `COMPUSTAT_COLS` (Engine L122) and not in `skip_winsorize` (Engine L1199-1207). Winsorized at 1%/99% within each `fyearq` year group via `_winsorize_by_year(comp[col], year_col)` (Engine L1208-1215). The function computes `quantile(0.01)` and `quantile(0.99)` on non-NaN values within each year, then clips. Years with fewer than 10 valid observations are skipped (Engine L446, L464).
9. **Manifest merge:** `CompustatEngine.match_to_manifest()` performs a `merge_asof` backward on `start_date` within `gvkey` (Engine L1301-1308). Each earnings call is matched to the most recent Compustat quarterly observation. `PayoutRatioBuilder` extracts `["file_name", "PayoutRatio"]` from the merged result (pattern identical to `BookLevBuilder`, `SizeBuilder`, etc.).
10. **Panel builder merge:** Builder L264-269 merges PayoutRatio into the call-level panel on `file_name` (left join, `delta == 0` assertion ensures no row inflation).
11. **Firm-year aggregation:** `aggregate_to_firm_year()` (Builder L110-173) applies `groupby(["gvkey", "fyearq"])[financial_cols].last()` (Builder L151-153). Since PayoutRatio is constant within a firm-fiscal-year (all calls in the same FY share the same Q4 Compustat value), `.last()` returns the unique value. Note: pandas `groupby.last()` returns the last non-null value, not strictly the last row's value; this distinction is immaterial here because the values are identical across calls.

**Kronmal (1993) denominator overlap with ROA:** Both PayoutRatio and ROA share `iby_annual` -- PayoutRatio in the denominator, ROA in the numerator (`ROA = iby_annual / avg_assets`). This creates a mechanical negative partial correlation. See Section 11b.

#### `PayoutRatio_lead`

| Property | Detail |
|----------|--------|
| **Formula** | PayoutRatio at fiscal year t+1 |
| **Code** | Builder `create_lead_payout_ratio()` L176-195 |

**Complete construction chain:**

1. **Input:** The firm-year panel after `aggregate_to_firm_year()`, with PayoutRatio already at the firm-year level.
2. **Sort:** `sort_values(["gvkey", "fyearq"]).reset_index(drop=True)` (Builder L183).
3. **Shift:** `groupby("gvkey")["PayoutRatio"].shift(-1)` (Builder L185) -- assigns the next fiscal year's PayoutRatio to the current row.
4. **Consecutive-year check:** `fyearq_int = df["fyearq"].astype("Int64")` and `next_fyearq_int = df["next_fyearq"].astype("Int64")`. `is_consecutive = (next_fyearq_int - fyearq_int) == 1` (Builder L188-190). Non-consecutive fiscal years (gap > 1) produce NaN: `df.loc[~is_consecutive, "PayoutRatio_next"] = np.nan` (Builder L191). This prevents assigning a payout ratio from a non-adjacent year.
5. **Rename:** Column renamed from `PayoutRatio_next` to `PayoutRatio_lead` (Builder L193).
6. **Winsorization:** Inherited from PayoutRatio, which was winsorized at the engine level (step 8 above) before the lead construction.

### 6b. Key independent variables

All four IVs follow an identical construction chain. The only difference is the speaker/context segmentation applied during Stage 2 textual analysis.

#### Construction chain (common to all four IVs)

**Stage 2.1 -- Tokenization and counting** (`src/f1d/text/tokenize_transcripts.py`):

1. **LM dictionary load:** The Loughran-McDonald Master Dictionary (`inputs/Loughran-McDonald_MasterDictionary_1993-2024.csv`) is loaded. Seven sentiment categories are extracted: Negative, Positive, Uncertainty, Litigious, Strong_Modal, Weak_Modal, Constraining. For each category, the set of words with a nonzero flag in that category column is collected (Tokenize L768-812). The union of all flagged words forms the vocabulary list.
2. **Tokenization:** Each speaker turn's `speaker_text` is uppercased, then tokenized using `sklearn.CountVectorizer` with pattern `r"(?u)\b[a-zA-Z]+\b"` (alphabetic tokens only) and `lowercase=False` (Tokenize L927-931).
3. **Category counting:** For each LM category (including Uncertainty), the count of matching tokens per speaker turn is computed via sparse matrix column slicing: `X[:, indices].sum(axis=1)`, stored as `{Category}_count` (Tokenize L949-954).
4. **Total token count:** `total_tokens` = count of all alphabetic tokens per speaker turn, computed via regex `r"(?u)\b[a-zA-Z]+\b"` (Tokenize L958-959).
5. **Output:** Year-partitioned `linguistic_counts_{year}.parquet` files with columns: `file_name`, `speaker_number`, `context`, `role`, `employer`, `speaker_name`, `{Category}_count` (x7), `total_tokens`.

**Stage 2.2 -- Variable construction** (`src/f1d/text/build_linguistic_variables.py`):

6. **Speaker flagging:** Each speaker turn is classified as `is_manager`, `is_ceo`, `is_analyst`, `is_operator` based on role/employer metadata and a regex pattern matched against the manifest (Build L533).
7. **Sample x Context aggregation:** Five speaker samples (Manager, Analyst, CEO, NonCEO_Manager, Entire) are crossed with three context segments (QA, Pres, All), producing 15 combinations. For each combination, the `aggregate_weighted()` function (Build L472-502) groups by `file_name`, sums `{Category}_count` and `total_tokens` across all matching speaker turns, then computes: `{Category}_pct = ({Category}_count / total_tokens) * 100.0` (Build L496). Division by zero is handled via `.replace(0, np.nan)` on total_tokens (Build L490).
8. **Column naming:** The resulting column is named `{Sample}_{Context}_{Category}_pct` -- e.g., `CEO_QA_Uncertainty_pct` = (sum of Uncertainty word counts across all CEO speaker turns in the QA section) / (sum of total tokens across all CEO speaker turns in the QA section) * 100.
9. **NaN semantics:** NaN means "no text in that section" (no speaker turns matched the sample/context filter). Zero means "text present but no LM Uncertainty words found" (Build L589).
10. **Output:** Year-partitioned `linguistic_variables_{year}.parquet` files.

**Stage 3 -- LinguisticEngine winsorization and builder loading:**

11. **LinguisticEngine** (`_linguistic_engine.py` L145-264): Loads all year-partitioned `linguistic_variables_{year}.parquet` files. Applies per-year winsorization at **0%/99% (upper-only)** to all `_pct` columns via `winsorize_by_year(combined, existing_pct_cols, year_col="year", lower=0.0, upper=0.99, min_obs=10)` (Engine L255-258). The lower bound of 0.0 means no lower clipping (these variables are naturally non-negative). The upper bound of 0.99 clips the top 1% of values within each year.
12. **Individual builder:** `CEOQAUncertaintyBuilder` (and analogously for the other three) queries the `LinguisticEngine` singleton via `get_engine().get_data(root_path, years)` and extracts `["file_name", "CEO_QA_Uncertainty_pct"]` (ceo_qa_uncertainty.py L29-50).

**Stage 3 -- Panel builder aggregation to firm-year (Avg_ prefix):**

13. **Call-level merge:** Builder L264-269 merges each IV into the call-level panel on `file_name` (left join, `delta == 0` assertion).
14. **Fiscal year attachment:** `attach_fyearq(panel, root_path)` (Builder L274) assigns `fyearq` to each call.
15. **Arithmetic mean aggregation:** `aggregate_to_firm_year()` (Builder L110-173) computes: `df.groupby(["gvkey", "fyearq"])[existing_ivs].mean()` (Builder L157-158). The result is renamed with the `Avg_` prefix: `{col: f"Avg_{col}" for col in existing_ivs}` (Builder L160). This yields one value per firm-fiscal-year, representing the arithmetic mean of the call-level uncertainty percentage across all quarterly earnings calls in that fiscal year.
16. **No additional winsorization:** The `Avg_` prefixed IVs are not further winsorized after aggregation. They inherit only the per-year 0%/99% upper-only winsorization applied at the LinguisticEngine level (step 11).

## 11. Identification & Inference Assessment

### 11a. Estimator-DV mismatch concerns

**PayoutRatio has no upper-bound censoring.** The variable is defined as dvy / iby. When a firm pays dividends exceeding its income (e.g., drawing on retained earnings), PayoutRatio exceeds 1.0. When iby is positive but very small, PayoutRatio can be extremely large. The implementation relies on 1%/99% per-year winsorization (engine level) to mitigate extreme values, but if many firms have PayoutRatio > 1 in a given year, the 99th percentile itself may be well above 1.0. Attig et al. (2013), the cited reference, typically census or bound payout ratios. OLS is applied to a variable that is (a) bounded below by zero, (b) unbounded above, and (c) has a potentially long right tail. This raises concerns about normality of residuals and the influence of extreme observations on coefficient estimates.

**Mass point at zero.** When dvy is NaN (treated as zero via `fillna(0)`) or truly zero, and iby > 0, PayoutRatio equals zero. Non-dividend-paying firms with positive earnings thus create a mass point at the lower bound. OLS on a dependent variable with a substantial mass at zero is econometrically questionable. A Tobit model or two-part model (first stage: payer vs non-payer; second stage: conditional payout intensity) would be theoretically more appropriate. The predecessor DV (DivIntensity) had this same issue (noted as J3: "45.4% zeros"), and the switch to PayoutRatio does not fully resolve it -- the mass point merely shifts from dividends/assets to dividends/income for the same set of non-payer firms.

**Recommendation for robustness:** Consider (a) censoring PayoutRatio at 1.0, (b) reporting Tobit or fractional logit (Papke and Wooldridge 1996/2008) results, or (c) restricting the sample to dividend payers only.

### 11b. ROA-PayoutRatio denominator overlap (Kronmal 1993)

ROA = iby_annual / avg_assets. PayoutRatio = dvy_annual / iby_annual. Both share `iby_annual` -- ROA in the numerator, PayoutRatio in the denominator. This creates a mechanical negative partial correlation: holding dvy constant, higher iby increases ROA and decreases PayoutRatio. This is the classic "spurious correlation from shared components" problem (Kronmal 1993, JRSS-A).

**Practical implications:** The ROA coefficient in the PayoutRatio regression may be biased toward negative values due to this mechanical linkage, potentially inflating or deflating the apparent effect of uncertainty IVs depending on the correlation structure. A sensitivity analysis dropping ROA from controls, or replacing ROA with a rank-transformed version (breaking the shared-denominator linkage), would strengthen the identification claim.

### 11c. Multicollinearity among IVs

Four uncertainty IVs enter simultaneously. CEO QA and Manager QA uncertainty are likely correlated; similarly for the Pres pair. No variance inflation factors (VIFs) or IV correlation matrices are reported. High multicollinearity would inflate standard errors without biasing coefficients, but could make individual IV significance unreliable. The design choice to include all four simultaneously is defensible for completeness but should be accompanied by VIF reporting.

**CEO-subset-of-Manager collinearity (RT-D3).** The Manager sample includes CEO speakers (`is_manager == True` includes `is_ceo == True`). In firms where the CEO is the only or dominant management speaker, `Manager_QA_Uncertainty_pct` and `CEO_QA_Uncertainty_pct` will be nearly identical (and likewise for the Pres pair). This creates near-mechanical collinearity between the CEO and Manager IV pairs, which can make individual IV coefficients unreliable and unstable across specifications. Reporting VIFs or running separate IV-pair regressions (CEO-only, Manager-only) as robustness checks would strengthen the identification claim.

### 11d. Fixed effects

Industry FE (FF12 dummies) absorbs time-invariant industry heterogeneity. Firm FE absorbs all time-invariant firm heterogeneity (strictly more demanding). Both are crossed with fiscal-year FE, which absorbs common time shocks. No Hausman test for RE vs FE is reported, which is standard practice in applied corporate finance.

### 11e. Clustering

Firm-level clustering is appropriate given the firm-year panel structure. Serial correlation within firms is the primary concern. The panel is unbalanced, which does not affect the validity of clustered SEs but may reduce power for the lead DV specifications (which require consecutive years).

## 14. Design Decisions & Rationale

### 14a. Redesign from old H12

The old H12 used `DivIntensity = dvy / atq` (dividends / total assets) as the DV. This was replaced with `PayoutRatio = dvy / iby` (Attig et al. 2013) for two reasons: (1) PayoutRatio is the standard payout measure in the corporate finance literature, and (2) DivIntensity is more properly an "asset-scaled dividend intensity" that conflates payout decisions with firm size.

**Legacy and variant variables still in engine:** The engine still computes `DivIntensity = dvy_annual / atq_annual_lag1` (Engine L1096-1103), the old H12 DV, as well as `PayoutRatio_q = (dvpspq * cshoq) / ibq` (Engine L1116-1125), a quarterly variant. Both are in `COMPUSTAT_COLS` and `ratio_cols` and are winsorized, but neither is used in the current H12 runner. They persist for backward compatibility and potential alternative specifications.

Additional redesign changes from the old H12:

| Change | Old H12 | New H12 |
|--------|---------|---------|
| DV | DivIntensity (dvy/atq) | PayoutRatio (dvy/iby) per Attig et al. |
| IVs | Single IV per regression | 4 simultaneous IVs |
| FE structure | Firm + Year FE only | Industry/Firm crossed with FiscalYear FE |
| Sample | 3-sample design (Main/Utility/Finance) | Main sample only |
| Time FE index | Calendar year (bug) | Fiscal year (fyearq_int) |
| Firm filter | MIN_CALLS_PER_FIRM (misleadingly named) | Removed -- include all firms |
| Extended controls | None | Added SalesGrowth, RD_Intensity, CashFlow, Volatility |

### 14b. PayoutRatio construction choices

| Decision | Choice | Rationale |
|----------|--------|-----------|
| dvy NaN -> 0 | `fillna(0)` before division | Firms with positive earnings but no reported dividends are treated as zero-payout. Conservative: avoids dropping these observations. |
| iby <= 0 -> NaN | Exclude from regression | Negative-earnings firms produce economically meaningless payout ratios (negative or sign-reversed). Standard in the literature. |
| No upper censoring at 1.0 | PayoutRatio left unbounded above | Winsorization at 99th percentile provides some protection. **Known limitation L1.** |
| Q4 cumulative for both dvy and iby | `_compute_annual_q4_variable` | Compustat dvy and iby are YTD cumulative; Q4 value gives the full fiscal year total. |

### 14c. Aggregation rule

IVs are averaged across all quarterly calls within a firm-fiscal-year. This follows Brochet, Loumioti & Serafeim and Hassan, Hollander, van Lent & Tahoun (QJE), both of whom aggregate call-level textual measures to the annual level via arithmetic mean. Financial variables use `groupby.last()` because they are constant within firm-year (sourced from the same Compustat Q4 observation).

### 14d. Lead variable construction

`PayoutRatio_lead` = PayoutRatio at t+1, constructed via `shift(-1)` within each firm (sorted by fyearq). Non-consecutive fiscal years produce NaN (Builder L188-191: `(next_fyearq_int - fyearq_int) == 1`). This prevents incorrect assignment of payout ratios from non-adjacent years and avoids survivorship bias from requiring panel continuity.

### 14e. DividendPayer exclusion

`DividendPayer` is explicitly excluded from controls (Runner L98) because it is endogenous with the DV: PayoutRatio = 0 if and only if the firm is a non-payer (given iby > 0), making the two variables mechanically dependent.

### 14f. No MIN_CALLS_PER_FIRM filter

The old H12 used a `MIN_CALLS_PER_FIRM` filter that was misleadingly named (it filtered firm-years, not calls per firm). This was removed entirely: all firms are included regardless of how many calls or firm-years they contribute to the panel. Firm FE specifications absorb firm-level heterogeneity without requiring a minimum panel length.
