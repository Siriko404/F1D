| **Companion Suite** | H13.1 (competition-moderated capex) uses the same panel (`h13_capex_panel.parquet`) and tests whether product market concentration (TNIC3HHI) moderates the uncertainty-capex relationship via interaction terms. H13 establishes main effects; H13.1 tests moderation. |

**Critical structural note -- Moulton problem.** The dependent variables (`CapexAt`, `CapexAt_lead`) are constant within firm-fiscal-year because they are computed from Q4 annual Compustat data joined to all calls in that fiscal year. The independent variables (uncertainty percentages) vary at the call level. This creates a Moulton (1990) pseudoreplication problem: the regression treats each call as an independent observation when the DV has no within-firm-year variation. The effective sample size for the DV is the number of firm-years, not the number of calls. Firm-clustered standard errors correct for within-firm serial correlation across years but do not fully address within-firm-year pseudoreplication. A firm-year-collapsed robustness check has not been implemented. See Sections 11 and 12 for further discussion.

## 6. Variable Dictionary

### 6.1 Dependent Variables

#### 6.1.1 `CapexAt` -- Capital Expenditure Intensity (Contemporaneous)

| Field | Detail |
|-------|--------|
| **Formula** | `capxy_annual_Q4 / atq_annual_lag1` |
| **Timing** | Fiscal year t (same year as the earnings call) |
| **Code** | `_compustat_engine.py` lines 1081-1087 |

**Complete construction chain:**

1. **Numerator -- `capxy_annual_Q4`:** Computed by `_compute_annual_q4_variable(comp, "capxy", "_capxy_annual")` (line 1081). `capxy` is a YTD-cumulative Compustat field: Q1 holds only Q1 capital expenditure, Q2 holds Q1+Q2, and Q4 holds the full fiscal year total. The function (lines 234-274) filters to rows where `fqtr == 4`, drops rows with null `fyearq`, casts `fyearq` to `int`, deduplicates by `(gvkey, fyearq)` keeping the last row (most recent restatement), then merges the Q4 value back to ALL quarterly rows by `(gvkey, fyearq)` via a left join with `validate="m:1"`. Every call in the same fiscal year receives the identical Q4 `capxy` value.

2. **Denominator -- `atq_annual_lag1`:** Computed by `_compute_annual_q4_variable_lag(comp, "atq", "_atq_annual_lag1")` (line 1054, reused at line 1083). The function (lines 277-304) follows the same Q4-filtering, deduplication, and join pattern as the numerator, but with one additional step: after extracting Q4 `atq` by `(gvkey, fyearq)`, it adds `+1` to `fyearq` (line 293). This shifts the fiscal year label forward, so that fiscal year t-1's Q4 `atq` joins to fiscal year t's rows. The result is lagged total assets: the denominator for period t is period t-1's end-of-year total assets.

3. **Division and NaN guard:** `CapexAt = np.where(atq_annual_lag1 > 0, capxy_annual / atq_annual_lag1, np.nan)` (lines 1082-1086). When lagged total assets are zero, negative, or missing, the result is NaN.

4. **Inf replacement:** After all ratio computations, `comp["CapexAt"].replace([np.inf, -np.inf], np.nan)` (line 1161 in the `ratio_cols` loop at lines 1152-1172).

5. **Winsorization:** Per-fiscal-year 1%/99% winsorization via `_winsorize_by_year(comp["CapexAt"], comp["fyearq"])` (lines 1194-1201). Minimum 10 observations per year-group required; groups smaller than 10 are left unchanged.

6. **Manifest matching:** `CompustatEngine.match_to_manifest()` (lines 1268-1302) performs a `merge_asof` backward on `(gvkey, start_date -> datadate)` to attach the nearest-prior Compustat quarter's computed `CapexAt` to each earnings call. Because `CapexAt` was joined back to all quarters within a fiscal year in step 1, the asof match selects the most recent quarterly Compustat observation before the call date, which carries the Q4-derived annual `CapexAt` for that fiscal year.

#### 6.1.2 `CapexAt_lead` -- Capital Expenditure Intensity (Forward, t+1)

| Field | Detail |
|-------|--------|
| **Formula** | `CapexAt` for fiscal year t+1, constructed via shift within firm |
| **Timing** | Forward one fiscal year |
| **Code** | `build_h13_capex_panel.py` lines 215-337 (`create_capex_lead`) |

**Complete construction chain:**

1. **Attach `fyearq`:** `attach_fyearq(panel, root_path)` (line 251) performs a `merge_asof` backward from the call's `start_date` to Compustat's `datadate`, attaching the Compustat fiscal year (`fyearq`) to each call. An integer version `fyearq_int` is created via `pd.to_numeric` (line 264).

2. **Select representative call per firm-year:** For each `(gvkey, fyearq_int)`, the call with the latest `start_date` is identified via `groupby(["gvkey", "fyearq_int"])["start_date_dt"].idxmax()` (line 280). The `CapexAt` value from that call is extracted into a firm-year lookup table `firm_year_eoy`. Since `CapexAt` is constant within firm-fiscal-year (all calls receive the same Q4-derived value), the `idxmax` selection is inconsequential -- it produces the same `CapexAt` regardless of which call is selected.

3. **Shift to create lead:** `firm_year_eoy` is sorted by `(gvkey, fyearq_grp)` (line 291). Within each `gvkey`, `fyearq_grp` and `CapexAt_eoy` are shifted by -1 (lines 294-299), producing the next fiscal year's `CapexAt` for each firm-year row.

4. **Consecutive-year validation:** A boolean mask checks whether `fyearq_lead == fyearq_grp + 1` (line 302). If fiscal years are not consecutive (gaps or last observed year), `CapexAt_lead` is set to NaN via `np.where` (lines 303-304). This prevents spurious leads across multi-year gaps.

5. **Merge back to call level:** The lead lookup is merged back onto the full call-level panel by `(gvkey, fyearq_int)` via a left join (line 322), with a row-count guard asserting no fan-out (lines 324-328). All calls in the same firm-fiscal-year receive the same lead value.

**Moulton note:** Both DVs are constant within firm-fiscal-year. All calls within the same `(gvkey, fyearq_int)` receive the identical DV value. The DV has no within-firm-year variation.

### 6.2 Key Independent Variables (4, all enter simultaneously)

#### 6.2.1 Complete Construction Chain for All Textual IVs

All four uncertainty IVs share the same three-stage pipeline. The variable name follows the pattern `{Speaker}_{Context}_Uncertainty_pct`, where Speaker is `CEO` or `Manager` and Context is `QA` or `Pres`.

| Variable | Speaker Filter | Context Filter | Builder Class |
|----------|---------------|----------------|---------------|
| `CEO_QA_Uncertainty_pct` | `is_ceo = True` (tiered: exact name match on `ceo_name` / `prev_ceo_name`, then last-name fallback) | `context == "QA"` | `CEOQAUncertaintyBuilder` |
| `CEO_Pres_Uncertainty_pct` | `is_ceo = True` (same tiered logic) | `context == "Pres"` | `CEOPresUncertaintyBuilder` |
| `Manager_QA_Uncertainty_pct` | `is_manager = True` (not analyst, not operator; role keyword match OR employer == company name) | `context == "QA"` | `ManagerQAUncertaintyBuilder` |
| `Manager_Pres_Uncertainty_pct` | `is_manager = True` (same logic) | `context == "Pres"` | `ManagerPresUncertaintyBuilder` |

**Stage 2.1 -- Tokenization and counting** (`src/f1d/text/tokenize_transcripts.py`):

1. **Dictionary loading:** `load_lm_dictionary()` (lines 768-812) reads the Loughran-McDonald Master Dictionary CSV file (`inputs/Loughran-McDonald_MasterDictionary_1993-2024.csv`). Words are uppercased. Seven sentiment categories are extracted: `Negative`, `Positive`, `Uncertainty`, `Litigious`, `Strong_Modal`, `Weak_Modal`, `Constraining`. For each category, words with a count > 0 in that category column are collected into a set. A vocabulary list is built from the union of all categorized words.

2. **Tokenization:** `tokenize_batch()` (lines 901-984) processes speaker-level text. Each speaker turn's `speaker_text` is uppercased. A `sklearn.CountVectorizer` with `vocabulary=vocab_list` and `token_pattern=r"(?u)\b[a-zA-Z]+\b"` (alphabetic tokens only) transforms each text into a sparse document-term matrix. For each LM category (including `Uncertainty`), the counts of all words in that category are summed across columns to produce `{Category}_count` per speaker-turn row. A separate `total_tokens` column counts all alphabetic tokens (not just dictionary words) via regex `r"(?u)\b[a-zA-Z]+\b"`.

3. **Output:** Per-year parquet files `linguistic_counts_{year}.parquet` with columns: `file_name`, `speaker_number`, `context`, `role`, `employer`, `speaker_name`, `{Category}_count` (7 categories), `total_tokens`.

**Stage 2.2 -- Variable construction** (`src/f1d/text/build_linguistic_variables.py`):

4. **Speaker flagging:** `flag_speakers()` (lines 420-464) classifies each speaker-turn row as `is_analyst`, `is_operator`, `is_manager`, or `is_ceo` using role keywords and CEO name matching (exact full-name match or last-name fallback against `ceo_name` / `prev_ceo_name` from the manifest).

5. **Weighted aggregation:** `aggregate_weighted()` (lines 472-502) filters to rows matching both a speaker mask (e.g., `is_ceo`) and a context mask (e.g., `context == "QA"`). Within each `file_name`, it sums `{Category}_count` and `total_tokens` across all matching speaker turns. The percentage is computed as `(sum(Uncertainty_count) / sum(total_tokens)) * 100.0`, with `total_tokens == 0` replaced by NaN to avoid division by zero. The result column is named `Uncertainty_pct`, prefixed with speaker/context to produce e.g. `CEO_QA_Uncertainty_pct`.

6. **Output:** Per-year parquet files `linguistic_variables_{year}.parquet` with one row per `file_name` and columns for all speaker-context-category percentage combinations.

**Stage 3 -- Winsorization and loading** (`_linguistic_engine.py`):

7. **Loading:** `LinguisticEngine.get_data()` (lines 165-264) concatenates all year files from `outputs/2_Textual_Analysis/2.2_Variables/latest/`. The engine is a cached singleton shared across all linguistic builders.

8. **Winsorization:** All `_pct` columns are winsorized per-year at 0%/99% (upper-tail only) via `winsorize_by_year()` with `lower=0.0, upper=0.99, min_obs=10` (lines 255-258). The lower bound of 0% means no lower clipping is applied (uncertainty percentages are naturally non-negative). The upper bound clips at the 99th percentile within each call year. This is a single-pass winsorization; no further winsorization is applied at any later stage.

9. **Builder extraction:** Each builder (e.g., `CEOQAUncertaintyBuilder`, lines 18-53 of `ceo_qa_uncertainty.py`) queries the singleton engine, extracts the `[file_name, {Column}_pct]` columns, and returns a `VariableResult`. The builder performs no additional transformation.

10. **Panel merge:** In `build_h13_capex_panel.py`, each builder's output is left-joined onto the manifest by `file_name` with zero-row-delta enforcement (lines 185-192).

#### 6.3.2 `TobinsQ`

| Field | Detail |
|-------|--------|
| **Formula** | `(cshoq * prccq + debt_book) / atq` |
| **Raw fields** | `cshoq` (shares outstanding), `prccq` (price close), `dlcq` (short-term debt), `dlttq` (long-term debt), `atq` (total assets) |
| **Code** | `_compustat_engine.py` lines 1069-1079 |

**Construction chain:** (1) Market capitalization: `mktcap = cshoq * prccq` (line 1069). (2) Debt components clipped at zero: `debt_c = dlcq.clip(lower=0).fillna(0)` and `debt_t = dlttq.clip(lower=0).fillna(0)` (lines 1070-1071). Negative book debt is treated as zero; missing individual debt components are filled with zero. (3) Both-null guard: `debt_book = np.where(comp["dlcq"].isna() & comp["dlttq"].isna(), np.nan, debt_c + debt_t)` (lines 1072-1073). If BOTH `dlcq` and `dlttq` are null, `debt_book` is NaN (not zero), preventing a false "no debt" assumption. (4) Division with guards: `TobinsQ = np.where(atq.notna() & (atq > 0) & mktcap.notna(), (mktcap + debt_book) / atq, np.nan)` (lines 1075-1078). NaN when `atq` is null/non-positive or `mktcap` is null. (5) Inf replaced with NaN. (6) Per-fiscal-year 1%/99% winsorization. (7) Matched to calls via `merge_asof`.

**Important divergence from textbook:** This is NOT `(atq + mktcap - ceqq) / atq`. The code computes market capitalization of equity plus non-negative book debt, divided by total assets. `ceqq` (common equity) is not used. The `__init__.py` docstring (line 21) documents the textbook formula, not the actual computation. The implemented formula is a defensible market-value-of-assets proxy commonly used in empirical finance but differs from textbook Tobin's Q by construction. See L03.

|-------|--------|

| **Raw fields** | `iby` (Income Before Extraordinary Items, annual YTD), `atq` (Total Assets, quarterly) |
| **Code** | `_compustat_engine.py` lines 1052-1062 |

(2) `atq_annual_lag1` (Q4 total assets for fiscal year t-1) computed via `_compute_annual_q4_variable_lag(comp, "atq", ...)` (line 1054). (3) `avg_assets = (atq_annual + atq_annual_lag1) / 2` (lines 1055-1058). (4) `iby_annual` (Q4 annual income) computed via `_compute_annual_q4_variable(comp, "iby", ...)` (line 1059). (5) `ROA = np.where(avg_assets > 0, iby_annual / avg_assets, np.nan)` (lines 1060-1061). (6) Inf replaced with NaN. (7) Per-fiscal-year 1%/99% winsorization. (8) Matched to calls via `merge_asof`.

#### 6.3.4 `BookLev`

| Field | Detail |
|-------|--------|
| **Formula** | `(dlcq.fillna(0) + dlttq.fillna(0)) / atq` |
| **Raw fields** | `dlcq` (short-term debt), `dlttq` (long-term debt), `atq` (total assets) |
| **Code** | `_compustat_engine.py` line 1041 |

**Construction chain:** (1) Missing debt components individually filled with zero: `dlcq.fillna(0) + dlttq.fillna(0)`. This treats missing short-term or long-term debt as zero, which is defensible but not identical to the TobinsQ both-null guard. A firm with both `dlcq` and `dlttq` null will have `BookLev = 0/atq = 0`, not NaN. (2) Division by `atq` (no explicit non-positive guard -- NaN propagates if `atq` is NaN; produces inf if `atq == 0`). (3) Inf replaced with NaN (line 1172). (4) Per-fiscal-year 1%/99% winsorization. (5) Matched to calls via `merge_asof`.

#### 6.3.7 `OCF_Volatility`

| Field | Detail |
|-------|--------|
| **Formula** | Rolling 5-year standard deviation of `oancfy / atq_lag` per gvkey |
| **Raw fields** | `oancfy` (Operating Cash Flow, annual YTD), `atq` (Total Assets), `fqtr`, `fyearq` |
| **Code** | `_compustat_engine.py` lines 307-356 (`_compute_ocf_volatility`) |

**Construction chain:** (1) Build annual Q4-only panel: filter to `fqtr == 4`, drop null `fyearq`, cast to int, deduplicate by `(gvkey, fyearq)` keeping last row (lines 317-324). (2) Compute within-gvkey lagged assets: `atq_lag = annual.groupby("gvkey")["atq"].shift(1)` (line 327). Note: this uses a simple positional shift on the sorted annual panel, NOT `_compute_annual_q4_variable_lag`. No consecutive-year validation is applied (gap years receive NaN `atq_lag` only if the prior row belongs to a different gvkey or is the first observation). (3) OCF ratio: `oancfy / atq_lag` when `atq_lag > 0`, else NaN (lines 329-333). Inf replaced with NaN (line 334). (4) Create dummy dates from `fyearq` (e.g., "2020-12-31") and set as DatetimeIndex (lines 336-337). (5) Rolling standard deviation: `groupby("gvkey")["ocf_ratio"].transform(lambda x: x.rolling("1826D", min_periods=3).std())` (lines 338-340). The `1826D` window spans approximately 5 calendar years; `min_periods=3` requires at least 3 annual observations. (6) Join back to all quarterly rows by `(gvkey, fyearq)` via left merge with `validate="m:1"` (lines 349-354). (7) Per-fiscal-year 1%/99% winsorization at engine level. (8) Matched to calls via `merge_asof`.

**Note on `CapexAt` exclusion from controls:** `CapexAt` is not in `BASE_CONTROLS` or `EXTENDED_CONTROLS` (runner lines 93-108). This is by design -- it is the DV family. Including it as a control when DV = `CapexAt_lead` would create near-tautological regression given high autocorrelation in CapEx intensity.

#### 6.4.1 `SalesGrowth`

### 7.5 MultiIndex Structure

The panel is indexed by `["gvkey", "fyearq_int"]` at runner line 312. Because the unit of observation is an individual earnings call (not a firm-year), this index has duplicate entries -- multiple calls within the same `(gvkey, fyearq_int)`. `PanelOLS` handles this by treating each row as a separate observation. Consequences:

1. `EntityEffects` demeans within gvkey (across all calls for that firm, not within firm-year).
2. `TimeEffects` demeans within fiscal year (across all calls in that year).
3. The "within R-squared" reported by `linearmodels` is not the traditional balanced-panel within R-squared. It is the R-squared after entity/time demeaning of call-level observations.

## 11. Identification & Inference Assessment

### 11.1 Moulton (1990) Pseudoreplication Problem (CRITICAL)

**Nature of the problem.** `CapexAt` and `CapexAt_lead` are constant within firm-fiscal-year. Each firm-year maps to approximately 1-3 earnings calls. The regression treats each call as an independent observation, but the DV has zero within-firm-year variation. The 4 IVs (uncertainty percentages) vary at the call level. This is a classic Moulton problem: standard errors are downward-biased because the effective number of independent DV observations equals the number of firm-years, not calls.

**Current mitigation.** Firm-clustered standard errors (`cluster_entity=True`) correct for arbitrary within-firm serial correlation across years. This partially addresses the Moulton problem by accounting for correlated residuals within firms. However, firm-clustering treats all calls within a firm (across years) as correlated, which is a broader correction than the specific within-firm-year pseudoreplication issue.

**What is NOT addressed.** (a) Firm-clustered SEs do not specifically model the within-firm-year inflation factor. The Moulton correction factor depends on the intra-cluster correlation of the regressors and the ratio of cluster size to total observations, which firm-level clustering may under- or over-correct for at the firm-year level. (b) No firm-year-collapsed robustness check has been implemented. (c) No multi-way clustering (firm + fiscal year) has been implemented.

**Referee risk.** A hostile referee would ask: "Why not collapse to firm-year level, average the IVs, and re-estimate? This eliminates the pseudoreplication entirely and provides a clean test of the hypothesis." The author should be prepared to respond.

### 11.2 Industry FE Significance vs. Firm FE Insignificance

All significance at p < 0.05 appears exclusively in Industry FE specifications (Cols 1, 3, 5, 7) and disappears completely in Firm FE specifications (Cols 2, 4, 6, 8). This pattern has two possible interpretations:

1. **Between-firm variation drives results.** The uncertainty-capex relationship reflects cross-sectional differences between industries in both uncertainty language norms and capital intensity. Once firm-level heterogeneity is absorbed, no within-firm relationship remains.

2. **Omitted variable bias under Industry FE.** Industry composition correlates with both uncertainty language conventions and capital intensity. The within-firm estimator (Firm FE) removes this confound; the fact that significance disappears is consistent with the Industry FE estimates reflecting omitted firm-level factors rather than a causal relationship.

The two interpretations are empirically indistinguishable without additional instruments or identification strategies.

### 11.3 IV Exogeneity and Timing (G01)

The 4 IVs (uncertainty percentages) are measured from earnings calls that occur during or after the fiscal year whose CapEx is measured. For `CapexAt` (contemporaneous), the call may occur AFTER Q4 CapEx decisions are already made, creating a simultaneity/reverse-causality concern: a firm that has already committed to high CapEx may discuss it with less (or more) uncertain language. The contemporaneous specifications (Cols 1-4) cannot distinguish whether uncertainty language causes CapEx decisions or reflects them.

### 11.4 Lead DV as Cleaner Identification (G02)

For `CapexAt_lead` (t+1), the speech occurs before or during the CapEx measurement period, which is a cleaner identification setting. The lead-DV specifications (Cols 5-8) should be emphasized as the more credible tests of the hypothesis, as they better satisfy the temporal precedence requirement for causal inference. The contemporaneous specs (Cols 1-4) are useful as a benchmark but are more vulnerable to simultaneity concerns.

### 11.5 Survivorship Bias in Lead Construction (G03)

Firms that exit the sample (delisting, merger, bankruptcy) before year t+1 receive NaN for `CapexAt_lead` and are dropped from the lead-DV estimation sample. These exits are non-random -- distressed firms likely have higher uncertainty language AND are more likely to exit. This creates a sample selection bias in the lead-DV specifications (Cols 5-8): the estimated relationship between uncertainty and future CapEx is conditional on firm survival, which may attenuate or distort the true effect.

### 11.6 `check_rank=False` in Industry FE Specifications

Industry FE specifications set `check_rank=False` (runner line 327), which suppresses rank-deficiency warnings from `PanelOLS`. If any FF12 industry dummies are collinear with controls or time effects (e.g., if all firms in one FF12 industry enter the sample in a single fiscal year), coefficient estimates may be unreliable for those industry-year cells. The `drop_absorbed=True` setting removes perfectly collinear columns, but near-collinearity can still inflate standard errors.

## 14. Design Decisions & Rationale

| ID | Decision | Rationale |
|----|----------|-----------|
| D01 | **Two-tailed test** | No strong prior on direction of the uncertainty-capex relationship. Uncertainty could increase CapEx (option-value of investment under uncertainty) or decrease it (precautionary delay of irreversible investment). |
| D02 | **CapexAt excluded from ALL controls** | CapexAt is the DV family. Including it as a control when DV = CapexAt_lead would create near-tautological regression given high autocorrelation in CapEx intensity. Follows the H4 pattern where BookLev is excluded even when DV = BookLev_lead. |
| D03 | **Fiscal year time index** | DV construction uses Q4 `capxy` (a fiscal-year variable). Using `fyearq_int` as the time dimension ensures the time FE aligns with the DV's measurement period. Prior H13.1 used calendar year `year`; this was fixed to `fyearq_int`. |
| D04 | **Call-level estimation despite firm-year-level DV** | The suite tests whether call-level speech measures predict firm-level outcomes. Call-level estimation preserves IV variation across calls within a firm-year (e.g., Q1 vs. Q4 calls may differ in uncertainty language). The Moulton problem is acknowledged (Sections 1, 11.1, 12) but call-level estimation is retained as the primary specification. A firm-year-collapsed robustness check would strengthen the analysis. |
| D05 | **4 simultaneous IVs** | All four uncertainty measures (CEO/Manager x Presentation/QA) enter each model simultaneously to test which source and context of uncertainty language has independent predictive power, controlling for the other three. |
| D06 | **Industry FE via `other_effects`, not C() dummies** | Using the `other_effects` parameter in PanelOLS is computationally more efficient than including C(ff12_code) as dummies in the formula. The `check_rank=False` setting suppresses rank-deficiency warnings that arise when some FF12 categories have few observations. |
| D07 | **Minimum 5 calls per firm** | Ensures sufficient within-firm variation for entity fixed effects estimation. Firms with fewer than 5 calls contribute little to within-firm identification and may introduce noise. |
| D08 | **Complete-case filtering** | All required variables must be non-null for a call to enter the regression. This is standard practice but means the estimation sample may differ across specifications (Base vs. Extended controls) if extended controls have lower coverage. In practice, only 5 calls are lost to extended-control missingness (Col 1 vs. Col 3). |
