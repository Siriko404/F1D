## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H3 -- Managerial Speech Vagueness and Payout Policy |
| Research question | Does managerial speech vagueness (uncertainty language) in earnings calls reduce dividend stability and increase payout flexibility? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `year` (time index, calendar year of call) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |

### A3) Outcome / Target Definition

| Field | Value |
|-------|-------|
| Dependent variables | (1) `div_stability_lead` -- negative of rolling 5-year SD of lagged annual payout ratio (dvy/iby); higher = more stable; (2) `payout_flexibility_lead` -- fraction of trailing 5 years in which \|delta DPS\| > 5% of prior DPS; higher = more flexible |
| Timing convention | Forward-looking (t+1): lead variables are the fiscal-year-ahead value of the contemporaneous measure, constructed via fiscal-year shift within firm |
| Lead construction | Last call per (gvkey, fyearq) anchors the annual value; shift(-1) within gvkey conditional on consecutive fiscal years; merged back to all calls via (gvkey, fyearq) |
| Missing policy | Drop (listwise deletion at regression stage via `dropna(subset=required)`) |

**REFEREE CONCERN -- DV distributional pathology (red-team finding RT-02/G01):** `payout_flexibility_lead` has only **11 unique values** (all multiples of 0.2 from the 5-year rolling window: 0, 0.2, 0.25, 0.33, 0.4, 0.5, 0.6, 0.75, 0.8, 1.0, plus 0 for no changes), with **46.4% at exactly 0.0** and 14.8% at exactly 1.0. This is effectively a discrete/ordinal variable, not continuous. OLS on a heavily zero-inflated discrete DV violates distributional assumptions; standard errors and inference are unreliable. A fractional response model (Papke-Wooldridge) or ordered probit would be more appropriate.

**REFEREE CONCERN -- Within-variation exhaustion (red-team finding RT-03/G02):** 39.3% of firms (951/2,421) have zero within-firm variation in `payout_flexibility_lead`. Only 28.5% of total variance is within-firm. Firm FE absorbs most variation; the regression identifies only from the ~61% of firms with any within-firm DV change. The effective identifying sample is substantially smaller than the reported N.

## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

| Variable group | Method | Thresholds | When applied | Code reference |
|---------------|--------|-----------|--------------|----------------|
| Compustat controls (Size, Lev, ROA, TobinsQ, CashHoldings, div_stability, payout_flexibility, earnings_volatility, fcf_growth, firm_maturity) | Per-year (fyearq) | 1%/99% | During `_compute_and_winsorize()`, before merge to manifest | `_compustat_engine.py` L1138-1161, `_winsorize_by_year()` L436-460 |

| `is_div_payer_5yr` | NOT winsorized | -- | In skip_winsorize set | `_compustat_engine.py` L1146-1151 |

**VERIFIED IMPLEMENTATION ISSUE:** The Compustat engine winsorizes by fyearq (fiscal year) while the linguistic engine winsorizes by calendar year. These are different year definitions. The mismatch is minor but creates inconsistency in the winsorization base for the same call.

### Missing data policy

- **Listwise deletion:** `prepare_regression_data()` calls `dropna(subset=required)` where `required` includes the DV, uncertainty measure, Lev, is_div_payer_5yr, and all controls. Any row with a missing value in any of these is dropped.
- **Inf handling at regression stage:** `replace([np.inf, -np.inf], np.nan)` applied before dropna.
- **VERIFIED ISSUE -- H3 vars excluded from engine-level inf cleanup (red-team finding G05):** The `ratio_cols` list in `_compute_and_winsorize()` (lines 1115-1131) does NOT include `div_stability`, `payout_flexibility`, `earnings_volatility`, `fcf_growth`, or `firm_maturity`. These H3-specific variables rely solely on the regression script's `replace([np.inf, -np.inf], np.nan)` for inf handling. **Verified:** no inf values currently exist in these variables in the panel artifact, so practical impact is nil, but this is a defensive gap.
- **Denominator protections:** Throughout `_compute_h3_payout_policy()`: `np.where(atq > 0, ..., np.nan)` for all ratios; `np.where(fcf_lag != 0, ..., np.nan)` for growth; `np.where(iby > 0, ..., np.nan)` for payout ratio.
- **fillna(0) on flow variables:** `oancfy.fillna(0)` and `capxy.fillna(0)` in FCF computation (line 827). This treats missing cash flow/capex as zero, which may be inappropriate if the data is truly missing rather than zero.

### Scaling

- `Size` = ln(atq): only log transform in the suite.
- No standardization of controls despite table note claiming "All continuous controls are standardized" (see Issue I-01).

## J. Known Issues / Ambiguities (Actionable)

### J-01: Table note claims standardized controls -- FALSE

The LaTeX table note at line 407 of `run_h3_payout_policy.py` states "All continuous controls are standardized." No standardization code exists anywhere in the estimation pipeline. Controls enter the regression in their raw (winsorized) units. **Impact:** Misleading table note; coefficients are not comparable across controls as implied.

**Fix:** Either standardize controls before regression (subtract mean, divide by SD) or remove the claim from the table note.

### J-02: firm_maturity formula mismatch with docstring

The `_compute_h3_payout_policy()` docstring (line 775) says `firm_maturity: RE / TE (retained earnings / total equity)`. The actual code (line 839-841) computes `req / atq` (retained earnings / total assets). Total equity would be `seqq`, not `atq`.

**Impact:** Variable documentation is incorrect relative to implementation. The implemented formula (RE/AT) is a standard measure (DeAngelo, DeAngelo, and Stulz 2006 use RE/TA), so the code is likely correct and the docstring wrong.

**Severity:** Low. Trivial documentation fix.

**Fix:** Correct the docstring to say "RE / TA (retained earnings / total assets)."

### J-03: fcf_growth extreme values despite winsorization

After per-year 1%/99% winsorization, `fcf_growth` still ranges from -29.91 to +29.81. The p1/p99 bounds are -9.18/+16.58. This suggests either (a) the year-level winsorization is ineffective because tail values are concentrated in specific years, or (b) the 1826D rolling window creates lags that are not well-captured by year-level clipping.

**Impact:** Extreme values may dominate regression estimates for this control. OLS is not robust to such outliers.

**Fix:** Consider tighter winsorization (e.g., 5%/95%) for fcf_growth, or use a log(1+|FCF|)*sign(FCF) transform.

### J-04: div_stability has mass point at zero

34% of non-missing div_stability values are exactly 0 (non-payers with zero payout ratio SD). Even among the is_div_payer_5yr==1 subsample, 1,843 (3.3%) are exactly zero. The lead DV (div_stability_lead) inherits this mass point.

**Impact:** A point mass at zero in the DV can distort OLS inference. A Tobit or hurdle model might be more appropriate.

### J-05: Econometric output exists only in archived directory

The primary `outputs/econometric/h3_payout_policy/` directory does not exist. However, archived econometric outputs with full 36-model results exist at `outputs/econometric/_archived/h3_payout_policy/`, with 17 timestamped runs. The latest run (2026-03-07_230800) uses the current panel artifact (confirmed via panel_hash in run_manifest.json).

**Impact:** Organizational issue only. A replicator who only checks the primary directory would incorrectly conclude Stage 4 has never been run. No symlink or convention designates the canonical run.

**Severity:** Low (organizational, not a data/methods issue).

**Fix:** Either create a symlink/copy from `_archived/` to the primary directory, or document the archive convention clearly.

### J-06: is_div_payer_5yr uses rolling max, not all-5-years requirement

The docstring for `is_div_payer_5yr` (line 7 of `is_div_payer_5yr.py`) says "1 if the firm paid dividends in each of the previous 5 years." The actual code (line 894-902 of `_compustat_engine.py`) computes a rolling 5-year MAX of the (DPS > 0) indicator. This means a firm that paid dividends in even ONE of the trailing 5 years qualifies, not "each of" the 5 years. This is a direct contradiction between documentation and implementation.

**Impact:** The sample includes firms with sporadic or lapsed dividend payments, not only consistent payers. This broadens the sample beyond what the documentation implies and may dilute the payout policy signal.

**Severity:** Medium. The rolling-max approach is arguably more inclusive and standard in payout literature, but the docstring must be corrected.

**Fix:** Either change the code to use rolling MIN (all 5 years must be payers) or correct the docstring to say "at least one of the previous 5 years."

### J-07: Lev uses fillna(0) for missing debt components

`Lev = (dlcq.fillna(0) + dlttq.fillna(0)) / atq` (line 1032). If both dlcq and dlttq are missing, Lev = 0/atq = 0, which codes a firm with missing debt data as having zero leverage. This creates a false zero rather than a missing value.

**Impact:** Firms with missing debt data are assigned Lev=0 and included in the regression sample, potentially biasing the Lev coefficient downward. 14,156 Lev=0 rows (12.5% of panel); 4,258 among div payers.

**Fix:** Set Lev=NaN when both dlcq and dlttq are missing.

### J-08: payout_flexibility_lead is inherently discrete with 11 unique values (RED-TEAM FINDING)

`payout_flexibility_lead` is constructed from a 5-year rolling window counting DPS changes, producing only k/n fractional values (0, 0.2, 0.25, 0.33, 0.4, 0.5, 0.6, 0.75, 0.8, 1.0, plus 0 for no changes). 46.4% of values are exactly 0.0; 14.8% are exactly 1.0. This is a heavily zero-inflated discrete variable treated as continuous in OLS.

**Impact:** OLS on a DV with only 11 mass points and 46% zeros violates distributional assumptions. Standard errors and coefficient estimates may be unreliable. A fractional response model (Papke and Wooldridge 1996) or ordered probit would be more appropriate.

**Severity:** High.

**Fix:** Use fractional logit/probit for payout_flexibility_lead; at minimum, document the discreteness and discuss OLS appropriateness.

### J-09: Within-firm variation exhaustion for payout_flexibility_lead (RED-TEAM FINDING)

39.3% of firms (951/2,421) have zero within-firm variation in `payout_flexibility_lead`. Only 28.5% of total variance is within-firm. After firm FE demeaning, these firms contribute nothing to identification.

**Impact:** The effective identifying sample is substantially smaller than the reported N. The regression identifies only from the ~61% of firms with any within-firm DV change.

**Severity:** High.

**Fix:** Report the effective number of identifying firms alongside total N; test sensitivity to dropping zero-variation firms.

### J-10: H3-specific Compustat variables excluded from engine-level inf-to-NaN cleanup (RED-TEAM FINDING)

The `ratio_cols` list in `_compute_and_winsorize()` (lines 1115-1131) does NOT include `div_stability`, `payout_flexibility`, `earnings_volatility`, `fcf_growth`, or `firm_maturity`. These rely solely on the regression script's `replace([np.inf, -np.inf], np.nan)` for inf handling.

**Impact:** Currently nil -- no inf values exist in these variables in the panel artifact. Defensive gap only.

**Severity:** Low.

**Fix:** Add H3 vars to ratio_cols for consistency.

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line referee verdict

This suite implements a standard panel FE framework testing whether managerial uncertainty language predicts next-year dividend stability and payout flexibility, conditional on dividend-payer status. The identification strategy relies on within-firm variation in speech vagueness after absorbing firm and year fixed effects, which supports descriptive/predictive association but not causal interpretation. The implementation has several material issues: (1) the table note falsely claims controls are standardized; (2) a key sample filter variable (is_div_payer_5yr) is documented as requiring 5 consecutive years of dividends but actually requires only 1 year; (3) `payout_flexibility_lead` is an inherently discrete variable (11 unique values, 46% zeros) for which OLS is a questionable estimator; (4) 39% of firms have zero within-firm DV variation, substantially reducing the effective identifying sample; (5) no robustness tests are implemented. Archived regression outputs exist (latest: 2026-03-07_230800, 5/36 significant, all in Finance/payout_flexibility) with very low within-R-squared (0.01-0.11).

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

The implementation currently supports: **descriptive association only.**

The write-up is likely to oversell the implemented evidence if causal language is used.

## N. Final Readiness Statement

**Can this suite be reproduced end-to-end from code alone?**
PARTIAL. The panel build (Stage 3) is reproducible and a valid panel artifact exists (2026-03-02). Stage 4 regressions have been run 17 times with archived outputs in `outputs/econometric/_archived/h3_payout_policy/`. The latest run (2026-03-07_230800) used the current panel artifact (confirmed via panel hash). However, the primary output directory is empty, and no convention designates the canonical run.

**Can a committee/referee understand exactly how the variables are built?**
Mostly yes, with two documentation errors: (1) is_div_payer_5yr docstring contradicts the code (rolling max vs all-5-years); (2) firm_maturity docstring says RE/TE but code computes RE/AT. Additionally, the discrete nature of payout_flexibility_lead (11 unique values) is not documented in the variable construction and should be made explicit.

**Are the implemented results trustworthy for descriptive inference?**
Partially. The latest archived run (2026-03-07_230800) shows 5/36 significant results (all in Finance/payout_flexibility), with very low within-R-squared (0.01-0.11). The false standardization claim must be corrected, linguistic controls should be added, and the DV distributional properties (11 unique values, 46% zeros, 39% of firms with zero within-firm variation) should be disclosed and addressed before descriptive claims can be trusted.

**Are they trustworthy for causal inference?**
No. The design (panel FE with lead DV) supports within-firm association only. No IV, no DiD, no exogenous shock exploited. Reverse causality from anticipated dividend decisions to speech patterns is not addressed.

**What is the single biggest threat to credibility?**
The combination of (a) `payout_flexibility_lead` being an inherently discrete variable with only 11 unique values and 46% zero inflation, making OLS inappropriate, and (b) the complete absence of robustness tests across 36 specifications. A thesis with zero robustness battery applied to a DV that violates OLS distributional assumptions would not survive committee review.

**What is the single highest-value next fix?**
Implement a robustness battery (Priority 1), with particular emphasis on testing a fractional response model for payout_flexibility_lead (Priority 2). Without robustness, no result is credible; without an appropriate estimator for the discrete DV, the payout_flexibility results are econometrically questionable.
