## 5. Merge & Sample Construction Audit

### 5.1) Panel Build Merges (Stage 3)

All merges in `build_call_level_panel()` follow the same pattern:

1. **Base:** Manifest (file_name unique, asserted at line 186--191).
2. **Each builder output** merged on `file_name` with `how="left"`.
3. **Zero-row-delta guard:** `after_len != before_len` raises `ValueError` (lines 227--231).
4. **Duplicate file_name guard:** Each builder output checked for duplicates before merge (lines 206--211).
5. **Conflicting column guard:** Overlapping columns (except file_name) dropped from builder data (lines 214--222).

### 5.2) Compustat Matching

- CompustatEngine loads raw Compustat parquet once (singleton pattern, `_engine` at module level).
- `(gvkey, datadate)` deduplicated keeping last row (most recent restatement) -- `_compustat_engine.py` line 1265.
- Individual builders call `engine.match_to_manifest()` which uses `pd.merge_asof` (backward direction) matching call `start_date` to Compustat `datadate` by `gvkey`.
- Result: each call gets the most recent Compustat quarterly observation as of the call date.
- **RT2-4 (No merge tolerance):** Both `match_to_manifest()` and `attach_fyearq()` use `pd.merge_asof` with `direction="backward"` but no `tolerance` parameter. This means a call could theoretically match to a Compustat observation from many years prior if no closer match exists. In practice this is unlikely (most firms have quarterly data), but a `tolerance=pd.Timedelta("365 days")` would be a prudent safeguard.

### 5.3) fyearq Attachment

- `attach_fyearq()` (from `panel_utils.py`) uses the same merge_asof pattern to attach Compustat fiscal year to each call.
- File_name uniqueness asserted before set_index (line 122--128).
- Match rate validated >= 80% or raises ValueError (lines 170--178).
- Calls with unparseable `start_date` receive NaN fyearq.

### 5.4) Lead Variable Construction

The `create_lead_variable()` function (panel builder lines 254--396) constructs `InvestmentResidual_lead`:

1. For each (gvkey, fyearq_int), select the call with the latest `start_date` (proxy for end-of-fiscal-year).
2. Extract that call's `InvestmentResidual` as the end-of-year value.
3. Sort by (gvkey, fyearq) and shift(-1) within gvkey to get next-fiscal-year's residual.
4. **Gap validation:** Only accept lead if next row's fyearq == current fyearq + 1. Non-consecutive fiscal years receive NaN lead.
5. Merge lead back to all calls by (gvkey, fyearq_int) -- all calls in the same firm-year get the same lead value.
6. Zero-row-delta guard on lead merge (lines 381--385).

### 5.5) Stage 4 Sample Filtering

The runner applies sequential filters in `prepare_regression_data()` (lines 196--227):

1. `filter_main_sample()`: Drop FF12 in {8, 11} (Finance, Utility).
2. Replace inf with NaN in all columns.
3. Drop rows where DV is NaN.
4. Complete-case analysis: require all regression variables non-missing.
5. Drop firms with fewer than 5 calls (`MIN_CALLS_PER_FIRM = 5`).

## 6. Variable Dictionary

### 6.1) Dependent Variables

#### InvestmentResidual (Biddle et al. 2009)

**Computed in:** `_compute_biddle_residual()`, `_compustat_engine.py` lines 472--770.

**Full construction chain (7 steps):**

**Step 1 -- Build the annual Q4-only panel (lines 527--555):**
The Biddle residual operates on an annual panel. From the full quarterly Compustat data, only rows with `fqtr == 4` are retained (the Q4 row holds the full fiscal-year YTD cumulative for flow variables). The annual panel is deduplicated on `(gvkey, fyearq)` keeping the last row (most recent restatement), then sorted by `(gvkey, fyearq)`.

**Step 2 -- Investment numerator (lines 577--608):**
```
inv_num = capxy.fillna(0) + xrdy.fillna(0) + aqcy.fillna(0) - sppey.fillna(0)
```
Where:
- `capxy` = capital expenditures (YTD cumulative, Q4 value = full fiscal year). Although `fillna(0)` is applied in the sum, a post-hoc guard sets `inv_num = NaN` whenever `capxy` is itself NaN (line 608: `inv_num = inv_num.where(annual["capxy"].notna(), np.nan)`). So capxy is effectively **required non-null**; NaN capxy produces NaN Investment.
- `xrdy` = R&D expense (annual YTD). `fillna(0)` per Biddle replication convention: most non-tech firms do not separately report R&D, so missing is treated as zero.
- `aqcy` = acquisitions (annual YTD). `fillna(0)` per convention: many firms have no acquisitions. The missing rate is logged to stdout for audit (lines 591--598).
- `sppey` = sale of property, plant, and equipment (annual YTD). `fillna(0)`: many firms have no asset sales.

**Step 3 -- Lagged total assets and Investment ratio (lines 610--623):**
```
at_lag = groupby("gvkey")["atq"].shift(1)   # prior fiscal year's total assets
```
Gap-year validation (line 614--615): if `fyearq - fyearq_lag != 1` (non-consecutive fiscal years), `at_lag` is set to NaN. This prevents stale denominators from contaminating the ratio.
```
Investment = inv_num / at_lag    (where at_lag > 0 and at_lag is not NaN; else NaN)
```
`Investment` is then cleaned: `inf`/`-inf` replaced with NaN (line 622), then **winsorized at 1%/99% by fiscal year** via `_winsorize_by_year(annual["Investment"], annual["fyearq"])` (line 623). The `_winsorize_by_year` function (lines 445--469) clips within each `fyearq` group at the 1st and 99th percentiles, requiring a minimum of 10 valid observations per year-group (m-2 fix); year-groups with fewer than 10 valid obs are left unclipped.

**Step 4 -- SalesGrowth construction (lines 650--667):**
```
sale_annual = saley.fillna(saleq)     # M-1 fix: prefer saley (annual total revenue);
                                      # fallback to saleq (quarterly revenue) when saley is NaN
sale_lag    = groupby("gvkey")["sale_annual"].shift(1)
```
Gap-year validation: `sale_lag` is set to NaN for non-consecutive fiscal years (same `gap_mask` as Step 3).
```
SalesGrowth = (sale_annual - sale_lag) / |sale_lag|    (where sale_lag is not NaN and |sale_lag| > 0; else NaN)
```
`inf`/`-inf` replaced with NaN (line 664). **Winsorized exactly once** at 1%/99% by fiscal year (line 667), before use as a first-stage predictor (C-6 fix: a second post-OLS winsorization was removed). The `skip_winsorize` set in `_compute_and_winsorize()` (line 1203) prevents re-winsorization when the full Compustat panel is processed later.

**Step 4a -- SalesGrowth_lag construction (lines 673--678):**
```
SalesGrowth_lag = groupby("gvkey")["SalesGrowth"].shift(1)
```
Two-level gap validation: (a) the base `gap_mask` (fyearq - fyearq_lag != 1) is applied, and (b) an additional check validates that `fyearq_lag - fyearq_lag2 == 1` (lines 676--678: `sg_gap_mask`). Both gaps set `SalesGrowth_lag = NaN`. This ensures the lagged predictor is from the true prior fiscal year.

**Step 4b -- TobinQ_lag (dead code, lines 627--672):**
`TobinQ` is computed on the annual panel as `(cshoq * prccq + debt_book) / atq` where `debt_book = dlcq.clip(0).fillna(0) + dlttq.clip(0).fillna(0)` (NaN when both `dlcq` and `dlttq` are missing). All three components (mktcap, debt_book, atq) are required non-null (C-4 fix). `TobinQ` is winsorized at 1%/99% by fiscal year (line 648). `TobinQ_lag = groupby("gvkey")["TobinQ"].shift(1)` with gap validation (lines 671--672). **However, TobinQ_lag is never used** in the first-stage OLS: `reg_cols = ["Investment", "SalesGrowth_lag"]` (line 707). This is dead code -- a remnant of a considered extended specification. It matches the Biddle et al. (2009) Equation 1 baseline which uses only SalesGrowth. See Section 12.

**Step 5 -- First-stage OLS by (FF48, fiscal_year) cell (lines 696--731):**

FF48 industry codes are derived from SIC via `_load_ff48_map()` (lines 361--442):
1. Load `inputs/FF1248/Siccodes48.zip`; find the `.txt` file by extension (M-4 fix).
2. Parse header lines (digit-leading, non-indented) as industry code definitions; parse indented range lines (e.g., `2000-2099`) as SIC-to-FF48 mappings.
3. Build a complete `Dict[int, int]` mapping every SIC 0--9999 to an FF48 code.
4. SIC codes not in any explicitly defined range are assigned to FF48 code 48 ("Almost Nothing"), the Fama-French residual category (M-3 fix: hardcoded `CATCHALL_FF48 = 48` at line 432, not derived from max code).

The first-stage OLS loop:
```python
for (ff48, fyear), grp in annual.groupby(["ff48_code", "fyearq"], dropna=False):
    # M-6 fix: dropna=False makes None-keyed cells visible; skip them explicitly
    if ff48 is None or pd.isna(ff48):
        cells_skipped_none += 1; continue
    valid_idx = grp.dropna(subset=["Investment", "SalesGrowth_lag"]).index
    if len(valid_idx) < 20:
        cells_skipped_size += 1; continue
    Y = annual.loc[valid_idx, "Investment"]
    X = sm.add_constant(annual.loc[valid_idx, ["SalesGrowth_lag"]])
    model = sm.OLS(Y, X).fit()
    annual.loc[valid_idx, "InvestmentResidual"] = Y - model.predict(X)
```
Key details:
- Minimum cell size: **20 valid observations** (line 715). Cells with fewer are skipped.
- OLS via `statsmodels.api.OLS` with `add_constant` (intercept included).
- Residual = actual Investment minus predicted Investment. Positive = overinvestment; negative = underinvestment.
- OLS errors are logged with `(ff48, fyear)` cell identity and exception type (m-3 fix; lines 725--731).
- Firm-years in skipped cells (too small or missing SIC) receive NaN `InvestmentResidual`.

**Step 6 -- Post-OLS winsorization (line 743):**
```python
annual["InvestmentResidual"] = _winsorize_by_year(annual["InvestmentResidual"], annual["fyearq"])
```
Winsorized at **1%/99% by fiscal year** via `_winsorize_by_year()`. This is the only winsorization applied to `InvestmentResidual`. The `skip_winsorize` set in `_compute_and_winsorize()` (line 1199) explicitly includes `"InvestmentResidual"`, preventing any re-winsorization during the global Compustat variable processing pass.

**Step 7 -- Join back to full quarterly panel (lines 753--770):**
The annual-level `InvestmentResidual` (one value per gvkey-fyearq) is joined back to the full quarterly Compustat panel via `merge(on=["gvkey", "fyearq"], how="left", validate="m:1")`. This means all quarterly rows for the same firm-year receive the identical `InvestmentResidual` value. The merge uses float-typed `fyearq` on both sides to avoid dtype mismatches. The resulting Series is aligned to the original `comp.index`.

**Winsorization summary for InvestmentResidual:**
| Stage | What | When | Code location |
|-------|------|------|---------------|
| Pre-OLS | `Investment` ratio winsorized 1%/99% by fyearq | After ratio computation, before first-stage OLS | Line 623 |
| Pre-OLS | `SalesGrowth` winsorized 1%/99% by fyearq | Before lag and use as predictor | Line 667 |
| Post-OLS | `InvestmentResidual` winsorized 1%/99% by fyearq | After OLS residual extraction | Line 743 |
| Skipped | No re-winsorization in `_compute_and_winsorize()` | Prevented by `skip_winsorize` set | Line 1199 |

#### InvestmentResidual_lead

**Computed in:** `create_lead_variable()`, `build_h2_investment_panel.py` lines 254--396.

The end-of-fiscal-year InvestmentResidual from fiscal year t+1, assigned to all calls in fiscal year t. Full construction:

1. `attach_fyearq()` merges Compustat `fyearq` onto the call-level panel via `merge_asof` (backward direction, matching call `start_date` to Compustat `datadate` by `gvkey`).
2. For each `(gvkey, fyearq_int)` group, select the call with the latest `start_date` (proxy for end-of-fiscal-year; line 319: `idxmax`). Extract that call's `InvestmentResidual` as `InvestmentResidual_eoy`.
3. Sort by `(gvkey, fyearq_grp)` and `shift(-1)` within `gvkey` to obtain the next fiscal year's end-of-year residual (lines 336--341).
4. Consecutive-year validation (line 344): `InvestmentResidual_lead` is set to NaN unless `fyearq_lead == fyearq_grp + 1`. Non-consecutive fiscal years (gaps) receive NaN.
5. Merge lead values back to all calls on `(gvkey, fyearq_int)` with `how="left"` (line 379). Zero-row-delta guard enforced (lines 381--385).
6. Calls in the last fiscal year per firm, fiscal-year gaps, or calls with missing `fyearq` receive NaN lead.

**Not winsorized separately:** `InvestmentResidual_lead` inherits its winsorization from the underlying `InvestmentResidual` (which was winsorized at Step 6 above). No additional winsorization is applied at the lead-variable stage.

### 6.2) Key Independent Variables (Textual -- NLP Pipeline)

All four key IVs follow the same construction chain. They differ only in the speaker filter and transcript section filter.

**Full NLP construction chain:**

1. **Stage 1 -- Transcript parsing and speaker segmentation:** Raw earnings-call transcripts are parsed to identify speaker turns. Each turn is tagged with a speaker role (CEO, CFO, other manager, analyst) and a transcript section (Presentation, Q&A). Speaker identification uses title/role matching from the transcript header.

2. **Stage 2 -- Tokenization and dictionary lookup:** Each speaker turn is tokenized using the Loughran-McDonald (LM) master dictionary approach with Python `CountVectorizer` tokenization (regex `r"(?u)\b[a-zA-Z]+\b"`, extracting alphabetic tokens only). Each token is looked up in the LM sentiment/uncertainty dictionary. The LM dictionary classifies words into categories including `Uncertainty` (297 words such as "approximate", "contingency", "depend", "fluctuate", "indefinite", "predict", "risk", "uncertain", "volatile", etc.), `Negative` (2,355 words), `Positive` (354 words), and others.

3. **Stage 2 -- Percentage computation:** For each `(file_name, speaker_role, section)` combination:
   ```
   {Speaker}_{Section}_Uncertainty_pct = (count of uncertainty words) / (total word count) * 100
   ```
   The denominator is the total number of tokens produced by that speaker in that section. Output is written to yearly parquet files: `linguistic_variables_{year}.parquet`.

4. **Stage 3 -- Loading via LinguisticEngine:** The `LinguisticEngine` singleton (`_linguistic_engine.py`) loads all yearly parquet files, concatenates them, and applies **per-year upper-only winsorization at 0%/99%** (lines 255--258):
   ```python
   winsorize_by_year(combined, existing_pct_cols, year_col="year",
                     lower=0.0, upper=0.99, min_obs=10)
   ```
   The `lower=0.0` means no lower clipping (percentages are naturally bounded at 0). The `upper=0.99` clips at the 99th percentile within each calendar year. Years with fewer than 10 valid observations for a given column are left unclipped. The winsorized data is cached for the process lifetime.

5. **Stage 3 -- Builder extraction:** Each individual builder (e.g., `CEOQAUncertaintyBuilder`) queries the `LinguisticEngine` singleton and extracts `["file_name", column_name]`. No further transformation is applied.

6. **Stage 3 -- Panel merge:** The extracted column is merged onto the manifest by `file_name` (left join, zero-row-delta enforced).

## 11. Identification & Inference Assessment

### 11.1) Causal Identification

H2 estimates a **reduced-form association** between speech uncertainty and investment efficiency. It does not establish a causal effect. The regression specification is:

```
InvestmentResidual_it = beta * Uncertainty_it + gamma * Controls_it + FE + epsilon_it
```

**Key threats to causal interpretation:**

1. **Reverse causality:** Managers who know about poor investment outcomes (e.g., a failing project) may speak with more uncertainty on earnings calls. The uncertainty language would then reflect information about investment, not cause investment inefficiency. The direction of influence is ambiguous without an exogenous shock or instrumental variable strategy.

2. **Omitted variable bias:** Firm distress or macroeconomic headwinds could simultaneously increase managerial speech uncertainty and reduce investment efficiency. Industry-year and firm FE absorb some of this, but time-varying firm-specific confounders remain.

3. **No instrumental variable strategy:** The suite uses OLS with fixed effects. No IV, difference-in-differences, regression discontinuity, or natural experiment approach is employed.

**Conclusion:** The H2 coefficients estimate a conditional correlation between speech uncertainty and investment efficiency, controlling for observable firm characteristics and fixed effects. Causal language (e.g., "higher uncertainty causes lower efficiency") is not supported by this design.

### 11.2) Unit of Observation Mismatch

InvestmentResidual is an annual firm-level variable. It is matched to all earnings calls within that firm-year. Within a given (gvkey, fyearq), all calls share the identical DV value, but IVs (uncertainty percentages) vary across Q1--Q4 calls.

**Implications:**
- The regression exploits within-firm-year variation in uncertainty against a constant DV. This is similar to using the annual average but with unequal weighting (each call gets one observation).
- Effective N is closer to the number of unique (gvkey, fyearq) pairs than the number of calls.
- Firm-clustered SEs account for within-firm correlation but do not fully resolve the Moulton problem at the firm-year level.
- A robustness check collapsing to one observation per firm-year (e.g., averaging IVs across calls) would strengthen interpretation.

### 11.3) Multiple Testing Exposure

The suite tests 4 IVs across 8 specifications = 32 one-tailed tests at alpha = 0.05. Under the null hypothesis of no effect, the expected number of false positives is 32 * 0.05 = 1.6. Finding 2/32 significant (Manager QA Uncertainty, industry FE specs) is barely above the null expectation.

No multiple-testing correction (Bonferroni, Benjamini-Hochberg, or similar) is applied. This should be disclosed when interpreting results.

### 11.4) One-Tailed Test Justification

The one-tailed test (beta < 0) requires a strong prior theoretical justification that the effect can only run in one direction. The Biddle et al. (2009) framework does not inherently predict directional effects of speech uncertainty -- uncertainty could lead to either over- or under-investment. The one-tailed test doubles statistical power in the predicted direction but makes it impossible to detect effects in the opposite direction.

### 11.5) Standard Error Adequacy

Firm-clustered SEs are appropriate for within-firm serial correlation. However:
- They do not account for cross-sectional dependence (e.g., industry-wide shocks affecting many firms in the same year). Two-way clustering (firm + year) would address this but is not implemented.
- The Moulton problem from the firm-year DV vs. call-level observation mismatch is only partially addressed by firm clustering.

## 12. Known Limitations & Unresolved Issues

### 12.1) CapexAt Shares capxy Component with InvestmentResidual (High)

**CapexAt** = capxy_Q4_annual / atq_annual_lag1 (`_compustat_engine.py` lines 1081--1087).

**InvestmentResidual** numerator = capxy + xrdy + aqcy - sppey, denominator = at_lag (`_compustat_engine.py` lines 601--619).

Both variables share the `capxy` component in their numerators and use variants of lagged total assets in their denominators. Including CapexAt as a second-stage control in a regression where the DV is partially constructed from capxy introduces partial mechanical correlation. The OLS residualization in the Biddle first stage attenuates but does not eliminate this overlap.

**Consequence:** The coefficient on CapexAt may be biased, and through collinearity, this could affect the uncertainty IV coefficient estimates.

**Mitigation:** Consider testing sensitivity to CapexAt exclusion as a robustness check.

### 12.2) H2b (Leverage Moderation) Documented But Never Tested (Medium)

The panel builder docstring (lines 32--33) states: "H2b: leverage attenuates the uncertainty-investment link (beta3 > 0)." The runner contains no interaction terms between BookLev and any uncertainty variable. No grep match for "interaction", "BookLev.*Uncertainty", or "H2b" in the runner.

**Status:** Either the docstring is stale from an earlier design iteration, or the interaction test was abandoned without documentation. Either way, the provenance is incomplete.

### 12.3) TobinQ_lag Dead Code in Biddle First-Stage (Medium)

## 14. Design Decisions & Rationale

### 14.1) CashFlow and SalesGrowth Excluded from Controls

CashFlow and SalesGrowth are first-stage inputs to the Biddle (2009) InvestmentResidual construction. Including them as second-stage controls would create mechanical correlation with the DV (the residual is defined relative to predicted investment based on SalesGrowth). Both variables are built by the CompustatEngine and available in the panel but intentionally excluded from `BASE_CONTROLS` and `EXTENDED_CONTROLS`.

### 14.2) Fiscal Year Time Index (fyearq_int)

The panel uses Compustat fiscal year (`fyearq_int`) rather than calendar year for:
- PanelOLS time fixed effects (absorb fiscal-year-specific shocks)
- Lead variable construction (shift by fiscal year, not calendar year)
- This correctly handles the ~30% of firms with non-December fiscal year-ends (C-5/M-9 audit fix)

### 14.3) CapexAt Retained as Control Despite Shared-Component Overlap

CapexAt shares the `capxy` numerator with the InvestmentResidual construction (see Section 12.1). It is retained as a control because:
- The residualization step in the first-stage OLS removes the systematic component of capxy that is predictable from SalesGrowth_lag.
- CapexAt captures the level of capital intensity, which is economically distinct from the residual's deviation from predicted investment.
- However, the partial mechanical overlap means the coefficient on CapexAt should be interpreted with caution, and sensitivity to its exclusion should be tested.

### 14.4) Extended Linguistic Controls

`Entire_All_Negative_pct` (negative sentiment) and `Analyst_QA_Uncertainty_pct` (analyst uncertainty) are included as extended controls to distinguish managerial uncertainty from broader call-level negativity or analyst-driven uncertainty. This addresses the concern that managerial uncertainty may proxy for general bad news rather than genuine forward-looking uncertainty.

### 14.5) Simultaneous IV Entry

All 4 uncertainty IVs enter every regression simultaneously. This design choice allows each IV's coefficient to be estimated conditional on the others, measuring the marginal contribution of each uncertainty dimension. The alternative (one IV per regression) would inflate the number of specifications and risk omitted-variable bias within the uncertainty construct.

### 14.6) Biddle (2009) First-Stage: SalesGrowth Only

The first-stage OLS uses only `SalesGrowth_lag` as a predictor (matching Biddle et al. 2009 Equation 1 baseline). An extended specification including TobinQ_lag was considered (TobinQ_lag is computed at `_compustat_engine.py` line 671) but was not used. The baseline specification is the standard choice in the investment efficiency literature.

### 14.7) One-Tailed Hypothesis Direction

The prior for beta < 0 follows from the precautionary-motive theory: managers facing greater uncertainty should invest more conservatively, leading to underinvestment (negative residual). The Biddle framework does not distinguish between over- and under-investment in the residual's sign, so a negative beta would indicate that uncertainty is associated with deviations from the predicted level of investment in the negative (underinvestment) direction. This directional prior should be explicitly justified in the thesis chapter with reference to the theoretical framework.
