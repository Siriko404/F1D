## A. Suite Overview

| Field | Value |
|---|---|
| **Suite name / ID** | H2 — Investment Efficiency |
| **Research question** | Do managers with higher speech uncertainty exhibit lower investment efficiency (H2a: β₁ < 0)? Does leverage attenuate this relationship (H2b: β₃ > 0)? |
| **Estimation unit** | Earnings call (one row per call); call-level OLS with firm + year fixed effects |
| **Dependent variable** | `InvestmentResidual_lead` — Biddle et al. (2009) abnormal investment in fiscal year t+1; >0 = overinvestment, <0 = underinvestment |
| **Key independent variables** | 6 uncertainty measures: `Manager_QA_Uncertainty_pct`, `CEO_QA_Uncertainty_pct`, `Manager_QA_Weak_Modal_pct`, `CEO_QA_Weak_Modal_pct`, `Manager_Pres_Uncertainty_pct`, `CEO_Pres_Uncertainty_pct` |
| **Interaction term** | `Uncertainty_x_Lev` — raw product (no mean-centering; firm FE absorbs within-group means) |
| **Controls** | `Lev`, `Size`, `TobinsQ`, `ROA`, `CashFlow`, `SalesGrowth` |
| **Fixed effects** | Firm FE (EntityEffects) + Year FE (TimeEffects) via PanelOLS within-transformation |
| **Standard errors** | Firm-clustered (`cov_type="clustered"`, `cluster_entity=True`) via `linearmodels.PanelOLS` |
| **Samples** | Main (FF12 codes 1–7, 9–10, 12), Finance (FF12=11), Utility (FF12=8) |
| **Minimum calls filter** | ≥ 5 calls per firm before regression |
| **One-tailed tests** | H2a: β₁ < 0 (p_one = p_two/2 if β₁ < 0); H2b: β₃ > 0 (p_one = p_two/2 if β₃ > 0) |

## G. Outliers, Missing Data, and Scaling Rules

### Winsorization

| Variable Group | Method | Threshold | Where Applied |
|----------------|--------|-----------|---------------|
| Linguistic `_pct` columns | Per-year upper-only | 99th percentile (no lower clipping; `lower=0.0`) | `LinguisticEngine.get_data()` |
| Compustat financial ratios | Per-fyearq (fiscal year) symmetric | 1%/99% | `CompustatEngine.get_data()` |
| Interaction term (`Uncertainty_x_Lev`) | None | — | Computed post-winsorization |

**Implementation:** `src/f1d/shared/variables/winsorization.py`

**Note on linguistic winsorization:** Linguistic percentage variables are winsorized using `lower=0.0, upper=0.99`, meaning only the upper tail is clipped at the 99th percentile. The lower tail is not clipped because many linguistic measures have legitimate zero or near-zero values (e.g., transcripts with no uncertainty words).

**Note on Compustat winsorization:** Compustat variables are winsorized by `fyearq` (Compustat fiscal year), not calendar year. This ensures consistent outlier treatment within fiscal years, which is important for firms with non-December fiscal year-ends.

## J. Known Issues / Ambiguities

### Issue 1: Fiscal Year vs Calendar Year (RESOLVED)

**Status:** Fixed via audit C-5/M-9

**Description:** Lead variable originally used calendar year (`start_date.dt.year`) for grouping calls into firm-years. This incorrectly handled ~30% of firms with non-December fiscal year-ends.

**Fix:** `build_h2_investment_panel.py` now uses `fyearq` (Compustat fiscal year) for grouping and consecutive-year gap validation. The `attach_fyearq()` function performs a backward `merge_asof` to match calls to their fiscal year.

**Code reference:** `panel_utils.py:76-192`, `build_h2_investment_panel.py:257-399`

### Issue 2: Within-R² Calculation (RESOLVED)

**Status:** Fixed via audit B8

**Description:** LaTeX table originally reported LSDV R² from PanelOLS, which includes variance explained by fixed effects. For interpretation, within-R² (net of FE) is more appropriate.

**Fix:** `run_h2_investment.py` now computes within-R² manually by demeaning y and ŷ within firm and year groups before computing R².

**Code reference:** `run_h2_investment.py:371-407`

### Issue 3: InvestmentResidual Interpretation

**Status:** DOCUMENTED

**Description:** The Biddle (2009) investment residual can be positive (overinvestment) or negative (underinvestment). H2a hypothesizes β₁ < 0, meaning higher uncertainty → lower |residual| (closer to efficient investment). This interpretation assumes |InvestmentResidual| measures inefficiency.

**Recommendation:** Consider clarifying in paper whether the hypothesis is about signed residual or absolute value. Current code uses signed residual.
