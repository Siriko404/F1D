## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H2 -- Speech Uncertainty and Investment Efficiency |
| Research question | Does managerial speech uncertainty (and CEO clarity residuals) in earnings calls predict lower investment efficiency, measured via the Biddle et al. (2009) investment residual? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `year` (time index, calendar year) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |
| Regularization | None |

### A3) Outcome / Target Definition

| Field | Value |
|-------|-------|
| Dependent variables (2) | `InvestmentResidual` (contemporaneous, t) and `InvestmentResidual_lead` (one-year ahead, t+1) |
| Construction | Biddle et al. (2009, JAE): `Investment = (capxy + xrdy + aqcy - sppey) / at_lag`; first-stage OLS by FF48-year cell: `Investment ~ SalesGrowth_lag` (min 20 obs); `InvestmentResidual = actual - predicted`. > 0 = overinvestment; < 0 = underinvestment |
| Timing convention | Contemporaneous DV uses fiscal year t investment matched to call date; lead DV uses fiscal year t+1. **VERIFIED IMPLEMENTATION ISSUE:** The contemporaneous DV suffers from look-ahead bias because Q4 annual investment data is joined to all quarters via gvkey+fyearq, meaning a Q1 call gets fiscal-year-end investment that has not yet occurred. |
| Lead construction | Last call per (gvkey, fyearq), shifted -1 within gvkey; gap check: only if `fyearq_lead == fyearq + 1` |

### A4) Main RHS and Controls

| Field | Value |
|-------|-------|
| Key independent variables (6) | **Uncertainty measures (4):** `Manager_QA_Uncertainty_pct`, `CEO_QA_Uncertainty_pct`, `Manager_Pres_Uncertainty_pct`, `CEO_Pres_Uncertainty_pct`; **Clarity residuals (2):** `CEO_Clarity_Residual`, `Manager_Clarity_Residual` |
| Interaction term | **NOT IMPLEMENTED.** The panel builder docstring (line 33-34) mentions H2b (leverage interaction: `Uncertainty_x_Lev`) but no interaction term appears in the regression formula (`run_h2_investment.py`). The formula is `{dv} ~ {iv} + Lev + controls + FE` with no `{iv}*Lev` term. H2b is therefore not testable from this code. |
| Controls | `Lev`, `Size`, `TobinsQ`, `ROA`, `CashFlow`, `SalesGrowth` (defined at `run_h2_investment.py` lines 113-120; `Lev` enters the formula directly as a main effect, separate from the control block) |
| Sample filters | `assign_industry_sample()` maps FF12 codes: 11 -> Finance, 8 -> Utility, all others (including NaN) -> Main. **Note:** NaN `ff12_code` values are classified as "Main" by the `default="Main"` argument in `np.select()` (`panel_utils.py` line 70). The count of affected firms is UNVERIFIED. |
| Minimum calls filter | >= 5 calls per firm before regression (`MIN_CALLS_PER_FIRM = 5`, line 123) |

### A5) Fixed Effects and Variance Estimator

| Field | Value |
|-------|-------|
| Fixed effects | Firm FE (EntityEffects) + Year FE (TimeEffects) via PanelOLS within-transformation |
| Standard errors | Firm-clustered (`cov_type="clustered"`, `cluster_entity=True`) via `linearmodels.PanelOLS` (line 367) |
| Year FE concept | **VERIFIED IMPLEMENTATION ISSUE:** Time FE is calendar year (`year = start_date.dt.year`, `build_h2_investment_panel.py` line 254), not fiscal year. The DV is constructed on a fiscal-year basis. This creates a mismatch for ~30% of firms with non-December fiscal year-ends. Firm FE absorb most firm-level fiscal-year structure, but year FE absorb less time variation for non-December FYE firms. |
| Clustering adequacy | Entity-only clustering is defensible but two-way clustering (firm + year) is standard in corporate finance. **Note:** Utility sample has only 71-84 clusters across specs (verified from `model_diagnostics.csv`), which is borderline for robust clustered inference. Cameron, Gelbach & Miller (2008) suggest concern below ~50 clusters. |

### A6) One-Tailed Tests

| Field | Value |
|-------|-------|
| Hypothesis direction | H2: beta1 < 0 (higher uncertainty/clarity_residual -> lower investment efficiency) |
| Implementation | `p_one = p_two / 2 if beta1 < 0 else 1 - p_two / 2` (line 389) |

| Variable | Formula | Source Columns | Winsorization |
|----------|---------|----------------|---------------|
| `InvestmentResidual` | Biddle (2009) residual from FF48-year OLS: `Investment ~ SalesGrowth_lag` where `Investment = (capxy+xrdy+aqcy-sppey) / atq_{t-1}` | `capxy`, `xrdy`, `aqcy`, `sppey`, `atq` (lagged), `saley` (with `saleq` fallback) | Per-fyearq 1%/99% on ratio inputs |

**Note on Biddle first-stage:** The first-stage regression uses only `SalesGrowth_lag` as the predictor (`_compustat_engine.py` line 698: `reg_cols = ["Investment", "SalesGrowth_lag"]`), despite computing `TobinQ_lag` at line 662. The original Biddle et al. (2009) specification includes both `SalesGrowth_lag` and `TobinQ_lag`. However, a SalesGrowth-only first-stage is common in replications (e.g., Gomariz & Ballesta 2014, Lara et al. 2016). This is a deviation from the canonical specification but within replication norms.

## J. Known Issues / Ambiguities

### J1: Calendar Year vs Fiscal Year FE Mismatch — VERIFIED IMPLEMENTATION ISSUE

**Severity:** Medium-High

The panel is indexed by `year` (calendar year from `start_date.dt.year`), so PanelOLS time FE absorb calendar-year averages. But the DV (`InvestmentResidual`) is constructed on a Compustat fiscal-year basis. For ~30% of firms with non-December fiscal year-ends, the calendar-year FE do not align with the DV's timing. Firm FE absorb most firm-level fiscal-year structure, but year FE absorb less time variation for non-December FYE firms.

**Code reference:** `build_h2_investment_panel.py` line 254 (`year = start_date.dt.year`); `run_h2_investment.py` line 363 (`set_index(["gvkey", "year"])`)

### J2: H2b Interaction Not Implemented — VERIFIED DOCUMENTATION GAP

**Severity:** Low-Medium

The panel builder docstring (line 33-34) and original provenance doc describe H2b (leverage interaction: `Uncertainty_x_Lev`, beta3 > 0). No interaction term exists anywhere in `run_h2_investment.py`. The formula is `{dv} ~ {iv} + Lev + controls + FE` with Lev as a control only. H2b is not testable from the current code.

**Code reference:** `build_h2_investment_panel.py` lines 33-34 (docstring mentions H2b); `run_h2_investment.py` (no interaction term in formula construction at lines 347-351)

### J3: No Linguistic Controls in Specifications — VERIFIED IMPLEMENTATION ISSUE

**Severity:** High

The regression formula includes only financial controls (`Lev`, `Size`, `TobinsQ`, `ROA`, `CashFlow`, `SalesGrowth`). No linguistic controls (e.g., tone, readability, length) are included. This is a serious omission for a linguistic-effects hypothesis — any observed relationship between uncertainty language and investment efficiency could be driven by correlated linguistic features.

**Code reference:** `CONTROL_VARS` at `run_h2_investment.py` lines 113-120

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-Line Referee Verdict

The H2 suite tests whether managerial speech uncertainty predicts investment efficiency, producing overwhelmingly null results: 2 of 28 regressions are significant at the 5% one-tailed level, which is statistically indistinguishable from chance (under the null hypothesis of no effect, the probability of observing >= 2 significant results out of 28 at alpha=0.05 is approximately 0.72). The contemporaneous DV (`InvestmentResidual`) has a genuine look-ahead timing issue (Q1 calls matched to fiscal-year-end investment that has not yet occurred). No robustness tests exist. No linguistic controls are included. The call-level observation unit with firm-year-level DV creates ~4x pseudo-replication (mean ~3.9 calls per firm-year sharing identical DV values), inflating all reported N and rendering all standard errors and significance levels unreliable. H2b (leverage interaction) is described in documentation but not implemented in code.

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

The implementation currently supports **descriptive association only** (and barely, given the null results). Causal language is not justified. The write-up would be overselling the evidence if it claimed support for H2. The committee should treat H2 as an informative null result.

### K2) Identification and Research Design Threats

## N. Final Readiness Statement

**Can this suite be reproduced end-to-end from code alone?**
Partially. The pipeline is deterministic and scripts are correct, but output file completeness is not guaranteed (12 of 28 .txt files missing in latest run). The `get_latest_output_dir()` pattern with 12 stale directories adds fragility.

**Can a committee/referee understand exactly how the variables are built?**
Yes. The variable dictionary is complete, with explicit formulas, source fields, winsorization rules, and code references for all 17 variables.

**Are the implemented results trustworthy for descriptive inference?**
Only marginally. The pseudo-replication issue (~4x N inflation from call-level analysis with firm-year DV) means all standard errors are understated. The 2/28 significance rate is consistent with no true effect (P(>=2) = 0.72 under null). After correcting for pseudo-replication and multiple testing, there would likely be zero significant results.

**Are they trustworthy for causal inference?**
No. Look-ahead bias in the contemporaneous DV, no robustness tests, no linguistic controls, entity-only clustering, and no identification strategy preclude causal interpretation.

**What is the single biggest threat to credibility?**
Pseudo-replication. With ~3.9 calls per firm-year sharing an identical DV, the reported N is approximately 4x the effective independent sample. All standard errors, t-statistics, and p-values are unreliable. This should be addressed before any results are reported.

**What is the single highest-value next fix?**
Collapse to firm-year level (or weight by 1/n_calls). This would produce honest N, honest SEs, and reveal whether any results survive — which, given the near-null findings at inflated N, is unlikely.
