## A. Suite Overview

| Field | Value |
|---|---|
| **Suite name / ID** | H1 — Cash Holdings Hypothesis (Speech Uncertainty → Cash Hoarding) |
| **Research question** | Do managers who use more uncertainty language in earnings calls hoard more cash in the following year (H1a: β₁ > 0)? Does leverage attenuate this relationship (H1b: β₃ < 0)? |
| **Estimation unit** | Earnings call (one row per call); call-level Panel OLS |
| **Dependent variable** | `CashHoldings_lead` — firm's end-of-fiscal-year cash holdings in year t+1 |
| **Key independent variables** | `Uncertainty` (6 measures: Manager/CEO QA/Pres Uncertainty & Weak Modal) |
| **Interaction term** | `Uncertainty_x_Lev` = Uncertainty × Leverage (raw product, not pre-centered) |
| **Controls** | `Size`, `TobinsQ`, `ROA`, `CapexAt`, `DividendPayer`, `OCF_Volatility`, `Lev` (main effect) |
| **Fixed effects** | Firm (`EntityEffects` via C(gvkey)) + Year (`TimeEffects` via C(year)) |
| **Standard errors** | Firm-clustered (`cov_type="clustered"`, `cluster_entity=True`) via `linearmodels.PanelOLS` |
| **Sample splits** | Main (FF12 1-7, 9-10, 12), Finance (FF12 11), Utility (FF12 8) |
| **Minimum calls filter** | ≥ 5 calls per firm before regression |
| **One-tailed tests** | H1a: β(Uncertainty) > 0; H1b: β(Uncertainty × Lev) < 0 |

### Lead Variable Construction (B6 Fix)

The `CashHoldings_lead` variable is constructed in `create_lead_variable()`:

1. **Attach fyearq**: Use `merge_asof` to attach Compustat fiscal year (`fyearq`) to each call via `panel.start_date` → `compustat.datadate` (backward match within gvkey)
2. **Select end-of-year proxy**: For each (gvkey, fyearq), take `CashHoldings` from the call with the **latest start_date** in that fiscal year
3. **Shift -1 within gvkey**: Sort by (gvkey, fyearq), shift to get next-fiscal-year cash
4. **Validate fiscal year continuity**: Only keep lead if next row is exactly fyearq+1 (not fyearq+2 due to gaps); set NaN otherwise
5. **Merge back**: Join lead values onto all calls by (gvkey, fyearq_int)

## G. Outliers, Missing Data, and Scaling Rules

### Winsorization

| Variable Set | Method | Thresholds | When Applied | Where Defined |
|--------------|--------|------------|--------------|---------------|
| **Compustat ratios** (Size, Lev, ROA, CashHoldings, TobinsQ, CapexAt, OCF_Volatility) | Per-year | 1%/99% within each year | At engine load time | `_compustat_engine.py:1043-1057` |
| **Linguistic percentages** (all *_pct columns) | Per-year | Upper 99th percentile only (lower=0.0); min_obs=1 | At engine load time | `_linguistic_engine.py:255-258` |

**Implementation:** `winsorization.py:winsorize_by_year()`
- Converts inf → NaN after ratio computation, before winsorization
- Linguistic vars: asymmetric (lower=0.0, upper=0.99) — no lower-tail clipping
- Compustat vars: symmetric (lower=0.01, upper=0.99) with min_obs=10 per year
- Returns copy (does not mutate)

### Missing Data Policy

1. **Panel construction:** All merges are left joins; missing values become NaN
2. **Regression preparation:** Complete cases only — rows with any NaN in required variables are dropped
3. **No imputation:** Missing values are never imputed

### Division-by-Zero Protection

| Variable | Protection | Where |
|----------|------------|-------|
| All ratios | inf → NaN after division | `_compustat_engine.py` |
| Size (log) | NaN if atq ≤ 0 | `_compustat_engine.py` (MAJOR-6 fix) |

### No Additional Scaling

- Variables enter regression in original units (after winsorization)
- No standardization, normalization, or logging beyond what's noted in formulas
- Firm FE absorb within-firm means (makes pre-centering redundant)

## J. Known Issues / Ambiguities

### 1. CEO Coverage Gap (68.0% vs 95.8%)

**Issue:** CEO uncertainty measures have only 68.0% coverage vs 95.8% for Manager measures.

**Cause:** CEO identification requires matching speaker roles to CEO titles; many calls lack explicit CEO speaker labels.

**Impact:** Half of H1 regressions (CEO measures) have ~28% fewer observations.

**Recommendation:** Accept as data limitation; document in paper footnote.

---

### 2. CurrentRatio Exclusion

**Issue:** `CurrentRatio` (actq/lctq) is NOT included in H1 controls, unlike H0.3.

**Reason:** `lctq` (current liabilities) is structurally missing (~80%) for financial firms (FF12=11), which would silently drop ~12,800 Finance observations. CurrentRatio is not in the standard Opler et al. (1999) / Bates et al. (2009) cash holdings specification.

**Impact:** Finance sample maintains ~17K observations; aligned with other hypothesis tests.

**Recommendation:** Keep exclusion; document in paper.

---

### 3. CashHoldings (Contemporaneous) Exclusion

**Issue:** `CashHoldings` (t) is not included as a control, only `CashHoldings_lead` (t+1) as DV.

**Reason:** "GAP-5" design decision — v2 specification did not include contemporaneous cash. Including it would absorb much of the variation we're trying to explain.

**Recommendation:** Keep exclusion; matches prior literature.

---

### 4. Moulton Correction (MAJOR-1 Fix)

**Issue:** HC1 standard errors were initially used, but `CashHoldings_lead` is constant within firm-year clusters (all calls in the same fiscal year share the same DV).

**Fix:** Changed to firm-clustered SEs (`cluster_entity=True`) to correct within-firm correlation.

**Verification:** All Stage 4 regressions now use `cov_type="clustered", cluster_entity=True`.

---

### 5. Fiscal Year Alignment (B6 Fix)

**Issue:** Initial implementation used calendar year (`start_date.dt.year`) to group calls for lead construction, but ~30% of firms have non-December fiscal year-ends.

**Fix:** Use Compustat `fyearq` (fiscal year) instead of calendar year. Attached via `merge_asof` to match call date to fiscal year.

**Verification:** `attach_fyearq()` in `panel_utils.py` handles this for all H-suite panels.

---

### 6. Pre-centering Interaction Terms (MAJOR-2 Fix)

**Issue:** Earlier versions mean-centered Uncertainty and Lev before creating the interaction term.

**Problem:** When firm FE are included, pre-centering is redundant (FE already demean within firms) and creates biased interaction interpretation.

**Fix:** Interaction term is now `Uncertainty * Lev` (raw product). No pre-centering.

**Verification:** `prepare_regression_data()` in `run_h1_cash_holdings.py` line 304.
