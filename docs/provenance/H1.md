## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H1 -- Speech Uncertainty and Future Cash Holdings |
| Research question | Does managerial speech uncertainty on earnings calls predict higher future corporate cash holdings (precautionary savings)? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `year` (time index, calendar year from `start_date`) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |
| Regularization | None |

### A3) Outcome / Target Definition

| Field | Value |
|-------|-------|
| Dependent variable | `CashHoldings_lead` -- next-fiscal-year cash holdings ratio |
| Construction | `cheq / atq` from Compustat, aggregated to firm-fiscal-year (last call per `(gvkey, fyearq_int)`), then shifted by -1 within `gvkey` to get t+1 value. Validated for fiscal-year continuity: non-consecutive fiscal years receive NaN. |
| Timing convention | Forward-looking: t+1 fiscal year. Calls in year t predict cash holdings at end of fiscal year t+1. |
| Missing policy | Drop (listwise deletion at Stage 4) |
| Boundary values | Winsorized at 1%/99% by fiscal year at Compustat engine level. Range in panel: [0.0000, 0.9938]. |

### A4) Main RHS and Controls

**Key independent variables (one per regression, 6 measures):**

| Measure | Variable | Description |
|---------|----------|-------------|
| 1 | `Manager_QA_Uncertainty_pct` | Manager (all) uncertainty language in Q&A, % of words |
| 2 | `CEO_QA_Uncertainty_pct` | CEO uncertainty language in Q&A, % of words |
| 3 | `Manager_QA_Weak_Modal_pct` | Manager weak modal language in Q&A, % of words |
| 4 | `CEO_QA_Weak_Modal_pct` | CEO weak modal language in Q&A, % of words |
| 5 | `Manager_Pres_Uncertainty_pct` | Manager uncertainty language in presentation, % of words |
| 6 | `CEO_Pres_Uncertainty_pct` | CEO uncertainty language in presentation, % of words |

**Control variables (all specs):**

| Variable | Source |
|----------|--------|
| `Lev` | (dlcq + dlttq) / atq from Compustat |
| `Size` | log(atq) from Compustat |
| `TobinsQ` | (atq + cshoq*prccq - ceqq) / atq from Compustat |
| `ROA` | iby_annual / avg_assets from Compustat |
| `CapexAt` | capxy_Q4_annual / atq_lag from Compustat |
| `DividendPayer` | 1(dvy_annual > 0) from Compustat |
| `OCF_Volatility` | Rolling 5-year std of (oancfy / atq_lag) from Compustat |

**Note:** `CurrentRatio` is built in the panel (actq/lctq) but deliberately excluded from Stage 4 controls because `lctq` is structurally missing (~80%) for financial firms (FF12=11), which would silently eliminate ~12,800 Finance-sample observations. Not in the standard Opler/Bates cash holdings specification.

**Note (unused builders):** The panel builder (`build_h1_cash_holdings_panel.py`) also loads `AnalystQAUncertaintyBuilder`, `NegativeSentimentBuilder`, `ManagerPresWeakModalBuilder`, `CEOPresWeakModalBuilder`, and `CurrentRatioBuilder`, which produce variables not used in the H1 regression control set or treatment measures. These unused builders increase panel build time and memory but do not affect estimation results. They may exist because the panel is shared across suites or was designed for future expansion.

**Industry Samples (3):**
- **Main:** FF12 codes 1-7, 9-10, 12 (non-financial, non-utility)
- **Finance:** FF12 code 11
- **Utility:** FF12 code 8

**Minimum calls filter:** Firms must have >= 5 calls after complete-case filtering. **Important:** This filter is applied AFTER listwise deletion on the measure-specific estimation sample (see `prepare_regression_data()` lines 284-287), meaning the actual criterion is ">= 5 complete-case calls for this specific measure," which can differ across measures. The thesis should document this as "firms with >= 5 complete-case calls in the estimation sample" rather than "firms with >= 5 calls."

**Total regressions:** 6 measures x 3 samples = 18 regressions.

### A5) Fixed Effects + Variance Estimator

| Field | Value |
|-------|-------|
| Entity FE | `gvkey` (firm) |
| Time FE | `year` (calendar year from `start_date`) |
| Standard errors | Firm-clustered (`cov_type="clustered"`, `cluster_entity=True`) |
| Justification | DV (`CashHoldings_lead`) is constant within (gvkey, fyearq) by construction -- all calls in the same firm-fiscal-year share the same lead DV. Firm-clustered SEs correct for this intra-cluster correlation (Moulton problem). |
| Small-sample correction | Default linearmodels cluster-robust correction |
| Hypothesis test | One-tailed: H1 beta > 0 (higher uncertainty -> more cash hoarding) |
| Cluster count note | Utility sample has only 70-83 clusters (firms) depending on spec. This is borderline for cluster asymptotics (Cameron, Gelbach & Miller 2008 suggest 50+ as minimum). Utility inference should be treated with additional caution. |

### A6) Model-Family Addendum

**Panel / FE specific:**
- Singleton dropping: `drop_absorbed=True` in PanelOLS handles singletons.
- Within-group variation: Not explicitly checked. Firms with only 1 year of data after FE absorption are implicitly dropped.
- Note: `drop_absorbed=True` may drop time periods (not just entities) if they are collinear, which could affect the year FE structure. The count of dropped observations is not logged.

**All other model families: N/A.**

### Missing Data Policy

- **Listwise deletion** at Stage 4: `prepare_regression_data()` drops all rows where ANY required variable is NaN.
- **inf -> NaN** replacement: Applied to all ratio variables in `_compute_and_winsorize()` (MINOR-9 fix) and in `prepare_regression_data()`.
- **Missing debt components:** `dlcq.fillna(0)`, `dlttq.fillna(0)` -- missing debt treated as zero per spec.
- **Missing `fyearq`:** Calls with NaN `fyearq` get NaN `CashHoldings_lead` (excluded from regressions by listwise deletion).

### Standardization

**No standardization is implemented.** Despite the LaTeX table note at `h1_cash_holdings_table.tex` line 38 claiming "All continuous controls are standardized within each model's estimation sample," the runner code (`run_h1_cash_holdings.py`) contains zero standardization logic. A grep for `standardiz`, `zscore`, `scale`, and `normalize` returns only the string literal at line 582 that generates the table note itself. **This is a false methodological claim in the output artifact.** See L16/J6.

### Denominator Protections

| Variable | Protection | Code reference |
|----------|-----------|----------------|
| `CashHoldings` | `cheq / atq` -- no explicit `atq > 0` guard in formula; relies on downstream inf->NaN replacement | `_compustat_engine.py` line ~1049 |
| `Lev` | `(dlcq.fillna(0) + dlttq.fillna(0)) / atq` -- no explicit `atq > 0` guard | `_compustat_engine.py` line ~1032 |
| `Size` | `np.where(atq > 0, np.log(atq), np.nan)` -- explicit guard (MAJOR-6 fix) | `_compustat_engine.py` line ~1027 |
| `TobinsQ` | `np.where(atq.notna() & (atq > 0) & mktcap.notna(), ...)` -- explicit guard | `_compustat_engine.py` lines ~1056-1060 |
| `ROA` | `np.where(avg_assets > 0, ..., np.nan)` -- explicit guard | `_compustat_engine.py` line ~1042 |
| `CapexAt` | `np.where(atq_annual_lag1 > 0, ..., np.nan)` -- explicit guard | `_compustat_engine.py` lines ~1063-1068 |
| `OCF_Volatility` | `np.where(atq_lag > 0, ..., np.nan)` -- explicit guard | `_compustat_engine.py` line ~323 |
| `CurrentRatio` | `actq / lctq.replace(0, np.nan)` -- zero denominator -> NaN | `_compustat_engine.py` line ~1045 |

**VERIFIED ISSUE:** `CashHoldings` and `Lev` lack explicit `atq > 0` guards in their formulas. For `CashHoldings = cheq / atq`, if `atq == 0`, the result is inf, which is caught by the subsequent inf->NaN replacement (MINOR-9 fix). However, for `atq < 0` (negative total assets -- rare but possible for distressed firms), the ratio produces a negative value that is NOT caught and enters the data. Given that negative `atq` is extremely rare in the Compustat-manifest intersection and these observations would likely be winsorized at the tails, the practical risk is low. Add guard as best practice but not material to results.

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line Referee Verdict

This suite implements a standard cash holdings prediction model (Opler et al. 1999 / Bates et al. 2009 style) with firm FE and year FE, testing whether managerial speech uncertainty predicts next-year cash holdings. The implementation is technically competent: the estimator is correctly specified, standard errors are appropriately firm-clustered (addressing the Moulton problem from the call-level observation structure), and the pipeline has extensive data quality safeguards. However, results are weak: only 2/6 Main-sample measures are significant at 5% (one-tailed), the within-R2 contribution of the treatment variable is near zero (~0.06), and presentation-section uncertainty measures are consistently insignificant or wrong-signed. The Q&A uncertainty result is concentrated in the Finance subsample, which raises concerns about generalizability. The implementation supports descriptive/predictive association at best; causal claims require much stronger identification. Additionally, the LaTeX table output contains a false claim about control standardization that must be corrected before thesis submission.

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

The implementation currently supports: **descriptive association only**.

The write-up would be overselling the evidence if it uses causal language (e.g., "uncertainty causes cash hoarding") or claims broad support across uncertainty measures. The result is fragile and measure-specific.

### K6) Interpretation and Claims Discipline

- **Causal verbs:** NOT justified. The implementation supports only descriptive association. The phrase "higher speech uncertainty -> more cash hoarding" in the docstring implies causality not supported by the research design.
- **Mechanism claims:** The precautionary motive interpretation (Opler et al. 1999) is a plausible mechanism but is not tested. The suite does not distinguish precautionary from other motives (e.g., agency, transaction cost).
- **Economic magnitudes:** A 1-unit increase in Manager_QA_Uncertainty_pct (roughly 3 standard deviations) is associated with a 0.44 percentage point increase in next-year cash holdings. Given mean cash holdings of 15.4%, this is approximately a 2.9% relative change -- small but not trivial. However, the p-value for the one-tailed test is 0.005, so the statistical significance is not matched by economic significance.
- **Fragile results:** 4 of 6 Main-sample specs are insignificant. Presentation-section measures are consistently insignificant across all samples. The result is concentrated in Q&A uncertainty, suggesting measurement specificity rather than a robust underlying relationship.
- **Non-results potentially hidden:** The one-tailed test with alpha=0.05 makes marginal results (M3, p=0.054) appear "just below significance." With two-tailed tests, only 2 of 18 regressions would be significant at 5%.
- **Table note accuracy:** The LaTeX table note contains TWO errors: (1) "All models use the Main industry sample" but the table actually shows all three samples (L12); (2) "All continuous controls are standardized within each model's estimation sample" but no standardization is performed (L16). Both must be fixed.
- **Controls are NOT standardized:** Despite what the table note claims, all variables enter the regression in their natural units. Coefficient interpretation is in raw units, which is standard but contradicts the output artifact.

## N. Final Readiness Statement

**Can this suite be reproduced end-to-end from code alone?**
PARTIAL. Stages 3 and 4 are fully scripted and deterministic, but require pre-existing Stage 1/2 outputs and non-committed raw data in `inputs/`. The `run_manifest.json` records the input panel hash (SHA-256), which helps verify that the correct panel was used. However, only 6 of 18 expected regression output `.txt` files are actually produced, reducing the completeness of the audit trail.

**Can a committee/referee understand exactly how the variables are built?**
YES. The variable construction chain is transparent: each variable has a dedicated builder with explicit formulas, and the Compustat engine contains well-documented computation logic with audit-fix annotations.

**Are the implemented results trustworthy for descriptive inference?**
YES, with caveats. The panel construction is sound, merges are guarded, and firm-clustered SEs are appropriate. However, the time FE misalignment (J1/L4), missing lagged DV control (L6), and false standardization claim in the LaTeX table (L16) must be fixed before any formal claims. The negative within-R2 values in Utility CEO specs should be discussed.

**Are they trustworthy for causal inference?**
NO. The suite exploits no exogenous variation. Firm FE + year FE absorb time-invariant confounders and common shocks, but time-varying omitted variables (macro uncertainty, industry conditions, firm-specific risk events) are not addressed. Reverse causality is not ruled out.

**What is the single biggest threat to credibility?**
The near-complete absence of robustness checks (L3), compounded by the false standardization claim in the LaTeX table (L16). The robustness gap means results cannot be defended; the false claim is an academic integrity risk that could undermine the entire suite's credibility if discovered by a referee.

**What is the single highest-value next fix?**
Fix the false standardization claim in the LaTeX table note (Priority 1 in Section M) -- this is a low-effort, high-impact correction that eliminates an immediate academic integrity risk. Then implement a standard robustness battery (Priority 2): alternative FE structures, double-clustering, sample window sensitivity, balanced panel, and firm-year collapse. These two fixes together represent the minimum required to defend the H1 findings in a thesis committee setting.
