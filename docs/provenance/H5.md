## 1. Suite Overview

| Field | Value |
|-------|-------|
| **Hypothesis** | H5: Speech uncertainty predicts higher post-call analyst forecast dispersion |
| **Research Question** | Does managerial speech uncertainty during earnings calls increase subsequent analyst forecast disagreement? |
| **Predicted Sign** | Positive (beta > 0): higher uncertainty words in speech lead to greater dispersion of analyst EPS forecasts |
| **Estimation Unit** | Individual earnings call (call-level) |

## 6. Variable Dictionary

### 6.1 Dependent Variable: `PostCallDispersion`

| Property | Value |
|----------|-------|
| **Formula** | SD(next-Q EPS forecasts) / \|Mean(next-Q EPS forecasts)\| x 100 |
| **Unit** | Percentage (coefficient of variation x 100) |
| **Timing** | +3 trading days after earnings call (`self.days_after = 3`, line 56) |
| **Trading day calendar** | `pandas.tseries.offsets.CustomBusinessDay(calendar=USFederalHolidayCalendar())` (line 61) |
| **Data source** | IBES Detail (individual analyst estimates), yearly files `inputs/tr_ibes/tr_ibes_{YYYY}.parquet` |
| **I/O-level filters** | MEASURE='EPS', FPI in ['6','7'], PDF in ['D'] (`_ibes_detail_engine.py` lines 187-191) |
| **CUSIP-to-GVKEY link** | CCM table (`inputs/CRSPCompustat_CCM/`), LINKPRIM in ['P','C'], `drop_duplicates(subset=["cusip8"], keep="first")` (lines 119-122) |
| **Winsorization** | 1%/99% pooled across all calls (`postcall_dispersion.py` lines 91-96): `clip(quantile(0.01), quantile(0.99))` |
| **Evidence** | `postcall_dispersion.py` lines 56, 61, 91-96, 241-276 |

**Complete construction chain** (verified in `postcall_dispersion.py`):

1. **Load manifest.** Read `file_name`, `gvkey`, `start_date` from master manifest; zero-pad gvkey to 6 digits; filter to requested year range (lines 72-78).
2. **Load IBES Detail.** `IbesDetailEngine.get_data()` reads yearly parquet files with PyArrow predicate pushdown: `MEASURE == "EPS" AND FPI IN ['6','7'] AND PDF IN ['D']`. Converts CUSIP to 8-character string, maps to gvkey via CCM. Converts ACTDATS and FPEDATS from string to datetime. Output columns: `gvkey, actdats, analys, value, fpedats` (`_ibes_detail_engine.py` lines 164-248).
3. **Compute post-call target date.** `date_after = start_date + 3 * CustomBusinessDay(calendar=USFederalHolidayCalendar())` (line 133). This skips weekends and US federal holidays.
4. **Match call to target FPEDATS.** For each call, `merge_asof` (direction="forward", tolerance=120 days) matches start_date to the nearest future FPEDATS in the IBES data. Executed per-gvkey to satisfy pandas sort requirements (lines 156-177).
5. **Join estimates.** Inner merge of matched calls with IBES estimates on `(gvkey, target_fpedats)`. This pools all FPI values (6 and 7) targeting the same FPEDATS (lines 193-198).
6. **Filter active estimates.** Keep only estimates where `actdats <= date_after` (line 241).
7. **Remove stale estimates.** Compute `_age_days = (date_after - actdats).dt.days`; drop rows where `_age_days > 180` (lines 244-245; threshold from `config.get("max_stale_days", 180)` at line 57).
8. **Deduplicate per analyst.** Sort by actdats, groupby `(file_name, analys)`, keep `.last()` (most recent estimate per analyst per call) (lines 251-256).
9. **Minimum analyst filter.** Require `count(analys) >= 2` per call via `transform("count")` (lines 259-260; `NUMEST_MIN = 2` at line 51).
10. **Compute dispersion.** `groupby("file_name")["value"].agg(["std", "mean"])`. Dispersion = `std / |mean| * 100`. Returns NaN when `|mean| == 0` or `std` is NaN (lines 266-276). Note: `std()` uses pandas default ddof=1 (sample standard deviation).
11. **Winsorize.** After computing both pre- and post-call dispersion, clip each at pooled 1st and 99th percentiles: `final[col] = final[col].clip(lo, hi)` where `lo = valid.quantile(0.01)`, `hi = valid.quantile(0.99)` (lines 91-96).

### 6.3 Key Independent Variables (4, all enter simultaneously)

All four IVs share an identical construction chain. They differ only in the speaker filter and transcript section filter applied in Stage 2.

**Common construction chain (Stage 2 pipeline):**

1. **Tokenization (Stage 2.1).** `src/f1d/text/tokenize_transcripts.py` loads the Loughran-McDonald Master Dictionary (`inputs/Loughran-McDonald_MasterDictionary_1993-2024.csv`). The dictionary is read via `pd.read_csv()`, words uppercased. Seven sentiment categories are extracted: Negative, Positive, Uncertainty, Litigious, Strong_Modal, Weak_Modal, Constraining. Words are mapped to categories using a vectorized melt operation (lines 772-812). A `CountVectorizer(vocabulary=vocab_list, token_pattern=r"(?u)\b[a-zA-Z]+\b", lowercase=False)` tokenizes each speaker turn's `speaker_text` (uppercased) and counts occurrences of each LM dictionary word (lines 927-932). Per-category counts (e.g., `Uncertainty_count`) and `total_tokens` (alpha-only word count via regex `r"(?u)\b[a-zA-Z]+\b"`) are stored per speaker turn (lines 949-959). Output: `linguistic_counts_{YYYY}.parquet` partitioned by manifest year.

2. **Variable construction (Stage 2.2).** `src/f1d/text/build_linguistic_variables.py` loads tokenized counts, flags speakers (Analyst, Manager, CEO, NonCEO_Manager, Operator, Entire) using manifest CEO names and a managerial-roles keyword list (lines 411-464). For each (speaker x section) combination, aggregates counts by `file_name` using ratio-of-sums: `{Category}_pct = (sum({Category}_count) / sum(total_tokens)) * 100` (lines 472-502). NaN is preserved when no text exists in that speaker-section combination (line 589). Output: `linguistic_variables_{YYYY}.parquet` with columns like `CEO_QA_Uncertainty_pct`.

3. **Loading and winsorization (Stage 3).** The `LinguisticEngine` singleton (`_linguistic_engine.py`) loads all `linguistic_variables_{YYYY}.parquet` files, concatenates them, and applies **per-year upper-only winsorization at 0%/99%** via `winsorize_by_year(combined, existing_pct_cols, year_col="year", lower=0.0, upper=0.99, min_obs=10)` (lines 255-258). This clips only the upper tail of percentage distributions, preserving zeros (legitimate "no uncertainty words in section") while trimming extreme right-tail outliers. The `winsorize_by_year` function (`winsorization.py`) replaces inf with NaN, then within each year group clips at the year-specific percentiles, skipping groups with fewer than 10 valid observations (lines 23-67).

4. **Builder extraction.** Each builder (e.g., `CEOQAUncertaintyBuilder`) calls `engine.get_data()` and selects its specific column from the cached, winsorized DataFrame (e.g., `ceo_qa_uncertainty.py` lines 30-43).

5. **Panel merge.** Left merge on `file_name` into the manifest. Zero-row-delta enforced (`build_h5_dispersion_panel.py` lines 154-159).

## 11. Identification & Inference Assessment

### 11.1 Identification Strategy

This suite tests whether managerial speech uncertainty predicts subsequent analyst forecast dispersion. The identification relies on within-firm (or within-industry) variation in speech uncertainty, controlling for pre-call dispersion levels and firm characteristics.

**Threats to identification:**

1. **Reverse causality.** Managers may adjust their speech when they anticipate analyst disagreement. If managers know analysts are dispersed and respond with more uncertain language, the coefficient captures a feedback effect, not a causal impact. PreCallDispersion partially addresses this by controlling for the pre-existing level of dispersion, but does not control for the information flow from the analyst community to management.

2. **Simultaneity.** Both speech uncertainty and analyst dispersion may be driven by the same underlying information environment (e.g., genuinely uncertain firm prospects). The firm FE specifications (cols 2, 4) absorb time-invariant firm-level confounders, and the fiscal year FE absorbs common time shocks, but time-varying firm-specific information quality remains uncontrolled.

3. **Omitted variable bias.** Unobserved call-level characteristics (e.g., complexity of reported results, presence of one-time items, guidance changes) could drive both uncertainty language and analyst disagreement.

**Partial mitigations in the design:**
- PreCallDispersion controls for pre-existing dispersion levels (persistence control).
- Extended controls include loss_dummy and SurpDec (earnings surprise), which proxy for information complexity.
- Analyst_QA_Uncertainty_pct and Entire_All_Negative_pct (extended) capture analyst-side and overall call tone.
- Firm FE absorbs stable between-firm differences.

**What this design cannot establish:** Causal identification of speech uncertainty on dispersion. The coefficients should be interpreted as predictive associations conditional on controls and fixed effects, not as treatment effects.

### 11.2 Inference Assessment

- **Clustering.** Firm-level clustering is appropriate for call-level data nested within firms. It is robust to within-firm serial correlation and heteroskedasticity. Two-way clustering (firm + time) is not implemented but could serve as a robustness check.
- **PanelOLS with duplicate (entity, time) index.** See Section 12 for details. The `set_index(["gvkey", "fyearq_int"])` creates non-unique entries because multiple calls occur per firm-year. For the firm FE specs (cols 2, 4), entity demeaning groups by gvkey (correct), but time demeaning treats all calls in the same firm-year identically. For industry FE specs (cols 1, 3), entity_effects=False, so only time FE and industry FE are applied -- no entity demeaning issues arise. Firm-clustered standard errors should absorb within-firm-year correlation.
- **Multicollinearity.** The 4 key IVs (CEO/Manager x QA/Presentation uncertainty) are likely correlated. No VIF diagnostics are produced. High correlation among IVs would inflate standard errors without biasing coefficients.

### 11.3 Robustness Status

The current implementation includes 4 specifications varying FE type and control set. This constitutes specification sensitivity analysis, not robustness testing. No alternative DV timing windows, placebo tests, instrumental variable approaches, or subsample analyses are implemented.

## 12. Known Limitations & Unresolved Issues

### L-1. FPI Filter Mismatch (MAJOR -- documentation)

**Issue:** The `PostCallDispersionBuilder` docstring (line 15) and metadata (line 109) state FPI='6' (next-quarter EPS forecasts only). The `IbesDetailEngine` actually loads FPI=['6','7'] (line 64 of `_ibes_detail_engine.py`). The builder does not filter to FPI=6 before computing dispersion. Estimates are joined on (gvkey, fpedats), pooling all FPI horizons targeting the same period end date.

**Functional impact:** Likely benign. FPI=6 and FPI=7 both produce estimates for a given FPEDATS (fiscal period end date); pooling them increases the analyst count for each call and is defensible as both reflect consensus about the same fiscal quarter's earnings. However, a replicator following the documentation literally would get different results.

**Status:** Documentation error. The engine comment on line 64 reads "FPI 6 = current quarter, 7 = next quarter (both needed: FPI is relative to estimate date, not call date)" -- this is the engine's rationale for loading both.

### L-2. PanelOLS Duplicate Multi-Index (MAJOR -- econometric)

**Issue:** `set_index(["gvkey", "fyearq_int"])` at line 291 of the runner creates non-unique (entity, time) pairs because most firms have 3-4 earnings calls per fiscal year. PanelOLS interprets the first index level as entity and the second as time.

**Implications:**
- **Industry FE specs (cols 1, 3):** `entity_effects=False`, `time_effects=True`, `other_effects=ff12_code`. No entity demeaning occurs. Time demeaning subtracts the mean of all observations in each fyearq_int, which is correct at the cross-section level. Industry FE absorbs industry means. Coefficients reflect between-entity within-industry-year variation.
- **Firm FE specs (cols 2, 4):** `EntityEffects + TimeEffects`. Entity demeaning groups by gvkey (first index level), which is correct regardless of duplicates. Time demeaning groups by fyearq_int (second index level), treating all calls in the same fiscal year identically -- this is the correct fiscal-year time effect.
- **Standard errors:** Clustering at the entity level (gvkey) absorbs within-firm-year correlation, which is the primary concern with duplicate time indices.

**Status:** Documented. The duplicate index does not introduce bias but should be disclosed. An alternative design would use a unique call-level time index (e.g., calendar quarter or call sequence number), though this would change the interpretation of the time fixed effect.

### L-3. `check_rank=False` in Industry FE Spec (MINOR)

**Issue:** Line 305 of the runner sets `check_rank=False` for the industry FE specification. This suppresses rank-deficiency warnings from PanelOLS.

**Mitigation:** `drop_absorbed=True` handles collinear regressors by dropping them. The rank check is primarily a diagnostic, and suppressing it is non-standard but not harmful given `drop_absorbed=True`.

### L-4. `std()` Uses ddof=1 (MINOR)

**Issue:** The `_dispersion_bulk` method (line 266 of `postcall_dispersion.py`) uses pandas' default `std()` with ddof=1 (sample standard deviation). With NUMEST_MIN=2, the sample standard deviation for a two-analyst call is sqrt(2) times larger than the population standard deviation (ddof=0).

**Impact:** Does not affect cross-sectional rankings but inflates the magnitude of dispersion for low-coverage calls. This is a common convention in the literature and consistent with Diether et al. (2002).

### L-5. Linguistic IV Winsorization Chain Unclear (MINOR)

**Issue:** The 4 key IVs and 2 extended linguistic controls are not explicitly winsorized in the panel builder or the runner. Winsorization depends entirely on whether Stage 2 outputs are pre-winsorized. The LaTeX table note (line 486) states "Variables winsorized at 1%/99%" as a blanket statement.

**Accurate characterization of winsorization by variable group:**
- PostCallDispersion, PreCallDispersion: 1%/99% pooled in `PostCallDispersionBuilder`.
- Compustat controls (Size, TobinsQ, ROA, BookLev, CapexAt, OCF_Volatility): 1%/99% per fiscal year in `CompustatEngine`.
- DividendPayer, loss_dummy: Binary, not winsorized.
- SurpDec: Ordinal decile, not winsorized.
- Linguistic variables (4 IVs + 2 extended): Winsorization status depends on Stage 2 upstream processing.

### L-6. CCM Linking Duplicate Resolution (MINOR)

**Issue:** When multiple gvkeys map to the same CUSIP8 in the CCM table, `drop_duplicates(subset=["cusip8"], keep="first")` is used. "First" is order-dependent and may not select the most appropriate link.

**Impact:** Standard practice in empirical finance. Affects a small number of observations.

### L-7. No Execution Evidence (PROCEDURAL)

**Issue:** This document describes the design and code but does not reference output from a completed run. Sample sizes, R-squared values, and coefficient signs have not been verified against actual output.

## 14. Design Decisions & Rationale

### D-1. Redesign from Pre-Existing H5

The prior H5 implementation had multiple critical issues:
- DV changed 3 times across history (`dispersion_lead` -> `dispersion` -> `delta_dispersion`)
- Code expected columns the panel did not contain
- Architecture was outdated (one IV per spec, 3 samples, calendar year FE)

The redesign follows the H1/H4 pattern: 4 simultaneous IVs, Industry/Firm FE, Base/Extended controls, Main sample only, fiscal year time FE.

### D-2. PostCallDispersion as DV (not delta or level)

The DV is the raw post-call dispersion level (SD/|Mean| x 100), not a change measure. PreCallDispersion is included as a control to capture persistence, effectively making the specification a partial-adjustment model. This follows Druz et al. (2020).

### D-3. FPI=['6','7'] Pooling

The IBES Detail engine loads both FPI=6 and FPI=7 estimates. Both FPI values can target the same FPEDATS (fiscal period end date). Pooling them increases analyst coverage per call. The engine comment explains: "FPI is relative to estimate date, not call date" -- meaning an estimate made well before the call with FPI=7 may target the same quarter as a more recent estimate with FPI=6. This is defensible because the dispersion computation aggregates by FPEDATS regardless of the FPI code.

### D-4. Call-Level Panel with Fiscal Year Time Index

The estimation unit is the individual call, preserving maximum within-firm variation. The time index uses fiscal year (fyearq_int) rather than calendar quarter, because the fixed effect is intended to absorb common annual shocks to analyst behavior. Multiple calls per firm-year are retained as separate observations. See L-2 for implications.

### D-5. Firm-Clustered Standard Errors

Clustering at the firm level accounts for serial dependence across calls within the same firm. This is the standard choice for call-level data with firm as the entity dimension. Within-firm-year correlation is absorbed by firm-level clustering.

### D-6. One-Tailed Test

The hypothesis predicts beta > 0 (more uncertain speech leads to more dispersed forecasts). The one-tailed test is directional: `p_one = p_two / 2` when beta > 0; `p_one = 1 - p_two / 2` otherwise.

### D-7. Minimum 5 Calls Per Firm

Firms with fewer than 5 calls are excluded to ensure sufficient within-firm variation for the firm FE specifications. This threshold is applied after complete-case filtering.

### D-8. Main Sample Only

Finance (FF12=11) and Utility (FF12=8) firms are excluded. These industries have distinct regulatory environments that may affect both analyst behavior and disclosure practices.
