## 5. Merge & Sample Construction Audit

### 5.1 Merge strategy

All variable builders produce DataFrames keyed on `file_name`. The panel builder merges each onto the manifest via `panel.merge(data, on="file_name", how="left")`. A zero-row-delta assertion (`if delta != 0: raise ValueError`) guarantees that no merge inflates or deflates the panel (builder lines 164-169).

### 5.2 CCM linkage -- LINKPRIM inconsistency

**This is a known methodological inconsistency in the current code.**

| Builder | LINKPRIM filter? | Evidence |
|---------|-----------------|----------|
| BidAskSpreadChangeBuilder (DV, PreCallSpread) | Yes -- `linkprim in ["P", "C"]` | `bidask_spread_change.py` lines 144, 159-161 |
| StockPriceBuilder | No -- loads only linkdt, linkenddt | `stock_price.py` lines 96-101 |
| TurnoverBuilder | No -- loads only linkdt, linkenddt | `turnover.py` lines 97-101 |
| VolatilityBuilder (via CRSPEngine.get_data()) | No -- CRSPEngine has no LINKPRIM filter | `_crsp_engine.py` lines 335-349 |

Consequence: The DV and PreCallSpread use primary-link PERMNOs only, while StockPrice, Turnover, and Volatility may use secondary-link PERMNOs. For the same firm-call, different variables could be computed from different PERMNOs. The practical impact is likely small (secondary links are rare for well-covered firms), but this constitutes a methodological inconsistency that should be resolved or defended.

### 5.3 Sample filters (attrition chain)

| Stage | Description | Applied where |
|-------|-------------|---------------|
| 1 | Full panel (all calls in manifest x years) | Builder |
| 2 | Main sample: FF12 not in {8, 11} | Runner line 214-218 |

The MIN_CALLS_PER_FIRM = 5 filter (runner line 121) excludes all firms with fewer than 5 calls in the regression sample. This is a survivorship-type restriction that may bias the sample toward larger, longer-listed firms. It is applied per-specification (base vs extended controls may produce different firm counts due to different missingness patterns).

## 6. Variable Dictionary

### 6.1 Dependent Variable: DSPREAD

| Field | Detail |
|-------|--------|
| **Variable name** | DSPREAD |
| **Definition** | Change in average relative bid-ask spread around the earnings call (Lee 2016, *The Accounting Review*) |
| **Final column** | `DSPREAD` (renamed from `delta_spread_closing` at panel builder line 181) |
| **Source builder** | `BidAskSpreadChangeBuilder` (`src/f1d/shared/variables/bidask_spread_change.py`) |
| **Winsorization** | Pooled 1%/99% at panel builder (`winsorize_pooled()`, builder line 202-203) |

**Complete construction chain:**

```
Step 1: GVKEY-to-PERMNO linkage (BidAskSpreadChangeBuilder._build_permno_map)
  Input:  inputs/CRSPCompustat_CCM/CRSPCompustat_CCM.parquet
  Load:   columns [gvkey, LPERMNO, linkdt, linkenddt, linkprim]
  Filter: linkprim in ["P", "C"]  *** LINKPRIM FILTER PRESENT ***
          (bidask_spread_change.py lines 144, 159-161)
  Join:   manifest[file_name, gvkey, start_date] x CCM on gvkey
  Date-bound: linkdt <= start_date <= linkenddt (line 193-194)
          linkenddt NaN -> filled to 2099-12-31 (line 179)
  Dedup:  multiple valid links -> take most recent linkdt per file_name
          (sort linkdt ascending, groupby("file_name")["LPERMNO"].last())
  Output: DataFrame[file_name, permno_int]

Step 2: Load raw CRSP daily data
  Input:  inputs/CRSP_DSF/CRSP_DSF_{year}_Q{q}.parquet
  Via:    CRSPEngine.get_raw_daily_data() (singleton, cached)
  Load:   columns [PERMNO, date, RET, VWRETD, VOL, PRC, BIDLO, ASKHI, BID, ASK, SHROUT]
          (_crsp_engine.py lines 72-77)
  Clean:  all numeric cols coerced via pd.to_numeric(errors="coerce")
          PRC = abs(PRC) (negative PRC = bid/ask midpoint in CRSP convention)
          PERMNO: coerce, drop NaN, cast to int (_crsp_engine.py lines 95-97)

Step 3: Vectorized spread computation (_compute_spread_change_vectorized)
  Filter: manifest rows with valid permno_int only (line 223-224)
  Year-chunking: process calls by call_year to control memory (line 246)
  Per-year CRSP filter: PERMNO in year_permnos AND crsp_year in [year-1, year+1]
                        (line 260-263)

Step 4: Per-year event-window processing (_process_year_calls)
  4a. Merge: year_calls[file_name, start_date, permno_int] x
             year_crsp[PERMNO, date, BIDLO, ASKHI, BID, ASK] on permno_int=PERMNO
             (inner join, line 309-314)
  4b. Calendar pre-filter: abs(date - start_date) <= 15 days (line 321-322)
  4c. Closing spread per day (lines 340-349):
        valid_closing_mask = ASK.notna() AND BID.notna() AND ASK > 0 AND BID > 0
        spread_closing = 2 * (ASK - BID) / (ASK + BID)
  4d. Reference date (lines 355-359):
        call_ref_date = last trading day on or before start_date
        (merged[date <= start_date], sort by date, groupby file_name, take last)
  4e. Trading-day position assignment (lines 368-383):
        days_from_ref = (date - call_ref_date).days
        Pre-call (days_from_ref < 0): rank by date descending -> pre_rank
          pre_rank=1 is trading day immediately before call_ref_date (day -1)
        Post-call (days_from_ref > 0): rank by date ascending -> post_rank
          post_rank=1 is first trading day after call_ref_date (day +1)
  4f. Window extraction (lines 386-387):
        pre_window  = pre_mask AND pre_rank  <= w   (w=3: days [-3, -2, -1])
        post_window = post_mask AND post_rank <= w  (w=3: days [+1, +2, +3])
  4g. Closing-spread aggregation (lines 412-434):
        pre_call_spread_closing  = mean(spread_closing) over pre_window per file_name
        post_call_spread_closing = mean(spread_closing) over post_window per file_name
        pre_n_valid_c  = count(non-null spread_closing) in pre_window
        post_n_valid_c = count(non-null spread_closing) in post_window
        delta_spread_closing = post_call_spread_closing - pre_call_spread_closing
  4h. Minimum valid days filter (lines 425-430):
        min_pre = max(1, w - 1) = 2 for w=3  (line 301)
        min_post = max(1, w - 1) = 2 for w=3 (line 302)
        If pre_n_valid_c < 2 OR post_n_valid_c < 2:
          delta_spread_closing -> NaN
          pre_call_spread_closing -> NaN

Step 5: Panel-level rename and winsorize
  Rename: delta_spread_closing -> DSPREAD (panel builder line 181)
  Winsorize: pooled 1%/99% via winsorize_pooled() (panel builder line 202-203)
    Method: global quantile(0.01) and quantile(0.99) across ALL observations
            (all industry samples, not just Main), clip to [p1, p99]
    Min obs: 100 (winsorization.py line 93)
```

**Minimum valid days rule:** With default window w=3, the builder requires at least 2 of 3 valid trading days in each window (`bidask_spread_change.py` lines 300-302). A call with only 2 valid pre-window or post-window days still receives a DSPREAD value. This relaxation increases coverage at the cost of noisier estimates for edge cases.

**LINKPRIM filter (critical inconsistency):** BidAskSpreadChangeBuilder applies `linkprim in ["P","C"]` when mapping GVKEY to PERMNO (line 159-161). This restricts the DV to primary-link securities only. StockPriceBuilder, TurnoverBuilder, and CRSPEngine (Volatility) do NOT apply this filter -- they load only `[gvkey, LPERMNO, linkdt, linkenddt]` from the CCM table (stock_price.py lines 96-101; turnover.py lines 97-101; _crsp_engine.py lines 335-349). This means the DV and PreCallSpread may use a different PERMNO than StockPrice, Turnover, and Volatility for the same firm-call if a secondary link exists.

### 6.2 PreCallSpread (Base Control / Lagged-DV Control)

| Field | Detail |
|-------|--------|
| **Variable name** | PreCallSpread |
| **Definition** | Pre-call relative spread level: mean of daily RelSpread over the [-3,-1] trading-day window (closing quotes) |
| **Final column** | `PreCallSpread` (renamed from `pre_call_spread_closing` at panel builder line 182) |
| **Source builder** | `BidAskSpreadChangeBuilder` (same builder as DSPREAD) |
| **Winsorization** | Pooled 1%/99% at panel builder (`winsorize_pooled()`, builder line 202-203) |
| **Role** | Lagged-DV control -- absorbs the pre-call spread level so DSPREAD captures the *change* conditional on the starting point |

**Construction chain:** Computed jointly with DSPREAD in Step 4g above. PreCallSpread = `pre_call_spread_closing` = mean(spread\_closing) over the pre-window (trading days [-3, -2, -1] relative to call\_ref\_date). Subject to the same minimum-valid-days filter (2 of 3 required). Uses the same LINKPRIM-filtered PERMNO mapping as DSPREAD.

#### 6.4.2 TobinsQ

| Field | Detail |
|-------|--------|
| **Formula** | `(cshoq * prccq + debt_book) / atq` |
| **Numerator** | Market equity (`cshoq * prccq`) + book debt (`dlcq.clip(0).fillna(0) + dlttq.clip(0).fillna(0)`) |
| **debt_book rule** | When BOTH dlcq and dlttq are NaN -> debt_book = NaN (not zero); when only one is NaN -> the other is used (NaN treated as 0 via fillna) |
| **NaN conditions** | atq is null, atq <= 0, or mktcap (cshoq*prccq) is NaN |
| **Source** | `_compute_and_winsorize()` (_compustat_engine.py lines 1067-1077) |
| **Approximation** | Chung & Pruitt (1994): market equity + interest-bearing book debt over total book assets |
| **Winsorization** | Per-year (fyearq) 1%/99%, inf -> NaN first (_compustat_engine.py line 1152, 1172, 1199-1201) |

**Construction chain:** `comp_na_daily_all.parquet` -> load `[cshoq, prccq, dlcq, dlttq, atq]` -> `mktcap = cshoq * prccq` -> `debt_c = dlcq.clip(0).fillna(0)` -> `debt_t = dlttq.clip(0).fillna(0)` -> `debt_book = np.where(dlcq.isna() & dlttq.isna(), NaN, debt_c + debt_t)` -> `TobinsQ = np.where(atq > 0 & mktcap.notna(), (mktcap + debt_book) / atq, NaN)` -> replace inf with NaN -> `_winsorize_by_year` -> merge_asof -> panel.

## 11. Identification & Inference Assessment

### 11.1 Identification strategy

The suite uses a horse-race design: four uncertainty measures enter simultaneously, allowing within-model comparison of which dimension of speech uncertainty (if any) predicts spread changes. The design relies on cross-sectional variation in call-level uncertainty after controlling for firm (or industry) and time fixed effects.

### 11.2 Endogeneity concerns

- **Reverse causality:** Firms experiencing market stress (wider spreads) may hold calls with more uncertain language. PreCallSpread as a control partially addresses this by absorbing the pre-call spread level, but does not fully resolve the concern.
- **Omitted variables:** No controls for institutional ownership, analyst coverage, or news flow around the call. These could confound both uncertainty language and spread changes.
- **Selection:** The sample conditions on firms having earnings calls, excluding smaller firms without transcripts.

### 11.3 Fixed effects adequacy

The time FE uses fiscal year (`fyearq_int`), not fiscal quarter. With ~4 calls per firm per fiscal year, this means:
- Annual macro shocks are absorbed.
- Quarterly seasonality in bid-ask spreads (e.g., end-of-quarter effects) is NOT absorbed.
- This is a defensible choice if within-year variation is desired for identification, but a fiscal-quarter FE robustness check would strengthen the design.

### 11.4 Multiple testing

With 4 IVs x 4 specifications = 16 hypothesis tests, a naive 5% threshold yields approximately 1 expected false positive under the null. No multiple-testing correction (Bonferroni, FDR) is applied. This should be disclosed in the thesis.

### 11.5 Clustering

Firm-clustered SEs are appropriate for a call-level panel where observations within the same firm are likely correlated. The clustering entity is `gvkey` (the first index level).

### 11.6 Robustness

**No robustness tests exist in the current code.** The prior version (pre-redesign) had a 174-specification robustness grid including placebo tests, alternative DV constructions (high-low spread, alternative windows w=1 and w=5), and control-set variants. All of this infrastructure was removed during the redesign. The prior red-team audit (`docs/provenance/Audits/H14_red_team.md`, dated 2026-03-15) found that the only significant primary-DV finding in the old code (CEO_Pres, p=0.024) failed the placebo test (placebo beta was 2.8x the primary beta) and did not survive alternative DV constructions. This history should inform interpretation of any future results.

## 14. Design Decisions & Rationale

### Decision 1: Closing quotes instead of intraday high-low

The DV uses closing BID/ASK quotes (`ASK`, `BID` columns in CRSP), not intraday high-low (`ASKHI`, `BIDLO`). This follows Lee (2016). The BidAskSpreadChangeBuilder computes both variants but the panel builder selects the closing-quote version by renaming `delta_spread_closing` to `DSPREAD` (builder line 181).

### Decision 2: 4 simultaneous IVs (reduced from 6)

The redesign reduced the IV set from 6 (including CEO/Manager Clarity Residuals) to 4 (uncertainty measures only). This is a deliberate simplification to focus the hypothesis test on uncertainty. The clarity residuals were removed because:
- The clarity residual is a constructed variable (residual from a cross-sectional regression) that adds complexity without clear theoretical motivation for H14.
- In the pre-redesign run, CEO Clarity Residual showed a positive coefficient (higher clarity -> wider spreads), which is theoretically puzzling and opposite to the expected direction. Removing it avoids presenting a result that lacks a coherent interpretation.
- This is NOT p-hacking: the removal narrows the hypothesis to a more focused test. However, the pre-redesign clarity results should be disclosed in the thesis discussion.

### Decision 3: 4-column grid (reduced from 174-spec robustness)

The redesign replaced the 174-specification robustness grid with a clean 4-column design (2 FE types x 2 control sets). This follows the thesis-standard pattern used across all redesigned suites. The trade-off is reduced robustness evidence, which should be restored in future work (placebo tests, alternative DV windows, alternative spread formulas).

### Decision 4: LINKPRIM filter applied to DV builder only

The LINKPRIM filter (`linkprim in ["P", "C"]`) was added to BidAskSpreadChangeBuilder during bug-fixing but was not propagated to StockPriceBuilder, TurnoverBuilder, or CRSPEngine. This inconsistency should be resolved by either (a) adding LINKPRIM filters to all CRSP builders, or (b) documenting an explicit rationale for the asymmetry. The current state is an incomplete fix.

### Decision 5: Fiscal year (not quarter) time FE

The time FE uses `fyearq_int = floor(fyearq)`, which is the fiscal year. This was a deliberate change from the pre-redesign code (which used calendar year-quarter). The fiscal-year FE absorbs annual macro shocks while preserving within-year variation for identification. The trade-off is that quarterly seasonality is not absorbed. A fiscal-quarter FE robustness check would strengthen the design.

### Decision 6: Removal of calendar-quarter deduplication

The pre-redesign code deduplicated observations on calendar quarter. The redesign removed this step because PanelOLS handles non-unique multi-index entries. This means multiple calls from the same firm in the same fiscal year all enter the regression, which is correct for a call-level analysis but means the "time FE" is fiscal-year, not per-call-occasion.
