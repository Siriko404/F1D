### DV Construction Detail

`REPO_callqtr` captures whether the firm repurchased shares in the fiscal quarter **containing** the earnings call (one quarter after the reporting quarter matched by `merge_asof`). Construction:

1. The Compustat engine's `merge_asof(direction="backward")` matches each call to its **reporting** quarter (the most recent `datadate <= start_date`).
2. The panel builder reads raw Compustat quarterly data and builds a `(gvkey, fyearq_int, fqtr_int) -> REPO_qtr` lookup where `REPO_qtr = 1 if cshopq > 0, 0 if cshopq == 0, NaN if cshopq is NaN`.
3. For each call, the next-quarter indices are computed: `next_fqtr = fqtr_int % 4 + 1` (yielding 1->2, 2->3, 3->4, 4->1) with `next_fyearq = fyearq_int + (fqtr_int == 4)` for fiscal year rollover.
4. A left merge on `(gvkey, next_fyearq, next_fqtr)` attaches the call-quarter REPO value.
5. Row-count validation: the merge is checked to ensure no fan-out (`after_len != before_len` raises `ValueError`).

**Duong et al. method identification:** This implements Method 1 from Duong, Do, and Do (2025) -- the simplest quarterly indicator based on `cshopq > 0`. Methods 2--5 in their paper use monthly timing refinements requiring 10-Q/10-K filing dates not available in our data. The quarterly approach is standard in the repurchase literature and is sufficient for testing the uncertainty-repurchase relationship at the quarterly horizon.

**Base rate:** Prior run reports 70.5% of calls with non-null REPO_callqtr have REPO_callqtr=1. See Section 11 for econometric implications.

## 6. Variable Dictionary

### Dependent Variable: `REPO_callqtr`

| Field | Detail |
|-------|--------|
| **Role** | Dependent variable (binary) |
| **Final formula** | `1` if `cshopq > 0` in the call quarter; `0` if `cshopq == 0`; `NaN` if `cshopq` is missing |
| **Method** | Duong, Do, and Do (2025), Method 1: quarterly `cshopq > 0` |
| **Winsorized** | No (binary; listed in engine `skip_winsorize` set, engine line 1205) |
| **Year restriction** | `max(config.data.year_start, 2004)` -- `cshopq` is 100% NaN in 2002, 99.7% NaN in 2003 (builder line 335) |

**Complete construction chain:**

1. **Raw field.** Compustat quarterly field `cshopq` ("Purchase of Common and Preferred Stock -- Quarter") is loaded from `inputs/comp_na_daily_all/comp_na_daily_all.parquet` (engine `REQUIRED_COMPUSTAT_COLS`, line 182). `cshopq` is a true quarterly flow (NOT a YTD cumulative like `capxy` or `dvy`), so no Q4-only extraction is needed.

2. **Engine-level REPO (reporting quarter).** `_compute_and_winsorize()` (engine lines 1158--1162) computes `REPO` on the full Compustat quarterly panel:
   ```python
   comp["REPO"] = np.where(
       comp["cshopq"].notna() & (comp["cshopq"] > 0),
       1.0,
       np.where(comp["cshopq"].notna(), 0.0, np.nan),
   )
   ```
   This preserves the three-way distinction: repurchased (1), did not repurchase (0), field not reported (NaN). `REPO` is added to `COMPUSTAT_COLS` (engine line 140) and excluded from winsorization via `skip_winsorize` (engine line 1205).

3. **RepurchaseIndicatorBuilder.** `repurchase_indicator.py` wraps the engine: calls `engine.match_to_manifest(manifest, root_path)` which performs `pd.merge_asof(manifest, comp, left_on="start_date", right_on="datadate", by="gvkey", direction="backward")`, matching each call to its **reporting quarter** (the most recent `datadate <= start_date`). Returns `file_name`, `REPO`, and `fqtr` (fiscal quarter 1--4).

4. **Quarter-lead logic (REPO_callqtr).** `create_repo_callqtr()` in the panel builder (lines 202--305) constructs the call-quarter indicator:
   - **Step 1:** `attach_fyearq(panel, root_path)` attaches the fiscal year (`fyearq_int`) via a separate canonical backward `merge_asof`.
   - **Step 2:** Reads raw Compustat quarterly again for `(gvkey, fyearq, fqtr, cshopq)` directly (builder lines 242--245). Computes `REPO_qtr` with the same three-way logic as the engine (builder lines 251--255). Deduplicates the lookup by `(gvkey, fyearq_int, fqtr_int)` keeping the last row (builder lines 258--263).
   - **Step 3:** Computes next-quarter indices (builder lines 271--272):
     ```python
     panel["next_fqtr"]   = panel["fqtr_int"] % 4 + 1     # 1->2, 2->3, 3->4, 4->1
     panel["next_fyearq"] = panel["fyearq_int"] + (panel["fqtr_int"] == 4).astype(float)
     ```
     When `fqtr_int == 4`, the fiscal year rolls over (e.g., Q4 FY2010 leads to Q1 FY2011).
   - **Step 4:** Left merge on `(gvkey, next_fyearq, next_fqtr)` attaches `REPO_callqtr` from the Compustat lookup (builder lines 282--286). Post-merge row-count validation: `after_len != before_len` raises `ValueError` (builder lines 288--292).
   - **Step 5:** Temporary columns `next_fqtr` and `next_fyearq` are dropped (builder line 295). If the next quarter does not exist in Compustat, `REPO_callqtr` is NaN.

5. **Year restriction.** Applied at builder line 335: `year_start = max(config.data.year_start, 2004)`, ensuring no pre-2004 calls enter the panel.

### Key Independent Variables (4 simultaneous)

All four IVs follow the same construction chain, differing only in the speaker-section combination.

| Variable | Speaker | Section | Column key |
|----------|---------|---------|------------|
| `CEO_QA_Uncertainty_pct` | CEO | Q&A | `CEO_QA_Uncertainty_pct` |
| `CEO_Pres_Uncertainty_pct` | CEO | Presentation | `CEO_Pres_Uncertainty_pct` |
| `Manager_QA_Uncertainty_pct` | All managers | Q&A | `Manager_QA_Uncertainty_pct` |
| `Manager_Pres_Uncertainty_pct` | All managers | Presentation | `Manager_Pres_Uncertainty_pct` |

**Complete construction chain (shared across all 4):**

1. **Raw transcripts.** Earnings-call transcripts are parsed and segmented by speaker and section (Presentation vs. Q&A) in Stage 1 of the pipeline.

2. **Tokenization and dictionary matching (Stage 2).** Each speaker-section segment is tokenized. The Loughran-McDonald (LM) Uncertainty dictionary is applied: each token is checked for membership in the LM Uncertainty word list. The uncertainty percentage is computed as:
   ```
   {Speaker}_{Section}_Uncertainty_pct = (count of uncertainty words / total word count) * 100
   ```
   Output: one parquet file per year at `outputs/2_Textual_Analysis/2.2_Variables/linguistic_variables_{year}.parquet`.

3. **LinguisticEngine loading and caching.** `_linguistic_engine.py` loads all year-partitioned parquet files, concatenates them, and caches the result as a singleton (lines 165--264). Year filtering is applied based on the `years` range argument.

4. **Per-year winsorization (upper-only 0%/99%).** The LinguisticEngine applies `winsorize_by_year()` (from `winsorization.py`) to all `_pct` columns with `lower=0.0, upper=0.99, min_obs=10` (engine line 255--258). This clips the upper tail at the 99th percentile within each year while leaving the lower bound at 0% (percentage variables cannot be negative). Years with fewer than 10 valid observations are left unwinsorized.

5. **Builder extraction.** Each builder (e.g., `CEOQAUncertaintyBuilder` in `ceo_qa_uncertainty.py`) queries the singleton LinguisticEngine and extracts `["file_name", column]`. No additional transformation is applied.

6. **Panel merge.** Each builder's output is left-merged into the panel on `file_name` (builder lines 176). Post-merge row-count validation ensures no fan-out.

#### `TobinsQ`

| Field | Detail |
|-------|--------|
| **Role** | Base control |
| **Code formula** | `(cshoq * prccq + debt_book) / atq` where `atq > 0` and `mktcap` is non-null (engine lines 1069--1079) |
| **Raw Compustat fields** | `cshoq` (Common Shares Outstanding), `prccq` (Price Close -- Quarter), `dlcq` (Debt in Current Liabilities), `dlttq` (Long-Term Debt -- Total), `atq` (Total Assets) |
| **Debt clipping logic** | `debt_c = dlcq.clip(lower=0).fillna(0)`, `debt_t = dlttq.clip(lower=0).fillna(0)`. Negative debt values are clipped to zero; NaN debt is filled to zero. However, if BOTH `dlcq` and `dlttq` are NaN, `debt_book = NaN` (engine lines 1070--1074), making TobinsQ NaN (no silent zero-debt assumption when both fields are missing). |
| **Guard conditions** | `atq > 0`, `atq` is not null, and `mktcap = cshoq * prccq` is not null. All three must hold; otherwise TobinsQ = NaN. |
| **Inf replacement** | Inf/-Inf replaced with NaN (engine line 1186) |
| **Winsorized** | Per fiscal year, 1%/99% (engine line 1215) |
| **Construction chain** | `comp_na_daily_all.parquet` -> `_compute_and_winsorize()` -> `mktcap = cshoq * prccq` -> `debt_book = clip(dlcq, 0).fillna(0) + clip(dlttq, 0).fillna(0)` (NaN if both missing) -> `(mktcap + debt_book) / atq` -> inf replacement -> per-year winsorization -> `merge_asof` -> panel merge |

**TobinsQ formula note:** The code computes `(market cap + interest-bearing book debt) / total assets`, i.e., `(cshoq*prccq + dlcq + dlttq) / atq`. This is a standard enterprise-value-based Tobin's Q proxy but differs from the textbook `(MV equity + BV liabilities) / BV assets` formula, which would use `atq - ceqq` in place of `dlcq + dlttq`. The code's formula captures only interest-bearing debt, excluding operating liabilities (accounts payable, accruals, deferred revenue). For a typical S&P 500 firm, operating liabilities are 15--30% of total assets, so the two formulations can differ materially. Both are standard in the empirical corporate finance literature. The code implementation is used consistently across all hypothesis suites.

#### `OCF_Volatility`

| Field | Detail |
|-------|--------|
| **Role** | Base control |
| **Code formula** | Rolling 5-year std of `oancfy / atq_lag` on Q4 annual observations, min 3 years (engine lines 309--358) |
| **Raw Compustat fields** | `oancfy` (Operating Activities -- Net Cash Flow -- YTD), `atq` (Total Assets), `fqtr` (fiscal quarter) |
| **Step-by-step** | (1) Filter to `fqtr == 4` rows, deduplicate by `(gvkey, fyearq)` keeping last. (2) Compute `atq_lag = groupby("gvkey")["atq"].shift(1)`. (5) Compute rolling std: `groupby("gvkey")["ocf_ratio"].transform(lambda x: x.rolling("1826D", min_periods=3).std())`. The 1826-day window (approx. 5 years) with `min_periods=3` requires at least 3 non-NaN observations. (6) Join back to full quarterly panel via `(gvkey, fyearq)` left merge with `validate="m:1"`. |

## 11. Identification & Inference Assessment

### Identifying Variation

The identifying variation comes from **within-entity, within-fiscal-year** variation in quarterly speech uncertainty predicting quarterly repurchase decisions, after absorbing entity (firm or industry) and fiscal year fixed effects. The unit of observation is the individual earnings call, not the firm-year.

### Panel Index: Non-Unique (gvkey, fyearq_int)

The panel is indexed by `(gvkey, fyearq_int)`, but the data is call-level: a firm with 4 quarterly calls in a fiscal year has 4 rows sharing the same `(gvkey, fyearq_int)` value. `PanelOLS` accepts this without error because `linearmodels` allows repeated observations per entity-time cell. However, this means:

1. **"Within-R-squared" interpretation:** The within-R-squared (~ 2%) measures variation explained after absorbing entity and time means computed over all observations in each cell, not the standard one-observation-per-cell panel within-estimator.
2. **Entity effects:** Firm FE absorbs the firm-level mean across all calls (including multiple calls per year), which is the correct within-firm variation for this research question.
3. **Time effects:** Fiscal year FE absorbs the year mean across all calls in that year, consistent with the quarterly-call-within-fiscal-year identification.

This is a legitimate design choice given the call-level research question, but differs from standard two-way FE panel interpretation and should be acknowledged in the thesis text.

### LPM Limitations

This suite uses a Linear Probability Model (OLS with binary DV). The following limitations apply:

1. **Heteroskedasticity is guaranteed.** With a Bernoulli DV, `Var(e|X) = p(X)(1-p(X))`. Firm-clustered standard errors are heteroskedasticity-consistent (HC1 by default in `linearmodels`), so inference is valid, but this is a **minimum requirement**, not a robustness feature.

2. **Predicted probabilities outside [0,1].** The LPM does not constrain fitted values to [0,1]. With a 70.5% base rate, the probability mass is concentrated near the upper bound, increasing the likelihood of predictions > 1. Per Horrace and Oaxaca (2006), if more than 5% of fitted values fall outside [0,1], LPM coefficient estimates may be inconsistent. **No predicted probability diagnostics have been computed for this suite.** This is a required diagnostic that should be added before thesis submission.

3. **Incidental parameters problem justifies LPM for firm FE specs (Cols 2, 4).** With firm fixed effects, logit/probit suffer from the incidental parameters problem in short panels. The LPM avoids this. However, **industry FE models (Cols 1, 3) have no incidental parameters problem**, and a logit robustness check for these specifications is straightforward and would strengthen the thesis.

4. **Marginal effects interpretation.** LPM coefficients are directly interpretable as average marginal effects on P(REPO_callqtr = 1). The LaTeX table note states "Coefficients represent marginal effects on the probability of share repurchase." This is correct as a statement about average marginal effects but does not hold at specific covariate values (unlike probit AMEs).

### High Base Rate Implications

The 70.5% base rate for REPO_callqtr=1 has several implications:

1. **Imbalanced binary outcome:** The unconditional probability is far from 0.5, meaning the DV is skewed toward 1.
2. **Limited within-firm variation:** Firms that always repurchase (prior run: 248 always-repurchase firms absorbed by firm FE) contribute no identifying variation to firm FE specifications.
3. **Low R-squared context:** The within-R-squared of ~ 2% is partially explained by the DV being near its ceiling, limiting the scope for additional explanatory power.
4. **LPM upper-bound violations:** With baseline predictions near 0.7, positive coefficients on controls or IVs can push fitted values above 1.0 more easily than with a 0.5 base rate.

### Endogeneity Concerns

The suite does not employ instrumental variables or exploit quasi-experimental variation. Potential endogeneity threats include:

- **Reverse causality:** Firms may adjust speech patterns because they plan to repurchase (e.g., downplaying uncertainty to support buyback announcements).
- **Omitted variable bias:** Unobserved quarterly shocks (board pressure, insider information, market timing) may jointly determine both speech uncertainty and repurchase decisions.
- **Simultaneity:** Quarterly earnings calls and repurchase decisions may respond to the same unobserved information.

The call-quarter alignment (uncertainty measured in the call, repurchases measured in the same quarter) means the speech and repurchase measurements are contemporaneous, not leading. This limits causal interpretation.

### Collinearity Risk

Four uncertainty measures enter simultaneously: CEO QA, CEO Pres, Manager QA, Manager Pres. These are likely correlated (especially within-role across sections, or within-section across roles). No VIF or condition number diagnostics are computed. With null results across all 16 IV-specification combinations, multicollinearity could be masking individual effects. VIF computation should be added to the diagnostics.

### Multiple Comparisons

4 IVs x 4 specifications = 16 tests. All results are null. Multiple comparison corrections (Bonferroni, Holm) are not needed when all results are insignificant -- corrections only matter for false positive control.

## 14. Design Decisions & Rationale

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | **Quarterly REPO (not monthly)** | Duong et al. (2025) Method 1 uses quarterly Compustat `cshopq`. Monthly timing refinements (Methods 4/5) require 10-Q/10-K filing dates not available in our data pipeline. The quarterly horizon matches the earnings call frequency. |
| D2 | **Call-quarter alignment (one-quarter lead)** | The engine's `merge_asof` matches calls to the reporting quarter. Duong et al. match calls to the quarter **containing** the call, which is one quarter after the reporting quarter. The panel builder implements this via fiscal-quarter-validated lead. |
| D3 | **LPM (not logit/probit)** | Firm FE specifications (Cols 2, 4) would suffer from the incidental parameters problem in nonlinear models. The LPM avoids this while producing coefficients directly interpretable as average marginal effects. Firm-clustered SEs correct for the guaranteed heteroskedasticity. Trade-off: no constraint to [0,1], no robustness check for industry FE specs. |
| D4 | **Year restriction 2004--2018** | `cshopq` is 100% NaN in 2002 and 99.7% NaN in 2003 (field not reported by Compustat for most firms). Enforced in code: `year_start = max(config.data.year_start, 2004)`. |
| D5 | **8 base controls (not 7)** | REPO is the DV, so all three of {BookLev, CashHoldings, CapexAt} enter as controls. In suites where one of these is the DV, only 7 base controls are used. |
| D6 | **Binary DV (not intensity)** | The extensive margin (whether to repurchase) is the primary object of interest. This is acceptable: identifying variation comes from firms that switch repurchase status. |
