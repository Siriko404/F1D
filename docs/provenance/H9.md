## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Suite ID | H9 |
| Suite name | Takeover Hazard Models |
| Research question | Does clarity in speech increase the likelihood of receiving a takeover bid, especially an uninvited bid? |
| Estimation unit | Call-to-call counting-process interval (firm × interval) |
| Primary keys | `gvkey` (firm) × `start`/`stop` (interval endpoints in days since 2000-01-01) |

### A2) Model family and estimator

| Field | Value |
|-------|-------|
| Model family | Survival / Hazard — Cox Proportional Hazards (time-varying covariates) |
| Estimator | `lifelines.CoxTimeVaryingFitter` (counting-process format) |
| Objective | Partial likelihood (Cox) |
| Optimizer | `lifelines` default (Newton-Raphson via internal implementation) |
| Tie method | Efron (lifelines default for `CoxTimeVaryingFitter`) |
| Regularization | None |
| Weighting | None (no weights/offset/exposure) |
| Model 1 | Cox PH — All Takeovers (event = `Takeover`) |
| Model 2 | Cause-specific Cox PH — Uninvited (event = `Takeover_Uninvited`) |
| Model 3 | Cause-specific Cox PH — Friendly (event = `Takeover_Friendly`) |

### A3) Outcome / target definition

| Outcome | Definition | Coding |
|---------|-----------|--------|
| `Takeover` | Any takeover bid (Completed, Withdrawn, Pending) for US public target in sample period | 1 in the interval containing the bid announcement; 0 otherwise |
| `Takeover_Uninvited` | Bid with Deal Attitude = Hostile or Unsolicited | 1 if `Takeover=1` AND `Takeover_Type='Uninvited'`; 0 otherwise |
| `Takeover_Friendly` | Bid with Deal Attitude = Friendly or Neutral | 1 if `Takeover=1` AND `Takeover_Type='Friendly'`; 0 otherwise |
| Unknown-type events | 26 events with `Takeover_Type='Unknown'` | Correctly treated as censored in both cause-specific models |

**Timing:** Event occurs at SDC `Date Announced`. Covariates measured at the call that opens each interval (pre-event). Intervals are call-to-call, not calendar.

**Missingness → drop:** Listwise deletion on all covariates + start/stop/event. Rows with NaN clarity or controls are dropped.

### A4) Main RHS and controls

**Key independent variables (3 clarity variants, run as separate models):**

| Variant | Variable | Family | Description |
|---------|----------|--------|-------------|
| CEO | `ClarityCEO` | primary_style | CEO fixed effect from H1 clarity regression |
| CEO_Residual | `CEO_Clarity_Residual` | secondary_residual | Residualized CEO uncertainty from H0.3 extended regression |
| Manager_Residual | `Manager_Clarity_Residual` | secondary_residual | Residualized manager uncertainty from H0.3 extended regression |

**Control blocks:**

| Block | Variables | Where defined |
|-------|-----------|---------------|
| Sparse (all models) | Size, BM, Lev, ROA, CashHoldings | `run_h9_takeover_hazards.py:130-136` |
| Expanded (robustness) | Sparse + SalesGrowth, Intangibility, AssetGrowth | `run_h9_takeover_hazards.py:139-143` |

**Sample filter:** Main sample = FF12 codes 1–7, 9–10, 12 (excludes Finance ff12=11 and Utility ff12=8). Applied at `run_h9_takeover_hazards.py:277`.

### A5) Fixed effects equivalent + variance estimator

| Field | Value |
|-------|-------|
| Fixed effects | None (no entity/time FE in base models) |
| Stratified baseline hazard | Year-stratified (`strata="year"`) and Industry-stratified (`strata="ff12_code"`) as separate robustness models |
| Standard errors | Robust sandwich estimator (lifelines default for `CoxTimeVaryingFitter`) |
| Clustering | **NOT CLUSTERED** — lifelines `CoxTimeVaryingFitter` uses robust sandwich SE but does NOT cluster by firm |
| Small-sample corrections | None applied |

### A6) Survival / Hazards addendum

| Field | Value |
|-------|-------|
| Time origin (t₀) | 2000-01-01 (arbitrary reference date) |
| Time scale | Calendar days since 2000-01-01 |
| Duration construction | `start` = call_date − 2000-01-01 (days); `stop` = min(next_call_date, takeover_date, censor_date) − 2000-01-01 |
| Event definition | `Takeover=1` only in the interval where SDC `Date Announced` falls within (start, stop] |
| Censoring definition | `Takeover=0` for all non-event intervals; firms without bids are censored at final interval close |
| Admin censor date | 2018-12-31 (= `year_end` from config) |
| Risk set construction | Counting-process (start/stop) format; one row per call-to-call interval; `CoxTimeVaryingFitter` handles late entry natively |
| Late entry / left truncation | Firms enter when their first earnings call occurs (late entry handled by counting-process format) |
| Ties method | Efron (lifelines default) |
| Competing risks approach | Cause-specific Cox PH (Model 2 = Uninvited, Model 3 = Friendly); Unknown-type events treated as censored in both cause-specific models |
| Interval cap | 4 years (1,461 days); event intervals exceeding cap are censored (`Takeover→0`) at `build_h9_takeover_panel.py:342-359` |
| Zero/negative duration | Removed at `build_h9_takeover_panel.py:373-377` |
| Multi-event validation | Validated: each firm has at most 1 event row (`build_h9_takeover_panel.py:384-390`) |

## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

All Compustat-derived controls are winsorized at 1%/99% **by fiscal year** (fyearq) in the CompustatEngine. Minimum 10 observations per year-group required; years with <10 are left unwinsorized. Applied before merge to manifest.

Implemented at `_compustat_engine.py:1146-1160` via `_winsorize_by_year()`.

**Variables NOT winsorized:** ClarityCEO (standardized upstream), CEO_Clarity_Residual, Manager_Clarity_Residual (regression residuals — no winsorization by design), Takeover/event indicators (binary), DividendPayer (binary), SalesGrowth (already winsorized inside `_compute_biddle_residual`).

### Missing data policy

- **Listwise deletion** within each model: `run_cox_tv()` drops rows where any covariate or start/stop/event is NaN (`run_h9_takeover_hazards.py:441-444`)

- **Interval cap:** 4 years (1,461 days). Intervals exceeding cap → stop capped at start + 1,461. Event intervals exceeding cap → `Takeover` set to 0 (censored). Rationale: unreasonably long intervals (e.g., firm stops filing calls) should not influence hazard estimates.
- **Zero/negative duration:** Removed (validated: `start < stop` for all retained rows)
- **Post-takeover calls:** Calls on or after takeover date are removed (firm no longer at risk)

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line referee verdict

This suite implements a carefully-constructed counting-process survival analysis examining whether managerial clarity affects takeover hazard. The interval construction is sound, with appropriate event-time construction, administrative censoring, interval capping, and cause-specific competing risks handling. However, the uniformly null results (all clarity coefficients statistically insignificant with p > 0.49, concordance indices near 0.50), combined with critical methodological gaps — particularly the **absence of clustered standard errors** (a serious deficiency for a multi-interval panel), the **unverified proportional hazards assumption**, and **substantial structural missingness** in the key explanatory variable — place this suite below thesis standard as currently implemented. The null finding itself is potentially informative, but the inference apparatus has material gaps that prevent a thesis committee from being confident that the null is genuine rather than an artifact of methodological limitations.

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

The implementation currently supports:
- **Descriptive association:** YES, with caveats about sample selection and clustering
- **Predictive relation:** NO — concordance near 0.50 across all models indicates no predictive power
- **Causal interpretation:** NO — no identification strategy beyond controls
- **Structural interpretation:** NO

The write-up would be overselling if it claims anything beyond "we find no descriptive association between managerial clarity and takeover hazard in this sample, but our power to detect cause-specific effects (especially uninvited) is limited."

## N. Final Readiness Statement

**Can this suite be reproduced end-to-end from code alone?**
YES. Both panel build and estimation scripts are fully runnable with clear input/output paths, timestamped outputs, and manifest documentation.

**Can a committee/referee understand exactly how the variables are built?**
YES. Variable construction is explicit, documented in builders, and traceable to raw Compustat/SDC fields through the CompustatEngine and TakeoverIndicatorBuilder.

**Are the implemented results trustworthy for descriptive inference?**
PARTIALLY. The null association between clarity and takeover hazard is consistently observed across 36 specifications. However, unclustered standard errors, unverified PH assumption, and structural missingness of the key explanatory variable introduce sufficient uncertainty that a referee cannot fully trust the precision of the null.

**Are they trustworthy for causal inference?**
NO. No identification strategy beyond controls. Known omitted variables (governance, ownership). The design supports only descriptive association.

**What is the single biggest threat to credibility?**
**Unclustered standard errors.** For a counting-process panel with ~40 intervals per firm, failing to cluster at the firm level is a fundamental inferential error. While the all-null result means this does not change the qualitative conclusion, it undermines the methodological credibility of the suite and would be flagged immediately by any econometrics-aware reviewer.

**What is the single highest-value next fix?**
**Implement firm-level clustered standard errors** — either via `CoxPHFitter` with `cluster_col='gvkey'` (requires restructuring to use last-observation data or aggregate panel), or via clustered bootstrap of the `CoxTimeVaryingFitter` estimates.
