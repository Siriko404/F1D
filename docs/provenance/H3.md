## A. Suite Overview

| Field | Value |
|---|---|
| **Suite name / ID** | H3 — Payout Policy |
| **Research question** | Does managerial linguistic uncertainty affect corporate payout policy? |
| **Estimation unit** | Earnings call (one row per call); call-level PanelOLS |
| **Dependent variables** | `div_stability_lead` (dividend stability at t+1); `payout_flexibility_lead` (payout flexibility at t+1) |
| **Key independent variables** | Uncertainty measures (6): `Manager_QA_Uncertainty_pct`, `CEO_QA_Uncertainty_pct`, `Manager_QA_Weak_Modal_pct`, `CEO_QA_Weak_Modal_pct`, `Manager_Pres_Uncertainty_pct`, `CEO_Pres_Uncertainty_pct` |
| **Controls** | `earnings_volatility`, `fcf_growth`, `firm_maturity`, `Size`, `ROA`, `TobinsQ`, `CashHoldings`, `Lev` |
| **Fixed effects** | Firm (`EntityEffects` via PanelOLS) + Year (`TimeEffects` via PanelOLS) |
| **Standard errors** | Clustered at firm level (`cov_type="clustered"`, `cluster_entity=True`) via `linearmodels.PanelOLS` |
| **Sample** | Main (FF12 codes 1–7, 9–10, 12), Finance (FF12=11), Utility (FF12=8) |
| **Min calls filter** | ≥ 5 calls per firm before regression |
| **Dividend payer filter** | `is_div_payer_5yr == 1` — firm must have paid dividends in trailing 5 years |

**Hypothesis tests (one-tailed):**

| DV | H3 |
|---|---|
| `div_stability_lead` | β₁(Uncertainty) < 0 (vagueness reduces stability) |
| `payout_flexibility_lead` | β₁(Uncertainty) > 0 (vagueness increases flexibility) |

### E.1 Stage 3 Panel Build — Master Merge Sequence

The panel builder (`build_h3_payout_policy_panel.py:build_panel()`) uses a single join key (`file_name`) for all merges. Every merge is `how='left'` from the manifest anchor. Zero-row-delta is enforced after each merge.

| LinguisticEngine (6 builders) | 112,968 | 112,968 | `file_name` | left | 95.8% Manager_QA; 68.0% CEO_QA | All 6 linguistic builders share same engine singleton |
| CompustatEngine (11 builders) | 112,968 | via merge_asof | `file_name` | left | 99.8% (Size non-null: 112,692/112,968) | H3-specific vars: div_stability, payout_flexibility, earnings_volatility, fcf_growth, firm_maturity, is_div_payer_5yr |

### E.2 Lead Variable Construction

Lead variables (`div_stability_lead`, `payout_flexibility_lead`) are computed in `build_h3_payout_policy_panel.py:create_lead_variables()`:

```
1. Attach fyearq to each call via attach_fyearq() (Compustat fiscal year quarter)
2. For each (gvkey, fyearq), take the LAST call's DV value (proxy for annual)
3. Shift forward 1 fiscal year within gvkey → next-year value
4. Validate consecutive years only (gap > 1 year → NaN)
5. Merge leads back onto all calls via (gvkey, fyearq_int)
```

| Variable | Valid lead count | Match rate |
|---|---|---|
| `div_stability_lead` | **86,459** | 76.5% |
| `payout_flexibility_lead` | **105,301** | 93.2% |

**Note:** The lower coverage for `div_stability_lead` (76.5% vs 93.2% for payout_flexibility_lead) reflects that `div_stability` requires 5 years of trailing dividend data with `min_periods=3`, while `payout_flexibility` uses `min_periods=2`, making it less restrictive.

## J. Known Issues / Ambiguities

### J.1 Limited Statistical Support for H3

**Issue:** Only 1/36 total hypothesis tests are significant at p < 0.05. The majority of coefficients are in the predicted direction but not statistically significant.

**Recommendation:** Report that H3 receives limited support. The payout_flexibility result in Finance is the only significant finding.

### J.2 Dividend Payer Filter Reduces Sample by 57%

**Issue:** The `is_div_payer_5yr == 1` filter excludes 48,302 calls (42.7%) from regressions. This is by design (testing payout policy among firms that could plausibly have payout policy), but substantially reduces power.

**Recommendation:** Consider a robustness check without this filter for payout_flexibility_lead (which can exist even for non-dividend-payers via repurchases).

### J.3 High Within-R² for Payout Flexibility (0.82–0.91)

**Issue:** The payout_flexibility_lead regressions have very high within-R² (0.82–0.91), suggesting the model may be near-saturated or the dependent variable has limited variation after firm/year demeaning.

**Verification:** `payout_flexibility_lead` has std=0.386, suggesting reasonable variation. The high R² likely reflects strong persistence in payout flexibility being captured by the controls.

**Recommendation:** Investigate whether any control variable is mechanically related to payout_flexibility (e.g., firm_maturity).

### J.4 CEO Q&A Coverage Gap (32% Missing)

**Issue:** Same as H0.3 — `CEO_QA_Uncertainty_pct` is missing for 32.0% of calls. This reduces sample size for CEO-based regressions.

**Recommendation:** Ensure CEO identification logic in Stage 2 is documented. Report coverage gaps in paper appendix.

### J.5 Lead Variable Construction Uses Last Call Per Fiscal Year

**Issue:** `create_lead_variables()` takes the last call's `div_stability`/`payout_flexibility` value per fiscal year as the annual proxy. This assumes the last call's values are representative of the full fiscal year.

**Recommendation:** Document this design choice explicitly. Consider sensitivity check using mean or first call per year.

### J.6 Different DV Coverage Rates (76.5% vs 93.2%)

**Issue:** `div_stability_lead` has lower coverage (76.5%) than `payout_flexibility_lead` (93.2%). This reflects the stricter data requirements for `div_stability` (requires 5-year trailing dividend history to compute stability).

**Recommendation:** Report different N for div_stability vs payout_flexibility regressions in table notes.
