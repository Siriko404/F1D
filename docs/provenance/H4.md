### 1.3 Redesign Rationale

The old H4 tested whether leverage disciplines speech (Lev_lag -> uncertainty). The redesign flips the causal arrow: does speech uncertainty predict leverage? (uncertainty -> BookLev / DebtToCapital). This follows the H1 simultaneous-IV pattern: 4 uncertainty IVs enter every model simultaneously, with 4 DVs (BookLev_t, BookLev_lead, DebtToCapital_t, DebtToCapital_lead) crossed with 2 FE structures and 2 control sets, yielding 16 model specifications.

## 5. Merge & Sample Construction Audit

### 5.1 Panel Build Merges

All 17 variable builders produce a `(file_name, variable)` DataFrame. Each is left-merged onto the manifest by `file_name`. A zero row-delta assertion is enforced after every merge (panel builder line 237--238: raises `ValueError` if `after_len != before_len`).

### 5.2 Compustat Matching Strategy

`merge_asof` with `direction="backward"`, keyed on `gvkey`, matching call `start_date` to Compustat `datadate`. No `tolerance` parameter is set, meaning a call can match to an arbitrarily old Compustat observation if no closer one exists. This is a shared-engine design decision affecting all Compustat-derived variables.

### 5.3 Temporal Variable Construction

`create_leverage_temporal_vars()` (panel builder lines 73--104):
1. For each leverage column (BookLev, DebtToCapital), sorts by `(gvkey, fyearq_int, start_date_dt)`.
2. Deduplicates to one value per `(gvkey, fyearq_int)` keeping the **last** call (latest start_date within fiscal year).
3. Computes lag/lead by shifting within gvkey groups, requiring consecutive `fyearq_int` values (gap > 1 year yields NaN).

**Asymmetry note:** The contemporaneous DV (e.g., `BookLev`) used in cols 1--4 is the raw `merge_asof`-matched value, which varies across calls within the same firm-quarter. The lead DV (`BookLev_lead`) comes from the temporal function which deduplicates to one value per `(gvkey, fyearq_int)` before shifting. This means the contemporaneous DV has more within-group variation than the lead DV.

### 5.4 Sample Filters (Runner)

Applied per-specification in `prepare_regression_data()`:
1. Append lag_control to controls list (runner L249--252)
2. Replace inf with NaN (runner L262)
3. Drop rows where DV is NaN (runner L272)
4. Complete-case filter on all required columns (runner L276--277)
5. Minimum 5 calls per firm (runner L281--283, `MIN_CALLS_PER_FIRM = 5`)

### 5.5 Industry Exclusion

`filter_main_sample()` (runner L234--237): `panel[~panel["ff12_code"].isin([8, 11])]`. FF12 codes 1--12; excluding 8 (Utility) and 11 (Finance) leaves codes 1--7, 9--10, 12.

## 6. Variable Dictionary

### 6.1 Dependent Variables

#### 6.1.1 `BookLev` (Contemporaneous, t)

| Property | Detail |
|----------|--------|
| **Definition** | Book leverage = interest-bearing debt / total assets |
| **Formula** | `(dlcq.fillna(0) + dlttq.fillna(0)) / atq` |
| **Code location** | `_compustat_engine.py` L1041 |
| **Used in cols** | 1, 2, 3, 4 |

**Raw Compustat fields:**

| Field | Compustat Mnemonic | Description |
|-------|-------------------|-------------|
| `dlcq` | Debt in Current Liabilities -- Quarterly | Short-term interest-bearing debt (commercial paper, current portion of long-term debt). Loaded as `float64` via `pd.to_numeric(errors="coerce")` (engine L1244). |
| `dlttq` | Long-Term Debt -- Total (Quarterly) | Long-term interest-bearing debt. Same numeric coercion. |
| `atq` | Assets -- Total (Quarterly) | Total assets. Denominator. |

**Missing-value handling:**

- `dlcq.fillna(0)`: Missing short-term debt treated as zero debt (not missing leverage). No `.clip(lower=0)` is applied, so negative Compustat values (rare data errors) pass through.
- `dlttq.fillna(0)`: Same zero-fill logic for long-term debt.
- If `atq` is NaN or zero, the division produces NaN or inf. Inf values are replaced with NaN at engine L1185--1186 (`comp[col].replace([np.inf, -np.inf], np.nan)`).
- **No `.clip(lower=0)` guard:** Unlike TobinsQ (engine L1070--1071 which clips debt components to zero), BookLev uses raw `fillna(0)` without clipping. Theoretically, negative dlcq or dlttq would produce negative leverage.

**Winsorization:** Per fiscal year, 1%/99%, via `_winsorize_by_year(series, year_col, min_obs=10)` (engine L1208--1216). Years with fewer than 10 non-NaN observations are not winsorized.

**Merge to panel:** `CompustatEngine.match_to_manifest()` (engine L1282--1308) uses `pd.merge_asof(left_on="start_date", right_on="datadate", by="gvkey", direction="backward")`. No `tolerance` parameter -- unbounded staleness. `BookLevBuilder.build()` (`book_lev.py` L46--48) calls the engine and extracts `["file_name", "BookLev"]`. Left-merged onto manifest by `file_name` in panel builder (L234), zero row-delta enforced (L237--238).

#### 6.1.2 `BookLev_lead` (Forward, t+1)

| Property | Detail |
|----------|--------|
| **Definition** | BookLev shifted forward by one fiscal year |
| **Formula** | Same as BookLev, but value from `fyearq_int + 1` |
| **Code location** | Panel builder `_create_temporal_vars_for_col()` L73--104 |
| **Used in cols** | 5, 6, 7, 8 |

**Construction chain:**

1. **Deduplication:** Panel builder sorts by `(gvkey, fyearq_int, start_date_dt)` and keeps `last` call per `(gvkey, fyearq_int)` (L81--85). This means if a firm has multiple calls in the same fiscal year, only the latest call's BookLev value is used as the representative for that firm-year.
2. **Shift:** `firm_year.groupby("gvkey")[col].shift(-1)` (L100). This shifts the BookLev value forward by one position within each gvkey group (sorted by `fyearq_int`).
3. **Consecutive-year check:** `is_consecutive_next = (firm_year["next_fyearq"] - firm_year["fyearq_int"]) == 1` (L101). If the next observation for the same firm is more than one fiscal year away (gap > 1 year), the lead value is set to NaN (L102).
4. **Re-merge:** The resulting `(gvkey, fyearq_int, BookLev_lead)` lookup is merged back onto the full panel by `(gvkey, fyearq_int)` (L154), so all calls within the same firm-year receive the same lead value.

**Asymmetry note:** The contemporaneous `BookLev` (cols 1--4) is the raw `merge_asof`-matched value that varies across calls within the same firm-quarter. The lead `BookLev_lead` (cols 5--8) comes from the deduplicated temporal function, so it is constant within `(gvkey, fyearq_int)`.

#### 6.1.3 `DebtToCapital` (Contemporaneous, t)

| Property | Detail |
|----------|--------|
| **Definition** | Total interest-bearing debt / total capital |
| **Formula** | `total_debt / (seqq + total_debt)` where `total_debt = dlcq.fillna(0) + dlttq.fillna(0)` |
| **Code location** | `_compustat_engine.py` L1043--1050 |
| **Used in cols** | 9, 10, 11, 12 |

**Raw Compustat fields (in addition to dlcq, dlttq above):**

| Field | Compustat Mnemonic | Description |
|-------|-------------------|-------------|
| `seqq` | Stockholders' Equity -- Total (Quarterly) | Total shareholders' equity. Loaded as `float64` via `pd.to_numeric(errors="coerce")` (engine L1244). |

**Missing-value handling:**

- `total_debt = dlcq.fillna(0) + dlttq.fillna(0)`: Same zero-fill as BookLev.
- `total_capital = seqq + total_debt` (engine L1045): If `seqq` is NaN, `total_capital` is NaN, and `DebtToCapital` becomes NaN.
- **NaN guard:** `np.where(total_capital > 0, total_debt / total_capital, np.nan)` (engine L1046--1050). When `total_capital <= 0` (e.g., negative equity exceeding total debt), the ratio is set to NaN, not computed. This prevents nonsensical ratios for firms with deeply negative equity.
- Inf values replaced with NaN at engine L1185--1186.

**Winsorization:** Same per-fiscal-year 1%/99% as BookLev.

**Merge to panel:** Same `CompustatEngine.match_to_manifest()` pathway. `DebtToCapitalBuilder.build()` (`debt_to_capital.py` L47--49) extracts `["file_name", "DebtToCapital"]`.

#### 6.1.4 `DebtToCapital_lead` (Forward, t+1)

| Property | Detail |
|----------|--------|
| **Definition** | DebtToCapital shifted forward by one fiscal year |
| **Formula** | Same as DebtToCapital, but value from `fyearq_int + 1` |
| **Code location** | Panel builder `_create_temporal_vars_for_col()` L73--104 |
| **Used in cols** | 13, 14, 15, 16 |

**Construction chain:** Identical to `BookLev_lead` (Section 6.1.2) but applied to the `DebtToCapital` column. Same deduplication (keep last call per firm-year), same `shift(-1)`, same consecutive-year NaN guard.

**Additional NaN source:** Because `DebtToCapital` itself has an extra NaN condition (denominator <= 0), the lead version has strictly more NaN observations than `BookLev_lead`. Attrition from this additional NaN source is not separately tracked (see Known Limitation L8).

#### 6.1.5 `BookLev_lag` (Lagged DV Control, t-1)

### 6.2 Key Independent Variables (4, all simultaneous)

All four IVs enter every model simultaneously (runner L327: `exog = KEY_IVS + controls`). Note: by the time `exog` is assembled, `controls` already includes the lagged DV control (appended at runner L249--252 in `prepare_regression_data()` and L312--315 in `run_regression()`).

#### 6.2.1 Full Construction Chain for Uncertainty Percentage Variables

The four uncertainty IVs share an identical construction pipeline, differing only in the speaker filter and transcript section filter:

| Variable | Speaker Filter | Section Filter |
|----------|---------------|----------------|
| `CEO_QA_Uncertainty_pct` | `is_ceo == True` | `context == "qa"` |
| `CEO_Pres_Uncertainty_pct` | `is_ceo == True` | `context == "pres"` |
| `Manager_QA_Uncertainty_pct` | `is_manager == True` | `context == "qa"` |
| `Manager_Pres_Uncertainty_pct` | `is_manager == True` | `context == "pres"` |

**Step 1: Dictionary** -- The Loughran-McDonald (LM) Master Dictionary (`inputs/Loughran-McDonald_MasterDictionary_1993-2024.csv`) is loaded in Step 2.1 (`tokenize_transcripts.py` L172--173). The dictionary defines word-level sentiment/category flags (Positive, Negative, Uncertainty, Litigious, Constraining, Strong Modal, Weak Modal). Each word in the dictionary has a binary flag for each category.

**Step 2: Tokenization** -- Each earnings call transcript is tokenized using `sklearn.feature_extraction.text.CountVectorizer` (`tokenize_transcripts.py` L56). The vocabulary is restricted to the LM dictionary terms. Output: per-turn (speaker-turn level) counts of each category's words plus `total_tokens` (total word count for that turn). Saved as `linguistic_counts_{year}.parquet`.

**Step 3: Speaker flagging** -- `build_linguistic_variables.py` flags each speaker turn:
- `is_ceo`: Exact title match ("Chief Executive Officer", "CEO") OR fuzzy last-name match against the manifest's CEO name field (L440--462).
- `is_manager`: Matched against a curated managerial-roles list from `inputs/Manager_roles/managerial_roles_extracted.txt` (L411--437).
- `is_analyst`: Role/employer heuristics (L392--410).

**Step 4: Aggregation** -- `aggregate_weighted()` (`build_linguistic_variables.py` L472--502):
1. Subset to turns matching the speaker filter AND section filter.
2. `groupby("file_name")` and sum both `Uncertainty_count` and `total_tokens` across all matching turns within the call.
3. Compute percentage: `pct = (sum(Uncertainty_count) / sum(total_tokens)) * 100.0`. If `total_tokens == 0` for a call, replaced with NaN (`total_tokens.replace(0, np.nan)` at L490).
4. Output column named `{Speaker}_{Section}_Uncertainty_pct`.

**Step 5: Winsorization** -- `LinguisticEngine.get_data()` (`_linguistic_engine.py` L255--258) applies per-year winsorization via `winsorize_by_year()` with `lower=0.0, upper=0.99, min_obs=10`. This is upper-only winsorization (0th percentile lower bound = no lower clipping). The year grouping column comes from the parquet filename year.

**Step 6: Builder** -- Each builder (e.g., `CEOQAUncertaintyBuilder` in `ceo_qa_uncertainty.py`) calls `LinguisticEngine.get_data()` (singleton, cached after first load), extracts `["file_name", "{column}"]`, and returns a `VariableResult`. Left-merged onto manifest by `file_name` in the panel builder.

**Missing-value semantics:** NaN in a textual variable means either (a) no matching speaker turns exist in that section of the call (CEO had no QA turns, or no presentation turns), or (b) the matching turns had zero total tokens. Zero means the speaker spoke but used no LM Uncertainty words.

#### 6.3.2 `TobinsQ`

| Property | Detail |
|----------|--------|
| **Definition** | Market value of assets / book value of assets |
| **Formula** | `(cshoq * prccq + debt_book) / atq` |
| **Code location** | `_compustat_engine.py` L1069--1079 |
| **Raw fields** | `cshoq` (Common Shares Outstanding -- Quarterly), `prccq` (Price Close -- Quarterly), `dlcq`, `dlttq`, `atq` |

**Detailed construction:**
1. `mktcap = cshoq * prccq` -- market capitalization.
2. `debt_c = dlcq.clip(lower=0).fillna(0)` -- short-term debt, clipped to non-negative, missing treated as zero (L1070).
3. `debt_t = dlttq.clip(lower=0).fillna(0)` -- long-term debt, same treatment (L1071).
4. `debt_book = np.where(dlcq.isna() & dlttq.isna(), np.nan, debt_c + debt_t)` -- NaN when BOTH debt fields are missing (L1072--1074).
5. `TobinsQ = np.where(atq.notna() & (atq > 0) & mktcap.notna(), (mktcap + debt_book) / atq, np.nan)` (L1075--1079).

**Key difference from BookLev:** TobinsQ uses `.clip(lower=0)` on debt components (L1070--1071, preventing negative values), while BookLev does not.

| **Winsorization** | Per fiscal year 1%/99% |
| **Merge** | Via `CompustatEngine.match_to_manifest()` |

#### 6.3.3 `ROA`

| Property | Detail |
|----------|--------|
| **Definition** | Return on assets (annualized) |
| **Formula** | `iby_annual / avg_assets` |
| **Code location** | `_compustat_engine.py` L1050--1060 |
| **Raw fields** | `iby` (Income Before Extraordinary Items -- Annual YTD), `atq` (Assets -- Total, Quarterly), `fyearq`, `fqtr` |

**Detailed construction:**
1. `atq_annual = _compute_annual_q4_variable(comp, "atq", ...)` -- extracts Q4-only `atq` per `(gvkey, fyearq)`, keeping last row if duplicates, then joins back to all quarters by `(gvkey, fyearq)` (engine L234--274).
2. `atq_annual_lag1 = _compute_annual_q4_variable_lag(comp, "atq", ...)` -- same Q4 extraction but shifts `fyearq += 1` before joining, so each row gets the prior fiscal year's Q4 total assets (engine L277--304).
3. `avg_assets = (atq_annual + atq_annual_lag1) / 2` -- average of current and prior year assets (L1053--1056).
4. `iby_annual = _compute_annual_q4_variable(comp, "iby", ...)` -- Q4-only income before extraordinary items, full fiscal year (L1057).
5. `ROA = np.where(avg_assets > 0, iby_annual / avg_assets, np.nan)` (L1058--1060).

| **Missing handling** | NaN if `avg_assets <= 0`, or if `iby` or `atq` is missing at Q4. |
| **Winsorization** | Per fiscal year 1%/99% |

#### 6.3.4 `CapexAt`

| Property | Detail |
|----------|--------|
| **Definition** | Capital expenditure intensity |
| **Formula** | `capxy_annual(Q4) / atq_annual_lag1` |
| **Code location** | `_compustat_engine.py` L1081--1087 |
| **Raw fields** | `capxy` (Capital Expenditures -- YTD cumulative), `atq`, `fyearq`, `fqtr` |

**Detailed construction:**
1. `capxy` is YTD cumulative within a fiscal year (Q1 = Q1 capex only, Q4 = full-year capex). ## 11. Identification & Inference Assessment

### 11.1 Endogeneity

Speech uncertainty and leverage are likely co-determined. Firms with deteriorating financial conditions may simultaneously increase leverage and exhibit more uncertain language. No instrumental variable strategy or natural experiment is employed. The two-tailed test acknowledges directional ambiguity but does not address causality.

### 11.2 Moulton Problem

Within a firm-quarter, multiple earnings calls (if they exist) share the same Compustat-matched BookLev value because `merge_asof` matches all calls in the same quarter to the same Compustat reporting date. Firm-clustered SEs partially address this by allowing arbitrary within-firm correlation. However, the effective number of independent DV observations is the number of firm-quarters, not the number of calls. If most firms have only one call per quarter this is largely moot; if multiple calls per quarter are common, the effective sample size is overstated relative to the call count.

**Precision:** The Moulton concern applies within a Compustat quarter (same `datadate` match), not within a fiscal year. Different quarters within the same fiscal year will generally match to different Compustat reporting dates, producing different BookLev values.

### 11.3 Multicollinearity Among IVs

Four uncertainty measures enter simultaneously. CEO_QA and Manager_QA (and their Pres counterparts) likely share substantial measurement overlap. No VIF or condition-number diagnostics are computed or reported. High collinearity among IVs would inflate standard errors, making individual IV coefficients unreliable. This is a concern for interpreting individual IV significance, though the joint F-test remains valid.

### 11.4 Simultaneous IV Interpretation

With 4 correlated uncertainty measures entering simultaneously, individual coefficients represent partial effects conditional on the other three. A null coefficient on CEO_QA does not mean CEO Q&A uncertainty is irrelevant, only that it adds no incremental explanatory power beyond the other measures. The thesis text must distinguish partial from marginal effects.

### 11.5 Leverage Persistence and Lagged DV Control

Leverage is highly autocorrelated. Every model includes a lagged DV control (`BookLev_lag` for cols 1--8, `DebtToCapital_lag` for cols 9--16) via the `lag_control` field in `MODEL_SPECS` (runner L111--126). This makes each specification a **partial-adjustment (dynamic panel) model**. The lagged DV is likely to absorb most of the cross-sectional and within-firm variation in leverage, potentially attenuating the uncertainty IV coefficients toward zero. If significant results survive the lagged DV control, they are more credible than they would be in a static specification.

**Interpretation:** The IV coefficients represent the *short-run* impact of uncertainty on leverage conditional on the prior level. The long-run multiplier is `beta_IV / (1 - rho)`, where `rho` is the lagged DV coefficient. The thesis text should report both short-run and long-run effects.

**Post-treatment bias concern:** `BookLev_lag` is measured at t-1 but could itself be influenced by uncertainty at t-1 (if the hypothesis is correct). Conditioning on it could attenuate the contemporaneous effect. This is a standard concern in dynamic panel models.

### 11.8 Nickell (1981) Bias

Even-numbered columns use firm FE alongside a lagged DV control. In a fixed-effects model with a lagged dependent variable, the within-group transformation creates a mechanical correlation between the transformed lagged DV and the transformed error term, producing downward bias in the lagged DV coefficient and potential bias in other regressors. The bias is of order O(1/T), where T is the average number of time periods per firm. With T potentially large (many fiscal years per firm), the bias may be small, but it should be explicitly acknowledged. Odd-numbered columns (industry FE) are less affected because industry groups are large, but the concern still applies if industry FE effectively acts as a group demeaning.

No Anderson-Hsiao or Arellano-Bond GMM estimator is employed as a robustness check.

### 11.6 Multiple Testing

4 IVs x 16 specifications = 64 individual IV-level hypothesis tests. No familywise error rate correction (Bonferroni, Holm, etc.) is applied. Significant individual coefficients should be interpreted cautiously.

### 11.7 Clustering

Firm-level clustering is employed. Double-clustering by firm and fiscal year (standard in long-panel corporate finance) is not implemented as a robustness check.

| L4 | MODERATE | **Asymmetric DV construction.** Contemporaneous DV is the raw merge_asof-matched value (varies across calls within a quarter). Lead DV is deduplicated to one value per (gvkey, fyearq_int) then shifted. This creates a systematic difference in within-group variation between contemporaneous and lead specifications. |
| L5 | MODERATE | **No merge_asof tolerance.** `match_to_manifest()` has no `tolerance` parameter. A call can match to an arbitrarily stale Compustat observation. |

| L10 | LOW | **No double-clustering robustness.** Firm + year double-clustering is not offered as a sensitivity check. |

A reader examining the LaTeX table would not know the model includes a lagged DV. |
| L12 | **MAJOR** | **Nickell (1981) bias with firm FE + lagged DV.** Even-numbered columns use firm FE alongside a lagged DV control. With finite T, the lagged DV coefficient is biased downward and other coefficients may inherit bias. Severity depends on average T per firm. No Anderson-Hsiao or Arellano-Bond GMM estimator is employed as a robustness check. See Section 11.8. |

Since the lag variable is constructed from the same temporal-var pipeline as the lead, it inherits the same deduplication (keep-last-call) and consecutive-year requirements. Missing lag values create additional complete-case attrition not tracked in the attrition table. |

## 14. Design Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| **4 simultaneous IVs** | Tests incremental contribution of each uncertainty channel (CEO vs Manager, QA vs Presentation) within a single model, rather than running separate univariate regressions that ignore collinearity. |
| **Two DVs (BookLev, DebtToCapital)** | BookLev captures interest-bearing debt relative to total assets; DebtToCapital captures debt relative to total capital (equity + debt). Different theoretical implications: BookLev relates to asset coverage, DebtToCapital to capital structure composition. |
| **Contemporaneous + lead DVs** | Contemporaneous (t) tests association; lead (t+1) provides a forward-looking test with weaker reverse-causality concerns. |
| **Two-tailed test** | No prior directional prediction for how speech uncertainty relates to leverage. Precautionary tradeoffs (more leverage as buffer) and risk-signaling (less leverage under uncertainty) both have theoretical support. |
| **Industry FE via `other_effects`** | Absorbs industry FE without creating dummy columns in the exog matrix. More memory-efficient than C() formula dummies. |
| **Firm FE via `EntityEffects`** | Absorbs firm-specific time-invariant heterogeneity. Combined with fiscal year FE, this is a two-way FE design. |
| **Fiscal year (`fyearq_int`) as time index** | Fixes calendar-year bug from old H4. Compustat fiscal year aligns leverage measures with the correct accounting period. |
| **Firm-clustered SEs** | Accounts for within-firm serial correlation in leverage. Standard in corporate finance panel regressions. |
| **Minimum 5 calls per firm** | Ensures sufficient within-firm variation for FE estimation. Prevents estimation artifacts from firms with very few observations. |
| **Main sample only (exclude FF12=8, 11)** | Standard practice: financial firms (FF12=11) and utilities (FF12=8) have regulated capital structures that confound leverage regressions. |
| **Missing debt treated as zero** | Per specification: `fillna(0)` for dlcq/dlttq assumes unreported debt is zero rather than missing. Common in Compustat-based leverage studies. |
| **8-column LaTeX table (BookLev only)** | This is a **code defect**, not an intentional design decision. The `_save_latex_table()` function was written for an earlier 8-model version and was not updated when DebtToCapital models were added. The full 16-model results are available in `model_diagnostics.csv` and individual `.txt` files. |
| **Lagged DV control (partial-adjustment model)** | Including `BookLev_lag` (cols 1--8) or `DebtToCapital_lag` (cols 9--16) as a control implements a partial-adjustment specification that isolates the incremental effect of uncertainty beyond leverage persistence. This is a deliberate modeling choice: if uncertainty coefficients are significant even after controlling for the prior leverage level, the result is more credible. The trade-off is Nickell (1981) bias in firm-FE models (see L12) and potential post-treatment bias if lagged leverage is itself influenced by lagged uncertainty. |
