### E2. Key difference from annual H12

H12Q operates at **call level** (no firm-year aggregation). This is a fundamental structural difference from annual H12:

- **No aggregation step:** IVs are call-level uncertainty measures, not firm-year averages. This preserves within-firm-within-year variation across quarterly calls.
- **No `Avg_` prefix:** IVs retain their original call-level column names (`CEO_QA_Uncertainty_pct`, not `Avg_CEO_QA_Uncertainty_pct`).
- **Multiple calls per firm-quarter:** The panel index is (gvkey, fiscal_qtr_id), but the unit of observation is individual calls. When a firm has multiple calls in the same fiscal quarter, they share the same panel index -- `PanelOLS` handles this by treating them as repeated observations within the entity-time cell (same pattern as H1, H5, H13).

## F. Variable Dictionary

### F1. Dependent variable: `PayoutRatio_q`

| Property | Detail |
|----------|--------|
| **Formula** | `(dvpspq.fillna(0) * cshoq) / ibq` |
| **Interpretation** | Quarterly dividends as a fraction of quarterly income before extraordinary items |
| **Guard** | `ibq > 0` required; `ibq <= 0` produces NaN (explicit negative earnings filter) |
| **Missing-dvpspq treatment** | `dvpspq.fillna(0)` -- firms with positive earnings but no reported dividends are treated as zero-payout |
| **Code** | Engine L1116-1125 |

**Complete construction chain:**

1. **Raw Compustat fields:** `dvpspq` (dividends per share by pay date, true quarterly field -- NOT YTD cumulative), `cshoq` (common shares outstanding, quarterly), `ibq` (income before extraordinary items, quarterly). All loaded from `inputs/comp_na_daily_all/comp_na_daily_all.parquet` via `CompustatEngine.get_data()` (Engine L1248-1252). Numeric columns coerced to `float64` (Engine L1256-1258).
2. **Deduplication:** `(gvkey, datadate)` duplicates dropped, keeping last row (most recent restatement). Engine L1263-1265.
3. **Computation:** `quarterly_div = comp["dvpspq"].fillna(0) * comp["cshoq"]` (Engine L1120). Then `PayoutRatio_q = np.where(comp["ibq"] > 0, quarterly_div / comp["ibq"], np.nan)` (Engine L1121-1125).
4. **Inf cleanup:** `PayoutRatio_q` is in the `ratio_cols` list (Engine L1177). All `inf` / `-inf` values replaced with NaN (Engine L1185-1186).
5. **Per-year winsorization:** `PayoutRatio_q` is in `COMPUSTAT_COLS` (Engine L124) and not in `skip_winsorize` (Engine L1199-1206). Winsorized at 1%/99% within each `fyearq` group via `_winsorize_by_year()` (Engine L1213-1215). Years with fewer than 10 valid observations are skipped (Engine L464).
6. **Manifest merge:** `CompustatEngine.match_to_manifest()` performs `merge_asof` backward on `start_date` within `gvkey` (Engine L1301-1308). Each call gets the most recent Compustat quarter's `PayoutRatio_q`.
7. **Panel merge:** Builder L130-146 merges `PayoutRatio_q` into the call-level panel on `file_name` (left join, row-count assertion).

### F2. Lead DVs

#### `PayoutRatio_q_lead_qtr` (next fiscal quarter)

| Property | Detail |
|----------|--------|
| **Method** | Shift(-1) within gvkey on sorted firm-quarter data |
| **Consecutive check** | Next fiscal_qtr_id must equal expected_next (fqtr < 4: curr + 1; fqtr == 4: next_year * 10 + 1) |
| **Code** | Builder L248-264 |

**Construction chain:**

1. Attach `fqtr` and `fyearq` to the panel via merge_asof (Builder L176-219).
2. Create `fiscal_qtr_id = fyearq_int * 10 + fqtr_int` (Builder L226).
3. Build firm-quarter lookup: for each (gvkey, fiscal_qtr_id), take PayoutRatio_q from the call with the latest start_date (Builder L239-243).
4. Within the lookup, `shift(-1)` within gvkey to get next quarter's PayoutRatio_q (Builder L248-249).
5. Validate consecutive quarters: expected_next computed via `np.where(fqtr_int < 4, curr_fq + 1, (fyearq_int + 1) * 10 + 1)` (Builder L256-260). Non-consecutive pairs produce NaN (Builder L262-264).
6. Merge back to call level on (gvkey, fiscal_qtr_id) (Builder L283-288).

## K. Referee Assessment

### K1. Identification

**K1a. Endogeneity of uncertainty language.**
The core identification concern applies identically to H12Q as to H12: managerial uncertainty language is not exogenous to payout decisions. Firms contemplating dividend cuts may simultaneously signal uncertainty. This suite has no instrumental variable, no regression discontinuity, no difference-in-differences design. The lead specifications (cols 5-8) provide some temporal separation, but with the lagged DV as a control, the interpretation is limited to "does uncertainty predict changes in future payout beyond what current payout predicts." This is a predictive association, not a causal effect.

**K1b. Call-level variation vs. firm-year variation.**
Unlike annual H12, this suite uses call-level observations. This means the IV varies across calls within the same firm-quarter. With firm FE, identification comes from within-firm variation in uncertainty language. However, the DV (PayoutRatio_q) is a *firm-quarter* variable attached via merge_asof -- it does NOT vary across calls within the same firm-quarter (unless calls map to different Compustat quarter-ends, which is rare). This creates a structural mismatch: within a firm-quarter, the DV is constant while IVs vary. The firm FE models are effectively identifying off between-quarter variation in both DV and IVs, with call-level noise in IVs. This inflates the apparent sample size without adding genuine variation in the DV.

**K1c. Multiple calls per firm-quarter.**
The panel index is (gvkey, fiscal_qtr_id). When multiple calls occur in the same firm-quarter, PanelOLS treats them as repeated observations within the same entity-time cell. This is not a standard panel structure. The implications for estimation consistency depend on whether `linearmodels` handles this correctly -- for PanelOLS with `EntityEffects`, within-estimation demeans within entity, which may produce unexpected results when multiple observations share the same time index.

### K2. Inference

**K2a. Firm-clustered SEs are appropriate** given the call-level panel with repeated observations per firm. Clustering at the firm level accounts for serial correlation within firms across quarters and across calls within quarters.

**K2b. One-tailed test implementation is correct** (Runner L304-306): divides two-tailed p by 2 when beta has the predicted sign, else 1 - p/2. This follows the standard one-tailed conversion.

**K2c. Multiple testing.** 8 specifications x 4 IVs = 32 hypothesis tests. No multiple-testing correction is applied. With independent tests at 5%, the probability of at least one false positive is 1 - (1-0.05)^48 = 91.5%. Even with high correlation among tests, the false-positive risk is substantial.

**K2d. Inflated N from call-level estimation.** Standard errors are clustered at the firm level, which should in principle account for within-firm correlation. However, the effective degrees of freedom may be overstated because the DV does not actually vary across calls within the same firm-quarter. The number of independent DV observations is the number of firm-quarters, not the number of calls.

### K3. Variable Construction

**K3a. dvpspq is per-share; ibq is total income.**
PayoutRatio_q = `(dvpspq * cshoq) / ibq`. The numerator converts per-share dividends to total dividends by multiplying by shares outstanding. However, `dvpspq` is dividends per share *by pay date*, not by ex-date or declaration date. The timing of dvpspq may not align perfectly with the fiscal quarter's ibq. Compustat documentation notes that dvpspq is the amount per share paid during the quarter based on the pay date. This is an industry-standard approach but introduces potential quarter-boundary timing noise.

**K3b. ibq vs. iby for negative earnings filter.**
The quarterly filter uses `ibq > 0` (single-quarter income). The annual H12 uses `iby > 0` (annual income). A firm could have positive annual income but negative quarterly income in some quarters, or vice versa. This is a conscious design choice for quarterly analysis but means the H12Q sample is not a strict subset of H12.

**K3c. fiscal_qtr_id encoding.**
`fiscal_qtr_id = fyearq_int * 10 + fqtr_int` (Builder L226). This encoding works correctly for fqtr in {1,2,3,4}, producing e.g. 20101, 20102, 20103, 20104. However, if fyearq or fqtr contains unexpected values (e.g., fqtr = 0 or fqtr > 4), the encoding could produce collisions. No explicit validation of fqtr range is present in the builder.

**K3d. Lead construction uses firm-quarter lookup (latest call).**
When multiple calls exist in the same firm-quarter, the lead lookup uses PayoutRatio_q from the call with the latest `start_date` (Builder L239). This is sensible but means the lead DV is tied to one specific Compustat match, not the average. Since PayoutRatio_q is constant within a firm-quarter (same Compustat quarter), this distinction is immaterial in practice.

### K4. Sample / Filtering

**K4a. MIN_CALLS_PER_FIRM = 5.**
This threshold (Runner L89) eliminates firms with very few calls. This is a common panel requirement but could introduce survivorship bias -- firms that go private, are acquired, or delist early are more likely to be excluded. The threshold is applied after DV filtering, so a firm could have 10 total calls but only 3 with ibq > 0, and be excluded.

**K4b. Complete-case filtering is per-specification.**
Different DVs and control sets produce different samples. The attrition table records only the Col 1 trajectory (Runner L571-577). No cross-column sample-consistency check is reported.

**K4c. Negative earnings exclusion is substantial.**
The filter `ibq > 0` drops all firm-quarters with zero or negative quarterly income. For an average Compustat panel, this could exclude 20-30% of observations, creating a selection on profitability. The remaining sample is non-randomly selected -- only profitable firm-quarters are analyzed. This could bias the payout-uncertainty relationship if uncertainty language is differentially associated with earnings sign.

### K5. Robustness

**K5a. No Tobit or fractional response model.**
With ~57% zeros and a DV bounded below at 0, OLS is suboptimal. A Tobit model (censored at 0) or a fractional probit/logit (Papke & Wooldridge 1996) would be more appropriate for a bounded, zero-inflated DV. This is acknowledged as a limitation (Runner L38) but not addressed with robustness checks.

**K5b. No two-part model.**
A Heckman-style selection model or two-part model (extensive margin: pay dividends or not; intensive margin: how much) would address the zero mass point more rigorously.

**K5c. No placebo test.**
No placebo or falsification test is implemented (e.g., using analyst uncertainty as a placebo IV, or testing against a DV that should not respond to managerial uncertainty).

**K5d. No subsample analysis.**
No tests for heterogeneity across dividend payers vs. non-payers, firm size quartiles, or time periods.

### K6. Interpretation / Framing

**K6a. The 57% zero rate is a first-order concern.**
At the quarterly level, most firms either do not pay dividends at all or pay only in certain quarters (typically one quarter per year for US firms). This means the identifying variation is overwhelmingly driven by the extensive margin (whether any dividend is paid this quarter), not the intensive margin (how much). An OLS coefficient reflects a mix of both margins, making economic interpretation unclear.

**K6b. Call-level vs. firm-quarter ambiguity.**
The doc describes this as a "call-level" panel, but the DV is a firm-quarter variable. The economic interpretation should be framed as "does the uncertainty language in a specific call predict the corresponding quarter's payout ratio" -- but since the DV is matched via merge_asof to the most recent prior Compustat quarter, the "corresponding quarter" is actually the most recent *past* quarter, not the quarter the call discusses. This creates a look-ahead concern: the payout decision may have already occurred before the call.

**K6c. Contemporaneous DV timing.**
The merge_asof backward match means `PayoutRatio_q` attached to a call reflects the most recent Compustat quarter *before* the call date. For a typical Q1 earnings call held in April, the matched Compustat quarter is Q1 (fiscal quarter ending March). The dividend in Q1 was already declared and paid before the April call. Thus the contemporaneous model (cols 1-4) tests whether uncertainty language is *associated with* recent payout, not whether it *predicts* it. The lead models (cols 5-8) provide genuinely forward-looking tests.

### K7. Academic Integrity

**K7a. Transparent documentation of limitations.**
The runner docstring (L37-38) explicitly documents the ~57% zero mass point and acknowledges OLS as a limitation. This is good practice.

**K7b. No data leakage detected.**
The lead construction (Builder L248-277) correctly uses fiscal quarter shifting and consecutive-quarter checks. The merge_asof backward direction ensures no future Compustat data leaks into contemporaneous variables.

**K7c. Relationship to existing H12.**
This suite complements the annual H12 by testing at quarterly frequency. The two suites should yield consistent conclusions if the relationship is genuine. If results diverge, this needs explicit discussion.
