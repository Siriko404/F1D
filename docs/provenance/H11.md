|-------|-------|
| Entity FE | `gvkey` (firm) |

**REFEREE CONCERN (Year FE):** Time FE is at the calendar YEAR level, not quarter. This is coarser than may be appropriate given that PRiskQ varies at the quarterly level. Year FE cannot absorb within-year time shocks that might confound the quarterly PRiskQ-uncertainty relation.

**REFEREE CONCERN (Non-unique index + clustering):** The (gvkey, year) index is non-unique (96.6% of cells have multiple calls, mean 3.9, max 38). Firm-level clustering handles within-firm correlation across years, but does not explicitly address intra-year dependence from multiple calls by the same firm in the same year. While firm-level clustering is conservative enough to likely absorb this, the additional source of within-cluster correlation is undocumented and untested. Testing (gvkey, year-quarter) clustering would verify robustness.

## E. Merge & Sample Construction Audit (Verified)

### E1) Panel builder merges (all 3 variants identical pattern)

Each panel builder starts with the manifest (112,968 rows) and left-joins each variable builder output on `file_name`.

| Merge step | Left | Right | Join type | Join key | Row delta | Notes |
|------------|------|-------|-----------|----------|-----------|-------|
| manifest (base) | -- | ManifestFieldsBuilder | -- | -- | -- | Starting frame: 112,968 rows |
| + linguistic DVs (6) | manifest | each builder output | LEFT | file_name | 0 (enforced) | Zero-row-delta check raises ValueError on fan-out |
| + PRiskQ variants | manifest | PRiskQBuilder (or Lag/Lead) | LEFT | file_name | 0 (enforced) | Match rate: 97.6% (base), 93.1%/91.7% (lag1/lag2), 95.1%/94.8% (lead1/lead2) |
| + linguistic controls (2) | manifest | Analyst/Negative builders | LEFT | file_name | 0 (enforced) | |
| + financial controls (8) | manifest | Size/Lev/ROA/... builders | LEFT | file_name | 0 (enforced) | |

All merges enforced via: `if after_len != before_len: raise ValueError(...)`.

Conflicting columns (other than `file_name`) are dropped before merge: `data = data.drop(columns=conflicting)`.

### E2) PRiskQ merge detail (the novel step for this suite)

**Base (PRiskQBuilder):**
1. Load manifest with (file_name, gvkey, start_date)
2. Compute `cal_q` from `start_date` (e.g., "2010q2")
3. Load PRisk quarterly CSV, filter to requested years, deduplicate (keep max PRisk per gvkey/cal_q)
4. Apply per-year 1%/99% winsorization on PRisk
5. Merge manifest with PRisk on (gvkey, cal_q) -- LEFT join
6. Realign to manifest by file_name, drop_duplicates
7. Result: 110,256 matched / 112,968 total (97.6%)

**Lag (PRiskQLagBuilder / PRiskQLag2Builder):**
- Same as base but computes `cal_q_lag = _get_prev_quarter(cal_q)` (or `_get_prev2_quarter`)
- Merges on `(gvkey, cal_q_lag)` against PRisk's `(gvkey, cal_q)`
- Loads PRisk data for year_start-1 (or year_start-2) to cover lag boundary
- Match rates: 93.1% (lag1), 91.7% (lag2)

**Lead (PRiskQLeadBuilder / PRiskQLead2Builder):**
- Same as base but computes `cal_q_lead = _get_next_quarter(cal_q)` (or `_get_next2_quarter`)
- Merges on `(gvkey, cal_q_lead)` against PRisk's `(gvkey, cal_q)`
- Loads PRisk data for year_end+1 (or year_end+2) to cover lead boundary
- Match rates: 95.1% (lead1), 94.8% (lead2)

## G. Outliers, Missing Data, Scaling, and Model-Specific Data Rules

### Winsorization

| Variable | Method | Thresholds | When applied | Code reference |
|----------|--------|-----------|--------------|----------------|
| PRiskQ (all variants) | `winsorize_by_year()` | 1%/99% per year | Inside each PRiskQ builder, BEFORE merge to manifest | `prisk_q.py` line 141, `prisk_q_lag.py` line 165, etc. |
| Financial controls | Via `VariableBuilder._finalize_data()` | 1%/99% by year (default) | Inside each builder's build() | `base.py::_finalize_data()` |
| Linguistic variables | Via `VariableBuilder._finalize_data()` | 1%/99% by year (default) | Inside each builder's build() | `base.py::_finalize_data()` |

### Missing data policy

- **Listwise deletion** at regression time: `panel.replace([np.inf, -np.inf], np.nan).dropna(subset=required)` in `prepare_regression_data()`.
- No imputation. No forward-fill. No interpolation.
- Infinity values converted to NaN before drop.
- PRiskQ match rates vary by variant (see Section E2), creating differential missingness.

### Deduplication

- PRisk quarterly data: deduplicated on (gvkey, cal_q) keeping the MAX PRisk value. This is an aggressive choice -- if a firm-quarter has multiple PRisk entries, the highest is kept. This could upward-bias the PRisk measure.
- After PRiskQ merge to manifest: `drop_duplicates(subset=["file_name"])` ensures one-to-one.

### Scaling

- `Size` is log-transformed (log(atq)).
- No other explicit scaling or standardization in the panel builder.
- Table notes in LaTeX claim "all continuous controls are standardized" -- this is NOT implemented in the code. **VERIFIED MISMATCH** between table notes and implementation.

### Denominator protections

- Financial variables use NaN for invalid ratios (via VariableBuilder convention). Not separately verified for each builder.

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line referee verdict

This suite implements a well-structured panel FE examination of whether Hassan et al. (2019) firm-level political risk exposure is associated with managerial speech uncertainty in earnings calls. The implementation is competent for a descriptive association study. However, the identification strategy has a serious flaw: the lead (placebo) tests -- explicitly designed to rule out reverse causality -- produce significant positive coefficients in all 12 Main-sample specifications and 21/28 overall. Lead betas are approximately one-third of base betas (e.g., Manager_QA: base 0.000135 vs lead1 0.000043), and PRiskQ within-firm autocorrelation is moderate (rho=0.36). This attenuation pattern is more consistent with persistence leakage than with reverse causality of equal magnitude, but it still means the lead tests FAIL to falsify the persistence-confounding channel. Two Utility lead2 specs have negative betas and 5 are non-significant, indicating the failure is concentrated in the Main sample. The suite currently supports descriptive association only, not causal interpretation, unless PRiskQ persistence is explicitly addressed (e.g., via first-differencing or quarter FE). Note also that the panel index (gvkey, year) is non-unique (96.6% of cells have multiple calls, max 38), making this a call-level analysis with firm + year FE, not a firm-year panel.

**Overall status: SALVAGEABLE WITH MAJOR REVISIONS**

- Supports: **descriptive association only**
- Does NOT support: causal interpretation, predictive relation (no out-of-sample validation)
- The write-up would be **overselling** if it uses causal language ("increases", "leads to", "affects") based on these results

### K6) Interpretation and claims discipline

- **Causal verbs NOT justified**: The lead test failure in the Main sample (12/12 significant, 21/28 overall) means causal language is unsupported as implemented. If the thesis uses causal verbs ("increases", "leads to", "causes"), the lead failure directly contradicts such claims. The ~3x attenuation of lead betas and moderate autocorrelation (rho=0.36) suggest the issue is addressable via persistence controls, but until implemented, only associational language is defensible.
- **Mechanism claims**: Not tested. The suite examines association only.
- **Economic magnitudes**: Beta coefficients are very small (e.g., 0.000135 for Manager_QA). A one-SD increase in PRiskQ (~161 units) increases Manager_QA_Uncertainty_pct by approximately 0.02 percentage points. Without standardized coefficients or one-SD effect-size interpretations, the suite cannot answer "how large is this effect?" This is a medium-severity gap: even statistically significant results may lack practical importance, and a committee cannot evaluate this from the current outputs.
- **Specification-mining risk**: HIGH. With 90 regressions, 6 DVs, 3 samples, and multiple timing variants, the probability of finding significant results by chance is substantial. No multiple-testing correction is applied.
- **Table notes accuracy**: Table notes claim "standardized" controls, which is false (see G/K3).

## N. Final Readiness Statement

**Can this suite be reproduced end-to-end from code alone?**
PARTIAL. The code is functional and deterministic, but reproduction depends on: (1) Stage 1-2 outputs existing at `outputs/1.4_AssembleManifest/` and `outputs/2_Textual_Analysis/`, (2) H0.3 clarity residuals existing, (3) raw `inputs/FirmLevelRisk/firmquarter_2022q1.csv` being present, (4) `get_latest_output_dir()` resolving correctly (fragile: 3 stale timestamped dirs exist for base econometric), (5) the linearmodels version should be documented, as PanelOLS behavior with non-unique multi-index may be version-dependent.

**Can a committee/referee understand exactly how the variables are built?**
YES, with caveats. The PRiskQ builders are well-documented. The docstring DV mismatch (cosmetic) and false standardization claim in table notes would cause confusion. The panel structure is misleadingly described if characterized as "firm-year" -- it is a call-level panel with a non-unique (gvkey, year) index (96.6% of cells have multiple calls, max 38).

**Are the implemented results trustworthy for descriptive inference?**
YES. The association between PRiskQ and speech uncertainty is well-documented across multiple DVs, samples, and timing variants. The implementation is sound for establishing correlation. However, economic significance is unclear: betas are ~0.0001, meaning a 1-SD PRiskQ increase (~161 units) produces ~0.02 ppt change in Manager_QA_Uncertainty_pct. Standardized effect sizes should be reported.

**Are they trustworthy for causal inference?**
NO, as currently implemented. The lead placebo test fails in all 12 Main-sample specifications and 21/28 overall. However, the failure pattern provides diagnostic information: lead betas are approximately one-third of base betas (e.g., Manager_QA: 0.000043 vs 0.000135), and PRiskQ within-firm autocorrelation is moderate (rho=0.36, not "high"). This attenuation pattern is consistent with persistence leakage rather than equal-magnitude reverse causality. The lead failure is therefore ADDRESSABLE -- specifically via: (a) first-differenced PRiskQ to remove persistence, (b) quarter FE to absorb within-year confounders, or (c) including all timing variants jointly. Until such remediation is implemented, causal claims remain unsupported. Note also that 7/28 lead specs are NOT significant (5 positive non-significant, 2 negative), concentrated in Finance and Utility subsamples -- the failure is primarily a Main-sample phenomenon.

**What is the single biggest threat to credibility?**
The failed lead (placebo) test in the Main sample. All 12 Main-sample lead coefficients are positive and significant (21/28 overall), contradicting the suite's causal identification strategy. However, this threat is mitigable: lead betas are ~1/3 of base betas, and PRiskQ autocorrelation is moderate (rho=0.36), indicating persistence leakage rather than outright reverse causality.

**What is the single highest-value next fix?**
Implement quarter-level time FE and first-differenced PRiskQ specifications. PRiskQ within-firm autocorrelation is moderate (rho=0.36), which means first-differencing should substantially reduce the persistence channel driving lead significance. Combined with quarter FE (to absorb within-year confounders that year FE misses), this two-pronged fix directly addresses both the persistence/lead-failure concern and the OVB concern. If lead coefficients become insignificant under first-differenced PRiskQ with quarter FE, the causal interpretation is substantially strengthened. Additionally, the ~3x attenuation of lead betas relative to base betas should be explicitly reported as diagnostic evidence.
