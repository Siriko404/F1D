## 6. Variable Dictionary

### 6.1 Dependent Variables (Textual -- Linguistic)

#### 6.1.1 `Manager_QA_Uncertainty_pct` -- DV (Manager models)

**Role:** Dependent variable in Manager Baseline and Manager Extended models.

**Complete construction chain:**

1. **Dictionary source.** The Loughran-McDonald (2011) financial sentiment dictionary provides a curated word list for the "Uncertainty" category. The dictionary is loaded by `load_lm_dictionary()` in the Stage 2 Python tokenization engine (`tokenize_transcripts.py:768`).

2. **Transcript ingestion and tokenization (Stage 2).** Raw earnings-call transcripts are processed by a Python tokenization engine (sklearn `CountVectorizer`, `tokenize_transcripts.py`) that: (a) segments each transcript into Presentation and Q&A sections using section-boundary markers in the raw text; (b) within each section, identifies speaker turns and classifies speakers as CEO, Manager (all executives including CEO), Analyst, or other; (c) tokenizes each speaker turn into lowercase words; (d) matches each token against the LM dictionary categories (Uncertainty, Negative, Positive, Constraining, Litigious, Strong Modal, Weak Modal).

3. **Word counting and percentage computation (Stage 2).** For the "Manager" speaker group in the "QA" section: `Manager_QA_Uncertainty_pct = (count of Uncertainty-category words in all-manager Q&A speech / total word count in all-manager Q&A speech) x 100`. The "Manager" group includes ALL executives identified in the transcript (CEO, CFO, other named executives). If no manager speech exists in the Q&A section, the value is NaN for that call.

4. **Storage.** Results are written to year-partitioned parquet files: `outputs/2_Textual_Analysis/2.2_Variables/{timestamp}/linguistic_variables_{year}.parquet`, one file per calendar year. Each file contains `file_name` (unique call identifier) plus all `_pct` columns.

5. **Loading via LinguisticEngine singleton** (`_linguistic_engine.py:145-314`). `LinguisticEngine.get_data(root_path, years)` loads all year files via `pd.read_parquet()`, concatenates them with `pd.concat(all_data, ignore_index=True)` (`_linguistic_engine.py:232`), adds a `year` column derived from the parquet filename (not from the file_name string) (`_linguistic_engine.py:215-216`), and caches the combined DataFrame.

6. **Winsorization** (`_linguistic_engine.py:254-258`). All `_pct` columns (including `Manager_QA_Uncertainty_pct`) are winsorized per-year at **0%/99% (upper-only)** via `winsorize_by_year(combined, existing_pct_cols, year_col="year", lower=0.0, upper=0.99, min_obs=10)`. This means: the lower bound is the 0th percentile (no lower clipping -- percentages cannot be negative by construction), and the upper bound is the 99th percentile within each calendar year. Years with fewer than 10 valid observations are skipped. Inf values are replaced with NaN before clipping (`winsorization.py:56-58`). The `winsorize_by_year` function operates on a copy (`winsorization.py:53`), groups by year with `dropna=True` (`winsorization.py:65`), computes `quantile(0.0)` and `quantile(0.99)` on non-NaN values (`winsorization.py:79-80`), and clips via `vals.clip(lower=p_lower, upper=p_upper)` (`winsorization.py:82`).

7. **Builder extraction** (`ceo_qa_uncertainty.py` -- analogous pattern for all linguistic builders). `ManagerQAUncertaintyBuilder.build()` calls `engine.get_data(root_path, years)` on the module-level singleton (`_linguistic_engine.py:308`), then extracts `all_data[["file_name", "Manager_QA_Uncertainty_pct"]].copy()` (`ceo_qa_uncertainty.py:43`, same pattern).

8. **Panel merge** (`build_h0_3_ceo_clarity_extended_panel.py:190-224`). Left merge on `file_name` into the manifest base. Uniqueness assertions on `file_name` before and after merge (lines 180-185, 200-205). Row count assertion: no change after merge (lines 220-224).

9. **Regression use.** NOT z-scored in regression (only financial/ordinal controls are standardized). Enters the formula as the dependent variable: `Manager_QA_Uncertainty_pct ~ C(ceo_id) + controls + C(year)` (`run_h0_3:369`).

**Coverage:** 108,215 / 112,968 = 95.8% (4,753 calls have no manager Q&A speech).

### 6.3 Base Firm Controls -- CRSP Variables

#### 6.3.1 `StockRet` -- Compound stock return (%)

**Complete construction chain:**

1. **Raw data.** CRSP Daily Stock File (DSF), stored in 96 quarterly parquet files: `inputs/CRSP_DSF/CRSP_DSF_{year}_Q{q}.parquet`. Key fields loaded: `PERMNO`, `date`, `RET`, `VWRETD`, `VOL`, `PRC` (`_crsp_engine.py:72-77`). All columns are coerced to numeric (`_crsp_engine.py:88-90`); `PRC` is absolute-valued (`_crsp_engine.py:92`). `PERMNO` is coerced to numeric, NaN rows dropped, then cast to int (`_crsp_engine.py:95-97`).

2. **PERMNO linkage via CCM** (`_crsp_engine.py:102-159`). The CRSP-Compustat Merged (CCM) linktable (`inputs/CRSPCompustat_CCM/CRSPCompustat_CCM.parquet`) maps `gvkey` to `LPERMNO`. Date-bounded lookup: for each call, the CCM link where `linkdt <= start_date <= linkenddt` is selected (CRITICAL-3 fix). Missing `linkenddt` is filled with `2099-12-31` (active link). If multiple valid links exist, the most recent `linkdt` is used. Output: one `permno_int` per `file_name`.

3. **Inter-call window computation** (`_crsp_engine.py:321-328, 361-366`). On the FULL manifest (all years, MAJOR-2 fix): `prev_call_date = groupby("gvkey")["start_date"].shift(1)` and `next_call_date = groupby("gvkey")["start_date"].shift(-1)`. Window boundaries: `window_start = start_date + 1 day` (constant `DAYS_AFTER_CURRENT_CALL = 1`, line 41); `window_end = next_call_date - 5 days` (constant `DAYS_BEFORE_NEXT_CALL = 5`, line 42). Calls with no `next_call_date` (last call for a firm) get NaN `window_end` and are excluded.

4. **Compound return formula** (`_crsp_engine.py:235-253`). For each call, CRSP daily records within `[window_start, window_end]` for the matched `PERMNO` are selected. The log-sum trick is used: `_log1p_ret = log1p(RET)` per day, then `sum_log_ret = groupby("file_name")["_log1p_ret"].sum(min_count=1)`. Final: `StockRet = expm1(sum_log_ret) x 100` (`_crsp_engine.py:253`). This is algebraically equivalent to `((1+r1)(1+r2)...(1+rN) - 1) x 100` -- a compound gross return expressed in percentage.

5. **Minimum trading days guard** (`_crsp_engine.py:244,251-252`). `n_valid = groupby("file_name")["_ret_valid"].sum()`. If `n_valid < MIN_TRADING_DAYS` (10), `StockRet` is set to NaN via `.where(sufficient)` (line 253).

6. **Winsorization** (`_crsp_engine.py:444-447`). Per-year 1%/99% via `winsorize_by_year(result_with_year, CRSP_RETURN_COLS, year_col="year")`. The year column is derived from the manifest's `start_date.dt.year` (`_crsp_engine.py:329`). The `winsorize_by_year` function (`winsorization.py:23-84`): replaces Inf with NaN, groups by year (`dropna=True`), requires `min_obs=10` valid values per year, computes `quantile(0.01)` and `quantile(0.99)` on non-NaN values, clips via `vals.clip(lower=p1, upper=p99)`.

7. **Builder extraction** (`stock_return.py:29-51`). `StockReturnBuilder.build()` calls `engine.get_data(root_path, manifest_path)` on the CRSPEngine singleton, then extracts `result_df[["file_name", "StockRet"]]`.

8. **Panel merge** (`build_h0_3:218`). Left merge on `file_name`.

9. **Regression standardization** (`run_h0_3:345-363`). Z-scored within model estimation sample: `(x - mean) / std` where `std > 0`.

**Coverage:** 105,777 / 112,968 = 93.6%. **Timing:** POST-call window (start_date+1d to next_call-5d).

#### 6.3.3 `EPS_Growth` -- Year-over-year EPS growth

**Complete construction chain:**

1. **Raw data.** Compustat North America quarterly: `inputs/comp_na_daily_all/comp_na_daily_all.parquet`. Raw field: `epspxq` (EPS excluding extraordinary items, quarterly). Loaded with all `REQUIRED_COMPUSTAT_COLS` (`_compustat_engine.py:142-181`). `gvkey` is zero-padded to 6 characters; `datadate` is converted to datetime; all numeric columns are coerced to float64 (`_compustat_engine.py:1239-1247`).

2. **Deduplication** (`_compustat_engine.py:1250-1257`). `drop_duplicates(subset=["gvkey", "datadate"], keep="last")` -- if the same (gvkey, datadate) appears multiple times (restatements), the last row (most recent restatement) is kept.

3. **Date-based lag computation** (`_compustat_engine.py:184-231`). Instead of `shift(4)` (which counts rows, not calendar quarters, and fails for missing quarters -- MAJOR-3 fix):
   - `target_lag_date = datadate - 365 days` (line 204).
   - `merge_asof` backward within `gvkey` to find the closest prior quarter's `epspxq` (line 207-214).
   - Accept the lag only if `|target_lag_date - lag_datadate| <= 45 days` AND `epspxq_lag` is not NaN AND `epspxq_lag != 0` (lines 217-221).
   - Formula: `EPS_Growth = (epspxq - epspxq_lag) / |epspxq_lag|` (line 224-228). Division by absolute value of lag EPS ensures correct sign interpretation for negative-EPS firms.

4. **Inf/NaN handling** (`_compustat_engine.py:1152-1172`). Inf values are replaced with NaN after ratio computation (MINOR-9 fix). `EPS_Growth` is computed after the general inf replacement (line 1175), but the lag function itself uses `np.where` which produces NaN (not Inf) for invalid cases.

5. **Winsorization** (`_compustat_engine.py:1194-1201`). Per-fyearq 1%/99% via `_winsorize_by_year(comp[col], year_col)` where `year_col = comp["fyearq"]`. The internal `_winsorize_by_year` function (`_compustat_engine.py:443-467`): groups by `fyearq`, requires `min_obs=10` (m-2 fix), computes `quantile(0.01)` and `quantile(0.99)`, clips via `vals.clip(lower=p1, upper=p99)`.

6. **Manifest matching** (`_compustat_engine.py:1268-1302`). `CompustatEngine.match_to_manifest()` performs `merge_asof` backward: `left_on="start_date", right_on="datadate", by="gvkey", direction="backward"`. This assigns to each call the most recent Compustat quarter ending on or before the call's `start_date`.

7. **Builder extraction** (`eps_growth.py`). `EPSGrowthBuilder.build()` calls `engine.match_to_manifest(manifest, root_path)`, extracts `merged[["file_name", "EPS_Growth"]]`.

8. **Panel merge and regression.** Left on `file_name`; z-scored in regression.

#### 6.3.4 `SurpDec` -- Earnings surprise decile (-5 to +5)

**Complete construction chain:**

1. **Raw data.** IBES Detail files: `inputs/tr_ibes/tr_ibes_{year}.parquet` (26 yearly files). Fields loaded: `CUSIP`, `ACTDATS` (estimate date), `ANALYS` (analyst ID), `VALUE` (forecast EPS), `MEASURE`, `FPI` (forecast period indicator), `FPEDATS` (fiscal period end date), `ACTUAL` (realized EPS). Loaded via PyArrow dataset API with predicate push-down filter (`_ibes_engine.py:130-143`).

2. **IBES filter chain** (`_ibes_engine.py:122-227`):
   - **(a) Measure/period filter** (I/O level, line 132-136): `MEASURE == "EPS"` AND `FPI in ["6", "7"]` (FPI 6 = current quarter estimate, FPI 7 = next quarter estimate).
   - **(b) CUSIP-to-gvkey linkage** (line 160-163): IBES `CUSIP` truncated to 8 characters -> mapped to `gvkey` via CCM linktable with `LINKPRIM in ["P", "C"]` (primary links only, line 76). Unmatched CUSIPs are dropped.
   - **(c) Most recent estimate per analyst** (line 194-195): Within each `(gvkey, fpedats, ANALYS)`, sort by `actdats` ascending, keep last (most recent estimate).
   - **(d) Stale estimate filter** (line 199-201): For each `(gvkey, fpedats)` group, compute `latest_dates = groupby.transform("max")` on `actdats`. Drop estimates where `(latest_dates - actdats).dt.days > 180` (stale threshold = 180 days, `self.max_stale_days`).
   - **(e) Consensus aggregation** (line 206-217): `groupby(["gvkey", "fpedats"]).agg(...)` produces `numest` (count), `meanest` (mean of VALUE), `stdev` (std of VALUE), `actual` (first ACTUAL -- same for all rows in group), `statpers` (max actdats -- when consensus was formed).
   - **(f) Quality filters** (line 220-227): Require `numest >= 2` (at least 2 analyst estimates), `|meanest| >= $0.05` (avoids near-zero denominators), `stdev` not NaN, and `actual` not NaN.
   - **(g) Surprise computation** (line 232-236): `surprise_raw = actual - meanest`. Also computes `earnings_surprise_ratio = |actual - meanest| / |meanest|`.
   - **(h) Winsorization** (line 111-114): Global (pooled) 1%/99% winsorization on `dispersion` and `earnings_surprise_ratio`. Note: `surprise_raw` is NOT winsorized at this stage.

3. **Call-level matching** (`earnings_surprise.py:106-117`). `merge_asof` backward: `left_on="start_date", right_on="fpedats", by="gvkey", direction="backward", tolerance=pd.Timedelta(days=120)`. This assigns to each call the most recent fiscal period whose `fpedats` falls within 120 days before the call's `start_date`.

4. **Decile ranking** (`earnings_surprise.py:34-61, 132-137`). Within each calendar quarter (`start_date.dt.to_period("Q")`), `surprise_raw` values are ranked into a -5 to +5 ordinal scale:
   - Positive surprises (`surprise_raw > 0`): ranked by descending magnitude into quintiles 1-5 (5 = largest positive surprise) using `rank(ascending=False, method="average", pct=True)` then `ceil(pct * 5)`, mapped to `6 - decile` (line 48-50).
   - Zero surprises: assigned 0 (line 52).
   - Negative surprises (`surprise_raw < 0`): absolute values ranked ascending into quintiles, assigned -1 to -5 (line 54-59).
   - Quarters with fewer than 5 valid surprises: all values set to NaN (line 40-41).

5. **Builder output** (`earnings_surprise.py:138`). Returns `file_name` + `SurpDec`.

6. **Panel merge and regression.** Left on `file_name`; z-scored in regression (treated as a continuous ordinal variable despite its discrete nature).

**Coverage:** 79,093 / 112,968 = 70.0%. **Not winsorized** as an ordinal variable (ranking inherently bounds values to [-5, +5]).

### 6.4 Extended Firm Controls -- Compustat Variables

**Shared Compustat loading chain** (`_compustat_engine.py:1206-1302`):

#### 6.4.2 `BM` -- Book-to-market ratio

- **Raw fields:** Compustat `ceqq` (common/ordinary equity, quarterly), `cshoq` (common shares outstanding, quarterly), `prccq` (price close, quarterly).
- **Formula** (`_compustat_engine.py:1036`): `BM = ceqq / (cshoq * prccq)`. No fillna -- if any component is NaN, BM is NaN. Denominator `cshoq * prccq` is market capitalization.

#### 6.4.3 `BookLev` -- Book leverage (interest-bearing debt only)

- **Formula** (`_compustat_engine.py:1039`): `BookLev = (dlcq.fillna(0) + dlttq.fillna(0)) / atq`. Missing `dlcq` or `dlttq` is treated as zero (conservative: assumes no debt when unreported). This uses interest-bearing debt only, not total liabilities (`ltq`), per Biddle (2009) convention.
- **Inf handling:** Inf replaced with NaN.
- **Winsorization:** Per-fyearq 1%/99%.
- **Builder:** `BookLevBuilder` (`book_lev.py`).
- **Regression:** Z-scored.

#### 6.4.4 `ROA` -- Return on assets (annual, Q4-based)

- **Raw fields:** Compustat `iby` (income before extraordinary items, annual YTD -- Q4 row has full-year value), `atq` (total assets, quarterly).
- **Formula** (`_compustat_engine.py:1050-1060`):
  1. `atq_annual = _compute_annual_q4_variable(comp, "atq", ...)` -- extracts Q4 `atq` per gvkey-fyearq and joins back to all quarters via gvkey+fyearq (`_compustat_engine.py:234-274`). Q4 rows identified by `fqtr == 4` (line 253). Deduplicated by `(gvkey, fyearq)` keep="last" (line 258-259).
  2. `atq_annual_lag1 = _compute_annual_q4_variable_lag(comp, "atq", ...)` -- same as above but with `fyearq += 1` (line 291) so prior year's Q4 assets join to current year.
  3. `avg_assets = (atq_annual + atq_annual_lag1) / 2` (line 1053-1056).
  4. `iby_annual = _compute_annual_q4_variable(comp, "iby", ...)` (line 1057).
  5. `ROA = np.where(avg_assets > 0, iby_annual / avg_assets, np.nan)` (line 1058-1059).
- **Inf handling:** Inf replaced with NaN.
- **Winsorization:** Per-fyearq 1%/99%.
- **Builder:** `ROABuilder` (`roa.py`).
- **Regression:** Z-scored.

- **Raw fields:** Compustat `actq` (current assets, quarterly), `lctq` (current liabilities, quarterly).
- **Formula** (`_compustat_engine.py:1062`): `CurrentRatio = actq / lctq.replace(0, np.nan)`. If `lctq` is zero, it is replaced with NaN to avoid division by zero, producing NaN CurrentRatio. No fillna on either component.
- **Inf handling:** Inf replaced with NaN.
- **Winsorization:** Per-fyearq 1%/99%.
- **Builder:** `CurrentRatioBuilder` (`current_ratio.py`).

#### 6.4.6 `RD_Intensity` -- R&D intensity

### 6.7 Standardization Detail

The 11 financial/ordinal controls are z-scored `(x - mean) / std` within each model's estimation sample (i.e., after complete-case filtering, Main sample filtering, and minimum-calls filtering):

```python
continuous_vars = [
    "Size", "BM", "BookLev", "ROA", "CurrentRatio", "RD_Intensity",
    "Volatility", "StockRet", "MarketRet", "EPS_Growth", "SurpDec",
]
for var in continuous_vars:
    if var in df_reg.columns:
        mean_val = df_reg[var].mean()
        std_val = df_reg[var].std()
        if std_val > 0:
            df_reg[var] = (df_reg[var] - mean_val) / std_val
```

Source: `run_h0_3_ceo_clarity_extended.py:345-363`

Standardization is computed **within each model's estimation sample separately**. Because the four models have different sample sizes (Manager Baseline N=53,070 vs CEO Baseline N=38,671), the same variable has different standardization parameters across models. If `std == 0` for any variable (degenerate case), that variable is left unstandardized.

The 3 linguistic controls (`Manager_Pres_Uncertainty_pct` or `CEO_Pres_Uncertainty_pct`, `Analyst_QA_Uncertainty_pct`, `Entire_All_Negative_pct`) are **NOT** standardized -- they remain in raw percentage units. This means coefficients in the same table are in mixed units: SD units for financial/ordinal controls, raw % for linguistic controls.

## 11. Identification & Inference Assessment

### 11.1 Identification Strategy

This suite is a **robustness check**, not a causal identification strategy. It tests whether CEO fixed-effect estimates of Q&A uncertainty are stable when augmenting baseline financial controls with seven additional firm-level controls. The key finding is that R-squared barely changes between Baseline and Extended models (0.418 -> 0.420 for Manager; 0.372 -> 0.372 for CEO), supporting the claim that measured "uncertainty style" reflects a genuine CEO personal characteristic rather than a proxy for omitted firm fundamentals.

**Supported interpretation:** Descriptive association. The suite demonstrates stability, not causality.

**Overselling risk:** Low, provided the write-up does not claim that stability under extended controls proves the fixed effects are causal or unconfounded. Many omitted variables (CEO tenure, age, board composition, governance quality) are not in the extended set.

### 11.2 Endogeneity and Reverse Causality

Not directly applicable to a robustness check. All controls are contemporaneous with the DV. The design does not attempt causal identification.

### 11.3 Post-Treatment Bias

None of the controls are post-treatment in this design (DV is contemporaneous uncertainty, not a future outcome).

### 11.4 Timing

StockRet, MarketRet, and Volatility use a POST-call window (`start_date + 1 day` to `next_call_date - 5 days`; `_crsp_engine.py:41-42`). These are therefore measured AFTER the dependent variable. For a contemporaneous robustness check, this creates a timing mismatch. However, since the Baseline-vs-Extended comparison uses the same return window in both models, the mismatch does not differentially affect the robustness conclusion.

### 11.5 Moulton Problem and Clustering

Standard errors are clustered at the CEO level, which is the level at which the treatment (CEO FE) varies. With 1,917-2,468 clusters across models, asymptotic cluster-robust inference is reliable (well above conventional minimum of ~50 clusters). Cluster-size distribution for Manager Baseline: min=5, p25=10, median=18, p75=30, max=74. No extreme concentration.

Statsmodels reports z-statistics (asymptotic normal) under `cov_type="cluster"`, not t-statistics. With 1,917+ clusters, the practical difference between z and t critical values is negligible.

### 11.6 Multiple Testing

Not applicable. This is a single robustness table, not a hypothesis test.

### 11.7 Robustness

| Check | Status | Note |
|-------|--------|------|
| Alternative variable definitions | Not tested | The suite IS the robustness check |
| Alternative FE | Not tested | Only CEO + Year; industry FE in other suites |
| Alternative clustering (firm-level) | Not tested | Would be informative given multiple CEOs per firm |
| Outlier sensitivity | Partial | Per-year winsorization at engine level |
| Cross-model sample consistency | Not tested | Manager N=53,070 vs CEO N=38,671 -- different samples |
| Exclude SurpDec (IBES selection test) | Not tested | Would verify results not driven by 30% IBES coverage gap |

### 11.8 Interpretation Constraints

- Coefficients are in mixed units: SD units for 11 financial/ordinal controls, raw % for 3 linguistic controls.
- The key metric is R-squared stability, not individual coefficient magnitudes.
- F-statistics are unreliable and should not be interpreted (see Section 12, L1).

## 14. Design Decisions & Rationale

| Decision | Rationale | Alternative considered | Why rejected |
|----------|-----------|----------------------|-------------|
| **Dummy FE via C(ceo_id) rather than absorbed FE** | Simplicity with statsmodels; numerically identical point estimates and SEs | PanelOLS / pyfixest absorbed FE | Not available in initial implementation; would solve F-stat issue (L1) and memory cost (L13) |
| **CEO-level clustering** | Treatment (CEO FE) varies at CEO level; CEO is the natural clustering unit | Firm-level clustering | Would be informative robustness check (multiple CEOs per firm); not implemented |
| **Main sample only (exclude Finance/Utility)** | Standard in corporate finance literature; financial firms have structurally different leverage/accounting | Include all industries | Would confound financial-variable coefficients with industry-specific accounting |
| **MIN_CALLS = 5** | Ensures meaningful within-CEO variation for FE estimation; prevents singleton FE | Lower threshold (3) or higher (10) | 5 is conventional; lower risks near-singleton FE, higher loses too many CEOs |
| **Within-model standardization of financial controls** | Prevents SVD convergence issues with large dummy matrices + heterogeneous-scale regressors | No standardization, or pre-model standardization | No standardization caused SVD failures; pre-model standardization would use a different (superset) sample |
| **Linguistic controls NOT standardized** | Preserves interpretability of percentage-point coefficients for the key variables of interest | Standardize all controls | Would obscure the natural unit interpretation of uncertainty-percentage coefficients |
| **RD_Intensity fillna(0)** | Standard accounting convention: firms not reporting R&D are assumed to have zero R&D | Drop firms with missing R&D | Would lose substantial sample; convention is well-established in literature |
| **BookLev = (dlcq+dlttq)/atq with fillna(0) on debt** | Interest-bearing debt only; standard definition (e.g., Biddle 2009). Missing debt items -> 0 is conservative. | Include all liabilities (ltq/atq) | ltq includes operating liabilities, distorting leverage interpretation |
| **SurpDec as ordinal control** | Captures earnings surprise non-linearity; ranked to handle outliers | Continuous surprise ratio | Ordinal ranking is more robust to extreme surprise values |
| **POST-call return window for StockRet/MarketRet** | Captures market reaction to call and inter-call period. Window is shared across Baseline and Extended, so does not differentially affect robustness comparison. | Pre-call window | Pre-call would require different data architecture; post-call is consistent across all suites using CRSPEngine |
| **Residuals saved from Baseline models only** | Extended model residuals are not needed downstream; Baseline residuals feed H9 takeover suite | Save all 4 models' residuals | Unnecessary disk/complexity cost |
| **Complete-case analysis (no imputation)** | Transparent; avoids imputation assumptions. Per-model complete-case ensures each regression uses maximum available data for its specification. | Multiple imputation | Would add complexity and assumptions; standard in the literature to use complete cases |
