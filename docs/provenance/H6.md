## A. Suite Overview

### A1) Identity

| Field | Value |
|-------|-------|
| Hypothesis suite name/ID | H6 -- SEC Scrutiny (CCCL) and Speech Vagueness |
| Research question | Does heightened SEC scrutiny (proxied by CCCL shift-intensity exposure) reduce managerial speech vagueness (uncertainty language) in earnings calls? |
| Estimation unit | Individual earnings call (`file_name`) |
| Primary keys | `gvkey` (entity) + `year` (time index, calendar year extracted from `start_date`) |

### A2) Model Family and Estimator

| Field | Value |
|-------|-------|
| Model family | Panel FE (OLS with entity + time fixed effects) |
| Estimator | `linearmodels.panel.PanelOLS` with `EntityEffects + TimeEffects`, `drop_absorbed=True` |
| Within transformation | Automatic via PanelOLS (demeaning within entity + time) |
| Optimizer/solver | Default `linearmodels` OLS solver |
| Weighting | None |

**VERIFIED IMPLEMENTATION ISSUE:** Despite the suite name referencing "CCCL" (an instrument), the actual estimation is **reduced-form OLS**, not IV/2SLS. The CCCL shift-intensity variable enters the regression directly as a right-hand-side variable, not as an instrument in a first-stage/second-stage framework. The variable `iv_cols` in the code (line 154-160) is a misnomer. No `linearmodels.IV2SLS` or `IV2SLS` estimator is used anywhere in this suite. This has significant implications for causal interpretation (see Section K).

### A3) Outcome / Target Definition

| DV | Description | Timing |
|----|-------------|--------|
| `Manager_QA_Uncertainty_pct` | Manager team uncertainty language in Q&A, % of words (LM dictionary) | Contemporaneous (t) |
| `CEO_QA_Uncertainty_pct` | CEO-only uncertainty language in Q&A, % of words | Contemporaneous (t) |
| `Manager_Pres_Uncertainty_pct` | Manager team uncertainty language in presentation | Contemporaneous (t) |
| `CEO_Pres_Uncertainty_pct` | CEO-only uncertainty language in presentation | Contemporaneous (t) |

| `CEO_Clarity_Residual` | Idiosyncratic CEO uncertainty component (residual from H0.3 clarity regression) | Contemporaneous (t) |

**Code reference:** `run_h6_cccl.py` CONFIG lines 83-90.

**Missing data policy:** Listwise deletion at regression stage (`.dropna(subset=required)`). Inf values replaced with NaN before drop.

### A4) Main RHS and Controls

**Key independent variable:**

| Variable | Description | Timing |
|----------|-------------|--------|
| `shift_intensity_mkvalt_ff48` | CCCL shift-intensity: market-value-weighted share of FF48 industry peers receiving SEC comment letters, normalized 0-1 | **Contemporaneous (t)** -- same fiscal year as the call |

**CRITICAL NOTE (from red-team audit RT-2):** The key RHS is **contemporaneous** CCCL exposure at time t, NOT lagged CCCL (t-1). The code at line 160 reads `iv_cols = ["shift_intensity_mkvalt_ff48"]`. The lagged variable `shift_intensity_mkvalt_ff48_lag` exists in the panel (constructed in `build_h6_cccl_panel.py` `create_lag_variables()`) but is **never used in any regression**. This eliminates temporal separation between treatment and outcome and creates severe reverse-causality risk.

**Financial controls (`BASE_CONTROLS`):**

| Variable | Source |
|----------|--------|
| `Size` | ln(atq) from Compustat |
| `Lev` | (dlcq + dlttq) / atq from Compustat |
| `ROA` | iby_annual / avg_assets from Compustat |
| `TobinsQ` | (mktcap + debt_book) / atq from Compustat |
| `CashHoldings` | cheq / atq from Compustat |
| `earnings_volatility` | Rolling std of quarterly earnings scaled by total assets, Compustat |
| `RD_Intensity` | xrdq / atq from Compustat |
| `Volatility` | Annualized std of daily returns from CRSP |

**Code reference:** `run_h6_cccl.py` lines 94-103.

**REFEREE CONCERN (RT-8, M10):** Three controls (`earnings_volatility`, `RD_Intensity`, `Volatility`) were added in the code refactoring (commit `cf25167`). `Volatility` (stock return volatility) and `RD_Intensity` may be affected by SEC scrutiny -- they could be post-treatment mediators. If CCCL exposure affects firm volatility or R&D behavior, controlling for them biases the CCCL coefficient toward zero. No specification without these potentially post-treatment controls is provided.

**Sample filters:**
- **Industry samples:** Main (FF12 codes 1-7, 9-10, 12), Finance (FF12=11), Utility (FF12=8)
- **Minimum calls filter:** Firms must have >= 5 calls with non-null `shift_intensity_mkvalt_ff48` to be included. (`run_h6_cccl.py` lines 493-498; note: counts non-null contemporaneous CCCL, not lag.)

**Pre-trends falsification (supplementary):** Each regression is re-run adding `shift_intensity_mkvalt_ff48_lead1` (t+1) and `shift_intensity_mkvalt_ff48_lead2` (t+2) as additional regressors.

**Hypothesis tests (from docstring lines 24-26):**
- H6-A: beta(CCCL) < 0 (Scrutiny reduces uncertainty)
- H6-B: |beta(QA)| > |beta(Pres)| (Stronger effect in spontaneous speech)

### A5) Fixed Effects + Variance Estimator

| Field | Value |
|-------|-------|
| Entity FE | `gvkey` (firm) |
| Time FE | `year` (calendar year) |
| SE/Cluster | Clustered at the firm level (`cov_type="clustered"`, `cluster_entity=True`) |
| Small-sample correction | Default linearmodels cluster correction |

**Code reference:** `run_h6_cccl.py` lines 201-203.

**REFEREE CONCERN:** The time index is `year` (calendar year), not fiscal quarter. With 15 time periods (2004-2018 for CCCL-valid data), time FE absorb only 14 degrees of freedom. This is a coarse time index -- within-year variation in CCCL exposure is not absorbed, which is appropriate for an annual instrument but means any call-quarter-level confounders are not controlled.

**REFEREE CONCERN (M6):** The PanelOLS index `(gvkey, year)` is **non-unique**: with mean 3.9 calls per firm-year, there are approximately 48,817 duplicate index entries in the Main MgrQA sample (67,393 obs). PanelOLS entity effects demean within gvkey and time effects within year, which is functionally correct for call-level analysis, but standard errors assume observations within a cluster are correlated. With ~4 calls per firm-year, within-firm-year correlation is likely high. This should be discussed and potentially addressed via firm-year aggregation or multi-way clustering.

### A6) Model-Family Addendum: IV / 2SLS

**N/A as implemented.** The suite does NOT implement IV/2SLS estimation despite the "CCCL Instrument" naming convention. The CCCL shift-intensity variable is used as a direct regressor in reduced-form OLS.

**What would be required for a proper IV setup:**
- Endogenous regressor: a firm-level SEC scrutiny measure (e.g., whether the firm itself received a comment letter)
- Instrument: `shift_intensity_mkvalt_ff48` (industry-level CCCL exposure)
- First stage: Firm-level scrutiny ~ CCCL exposure + controls + FE
- Second stage: Uncertainty ~ Predicted scrutiny + controls + FE
- Diagnostics: First-stage F-stat, weak instrument tests, exclusion restriction argument

- **Listwise deletion:** `df_reg.replace([np.inf, -np.inf], np.nan).dropna(subset=required)` at regression time (line 163)
- **Inf replacement:** Inf values are replaced with NaN before dropping
- **No imputation:** NaN values are never imputed

### Denominator Protections

- Size: `np.where(atq > 0, np.log(atq), np.nan)` -- zero/negative atq -> NaN

- CashHoldings: `cheq / atq` -- no explicit zero-denominator protection (atq=0 -> inf, then caught by inf->NaN replacement)

## K. Corporate Finance Thesis Referee Assessment

### K1) Bottom-line Referee Verdict

The H6 suite is a reduced-form OLS analysis testing whether industry-level SEC scrutiny (proxied by CCCL shift-intensity exposure at time t) is associated with lower managerial speech vagueness in earnings calls. The suite is mechanically competent in its OLS implementation (correct FE, clustering, and merge logic). However, the suite faces three material identification challenges that collectively prevent causal interpretation:

1. **Contemporaneous treatment timing** (CCCL at t regressed on uncertainty at t) creates unresolved reverse-causality risk. The lagged variable exists in the panel but is never used.
2. **Significant pre-trends** (lead1 CCCL coefficient p=0.038 in the primary spec) indicate that future CCCL exposure predicts current uncertainty language, undermining the assumption that CCCL exposure is exogenous to speech patterns.
3. **Within-R-squared is extremely low** (0.03%-1.7% across all specs), meaning the treatment explains virtually none of the within-firm variation in outcomes.

Of 10 successful models (6 Main + 4 Finance), only 2 reach conventional significance at the 5% level (one-tailed): Main Manager QA (p=0.015) and Finance Manager QA (p=0.033). 8 of 10 specs are insignificant. 16 of 36 attempted regressions fail entirely (8 rank-deficiency errors, 8 silent skips). The "instrumental variable" framing is fundamentally misleading because no IV/2SLS estimation is performed.

**Overall status:** SALVAGEABLE WITH MAJOR REVISIONS

**Supported interpretation:** Descriptive association only (reduced-form correlation between contemporaneous industry CCCL exposure and manager uncertainty language, conditional on firm FE and controls). Causal language is **not supported**.

**Would thesis write-up be overselling?** Almost certainly YES if causal language is used. Even the descriptive association is fragile: only 2 of 10 specs are significant, pre-trends are violated, and the economic magnitude is negligible (a one-SD increase in CCCL implies only ~0.004 pp decrease in uncertainty -- economically trivial).

### K6) Interpretation and Claims Discipline

- **Are causal verbs justified?** No. The reduced-form OLS with a contemporaneous, non-instrumented industry exposure variable cannot support causal language. The significant pre-trends (lead1 p=0.038) further disqualify causal interpretation. The thesis should state "is associated with" or "predicts," not "causes" or "reduces."
- **Are mechanism claims tested?** No. The H6-B hypothesis (|beta(QA)| > |beta(Pres)|, "stronger in spontaneous speech") is stated in the docstring (line 26) but no formal Wald test or coefficient comparison is implemented.
- **Are economic magnitudes interpretable?** Barely. The coefficient on Main-MgrQA (-0.1125) means a one-unit increase in CCCL exposure is associated with a 0.1125 pp decrease in manager QA uncertainty. Given CCCL's extreme distribution (mean=0.008, max=1.0), a one-unit increase is economically implausible. A one-standard-deviation (0.038) increase implies only ~0.004 pp decrease -- economically trivial. Within-R2 of 0.38% confirms negligible explanatory power.
- **Are non-results or fragile results hidden?** Risk is present. 8 of 10 successful specs are insignificant. 16 of 36 attempted regressions fail entirely. The LaTeX table shows only Main-sample columns and may give an impression of broader significance if only the significant result is discussed.
- **Are p-values likely understated?** Probably. (1) One-tailed tests reduce p-values by half relative to two-tailed. (2) Clarity residual regressions do not adjust SEs for generated regressor bias. (3) No multiple testing correction across 10+ tests. (4) Pre-trends violation suggests the "significant" result may reflect pre-existing trends, not a treatment effect.
- **Specification mining risk:** High. With 6 DVs and 3 samples, the single clearly significant result (Manager QA, Main) plus one borderline result (Manager QA, Finance) is consistent with chance. The probability of at least one p<0.05 under the null with 10 independent tests is approximately 40%.

## N. Final Readiness Statement

- **Can this suite be reproduced end-to-end from code alone?** PARTIAL. Code is present and executable, but depends on prior stage outputs (Stages 1-2, H0.3) and raw data inputs that are not in the repository. The H0.3 dependency for Clarity Residuals is undocumented. Utility regressions will fail with rank-deficiency errors (expected but undocumented).

- **Can a committee/referee understand exactly how variables are built?** YES, with effort. The builder pattern is well-documented and traceable. Variable construction can be followed from raw data through engines to builders to panel.

- **Are the implemented results trustworthy for descriptive inference?** MARGINALLY. The reduced-form association between contemporaneous industry CCCL exposure and speech uncertainty is a valid descriptive finding, but: (1) the non-unique panel index (L7) raises questions about correct standard error computation, (2) only 2 of 10 successful specs reach significance, (3) within-R2 is near zero (<0.4% in primary specs), and (4) no multiple testing correction is applied.

- **Are they trustworthy for causal inference?** NO. The suite does not implement IV/2SLS despite the naming. The contemporaneous CCCL variable (same fiscal year as the outcome) provides no temporal separation between treatment and outcome. The pre-trends test fails (lead1 p=0.038), indicating that future CCCL predicts current uncertainty -- a direct violation of the identifying assumption. Look-ahead bias (L6), own-firm contamination (L10), and omitted industry-time confounders (L13) further undermine identification.

- **What is the single biggest threat to credibility?** The combination of contemporaneous treatment timing (L1) and significant pre-trends (L3). Together, these indicate that the "treatment" and "outcome" are jointly determined and that pre-existing trends are confounding the estimated association. Causal language is indefensible.

- **What is the single highest-value next fix?** Switch the primary specification to lagged CCCL (t-1) (Priority 1). The lag variable already exists in the panel and requires only a one-line code change (`iv_cols = ["shift_intensity_mkvalt_ff48_lag"]`). This would establish temporal ordering, mitigate look-ahead bias, and partially address the pre-trends concern. Combined with honest "reduced-form" framing (Priority 2), this would substantially improve the suite's credibility.
