## 1. Suite Overview

| Field | Value |
|-------|-------|
| Hypothesis | H6: Does heightened SEC scrutiny (proxied by CCCL shift-intensity exposure) reduce managerial speech vagueness (uncertainty language) in earnings calls? |
| Research question | Whether industry-level SEC comment-letter exposure is associated with reduced uncertainty language in earnings calls |
| Unit of observation | Individual earnings call (`file_name`) |

**Critical framing note.** Despite the suite name referencing "CCCL" (an instrument), the actual estimation is reduced-form OLS, not IV/2SLS. The CCCL shift-intensity variable enters the regression directly as a right-hand-side variable. No `IV2SLS` estimator is imported or invoked anywhere in `run_h6_cccl.py`. The variable `iv_cols` (line 154) is a misnomer -- it holds ordinary regressors. This distinction is material to interpretation: the estimated coefficient captures the reduced-form association between industry-level CCCL exposure and speech vagueness, not a local average treatment effect.

## 6. Variable Dictionary

### 6.1 Dependent Variables (Linguistic Uncertainty Measures)

All four DVs share an identical construction pipeline. The only variation is which speaker/context subset of the transcript is used.

**Full construction chain (common to all four DVs):**

1. **Dictionary.** The Loughran-McDonald (LM) master dictionary CSV is loaded by `tokenize_transcripts.py` function `load_lm_dictionary()` (line 768). The `Uncertainty` column in the CSV identifies uncertainty words; any word with `Uncertainty > 0` is added to the `Uncertainty` category set. The dictionary is loaded once per process and reused for all years.

2. **Tokenization.** Each transcript turn is tokenized with a regex tokenizer `r"(?u)\b[a-zA-Z]+\b"` (line 958) that matches alphabetic-only tokens. All tokens are uppercased for dictionary matching. A sparse feature matrix is constructed via scikit-learn `CountVectorizer` restricted to the LM vocabulary. For each turn, `Uncertainty_count` = sum of non-zero entries in the sparse matrix columns corresponding to LM Uncertainty words (line 952: `X[:, indices].sum(axis=1)`). `total_tokens` = count of all alphabetic tokens in the raw text (line 959: `len(regex.findall(x))`).

3. **Speaker/context classification.** Each turn is classified as Manager (keyword + employer match), Analyst (role contains "analyst"), CEO (exact/last-name match against Execucomp CEO names), and by context (Presentation vs Q&A). Code: `build_linguistic_variables.py` `classify_speakers()` lines 430-464.

4. **Aggregation.** For each (speaker, context) combination, word counts and total tokens are summed across all turns within a call (file_name). The percentage is computed as `(sum(Uncertainty_count) / sum(total_tokens)) * 100.0`, with `total_tokens == 0` mapped to NaN. Code: `aggregate_weighted()` lines 472-502. Output column naming: `{Speaker}_{Context}_Uncertainty_pct`.

5. **Partitioned output.** Results are saved as per-year parquet files: `linguistic_variables_{year}.parquet` in `outputs/2_Textual_Analysis/2.2_Variables/{timestamp}/`.

6. **Winsorization at load time.** `LinguisticEngine.get_data()` (lines 254-258) loads all year files, concatenates them, and applies per-year winsorization via `winsorize_by_year()` with parameters `lower=0.0, upper=0.99, min_obs=10`. This is upper-only clipping: within each calendar year, values above the 99th percentile are clipped to the 99th percentile; no lower clipping is applied (lower bound is 0.0, meaning the 0th percentile, which is the minimum). The natural floor of 0% makes lower winsorization unnecessary.

7. **Builder extraction.** Each builder (e.g., `ManagerQAUncertaintyBuilder`) queries the `LinguisticEngine` singleton and extracts `[file_name, {column}]`. The engine caches after first load; all four DV builders share a single load pass.

8. **Panel merge.** Each builder's output is left-joined onto the manifest by `file_name` in `build_h6_cccl_panel.py` (lines 222-228). Zero-row-delta enforced.

### 6.2 Key Independent Variable (CCCL Shift-Intensity Instrument)

**Full construction chain for `shift_intensity_mkvalt_ff48`:**

1. **External data source.** Pre-computed CCCL (SEC Comment Letter) shift-share instrument file: `inputs/CCCL_instrument/instrument_shift_intensity_2005_2022.parquet`. This file contains 145,693 rows (18,132 unique gvkey; year range 2005-2021; 0 missing values for `shift_intensity_mkvalt_ff48`). The instrument is at firm-year (`gvkey x year`) granularity with zero duplicate `(gvkey, year)` pairs.

2. **What it measures.** The variable is a Bartik-style shift-share instrument that captures a firm's exposure to SEC comment-letter scrutiny at the industry level. It is constructed (externally, not in this codebase) as the market-value-weighted share of a firm's Fama-French 48-industry peers that received SEC comment letters. Higher values indicate that the firm operates in an industry where a larger fraction of peers (weighted by market capitalization) were subject to SEC scrutiny. The variable is bounded [0, 1].

3. **Column selection via config.** `CCCLInstrumentBuilder.__init__()` (line 45) reads `config.get("column", "shift_intensity_sale_ff48")` -- the default is `shift_intensity_sale_ff48` (sales-weighted). This is overridden by `config/variables.yaml` lines 496-499 which set `column: "shift_intensity_mkvalt_ff48"` (market-value-weighted). The parquet file contains multiple instrument variants; only the configured column is extracted.

4. **Loading.** `CCCLInstrumentBuilder.build()` reads the parquet with `columns=["gvkey", "year", self.target_col]` (line 113-116). gvkey is zero-padded to 6 characters. The `year` column is renamed to `fyearq_int` and cast to `Int64` (lines 120-121).

5. **Uniqueness assertion.** Before merge, `cccl.duplicated(subset=["gvkey", "fyearq_int"]).any()` is checked (lines 124-129). If duplicates exist, a `ValueError` is raised to prevent silent fan-out.

6. **fyearq attachment to manifest.** The manifest (with `file_name`, `gvkey`, `start_date`) is matched to Compustat via `pd.merge_asof()` on `start_date` (left) to `datadate` (right), grouped by `gvkey`, direction `"backward"` (lines 77-84). This attaches the Compustat fiscal year (`fyearq`) from the most recent quarterly filing date on or before the call date. The attached `fyearq` is converted to integer as `fyearq_int` (lines 90-92). Match rate: ~99.9%.

7. **CCCL merge.** The manifest-with-fyearq is left-joined to the CCCL instrument on `["gvkey", "fyearq_int"]` (lines 133-136). Zero-row-delta enforced (lines 140-144). Match rate: 84.5% (95,422/112,968 non-null). The 15.5% missing rate is driven by calls in fiscal years before 2005 (when the CCCL instrument begins) and firms not present in the CCCL dataset.

8. **No winsorization.** The CCCL instrument is not winsorized at any stage.

| Variable (code) | Paper label | Type | Timing | Winsorization | Non-null |
|-----------------|-------------|------|--------|---------------|----------|
| `shift_intensity_mkvalt_ff48` | CCCL Exposure_t | Main RHS | Contemporaneous (t): same fiscal year as call, matched via fyearq | None | 95,422 (84.5%) |

**Code references:** `cccl_instrument.py` lines 35-175 (full builder); `config/variables.yaml` lines 496-499 (column override).

### 6.3 Lag and Lead Variables (Derived from CCCL Instrument)

**Full construction chain (common to lag, lead1, lead2):**

All three are constructed in `build_h6_cccl_panel.py` function `create_lag_variables()` (lines 69-156).

1. **fyearq attachment.** `attach_fyearq()` from `panel_utils.py` attaches Compustat fiscal year to the panel via `merge_asof` (call `start_date` backward to Compustat `datadate`, per `gvkey`). This is the canonical shared implementation used across all suites.

2. **Firm-year deduplication.** The panel is sorted by `["gvkey", "fyearq_int", "start_date_dt"]` and deduplicated to keep the last call per `(gvkey, fyearq_int)` (lines 94-98). This produces a firm-year-level lookup of contemporaneous `shift_intensity_mkvalt_ff48`.

3. **Temporal shifting within gvkey.** The firm-year lookup is sorted by `["gvkey", "fyearq_int"]` and shifted using `groupby("gvkey").shift()`:
   - **Lag (t-1):** `shift(1)` on `shift_intensity_mkvalt_ff48` (lines 105-106). The shifted `fyearq_int` is also retrieved via `shift(1)` on the `fyearq_int` column (line 104).
   - **Lead1 (t+1):** `shift(-1)` on `shift_intensity_mkvalt_ff48` (lines 113-114). Shifted `fyearq_int` via `shift(-1)` (line 112).
   - **Lead2 (t+2):** `shift(-2)` on `shift_intensity_mkvalt_ff48` (lines 121-122). Shifted `fyearq_int` via `shift(-2)` (line 120).

4. **Consecutive-year gap check.** Each shifted value is validated for consecutive fiscal years:
   - Lag: `(fyearq_int - prev_fyearq) == 1` (line 108). If the gap is not exactly 1 year, the lag is set to NaN (line 109).
   - Lead1: `(next_fyearq - fyearq_int) == 1` (line 116). Non-consecutive -> NaN (line 117).
   - Lead2: `(next_fyearq2 - fyearq_int) == 2` (line 124). Non-consecutive -> NaN (line 125).
   This prevents contamination from non-adjacent fiscal years (e.g., a firm missing from the CCCL data for one year would not have its t-2 value misattributed to t-1).

5. **Merge back to call level.** The firm-year lookup (with lag/lead columns) is left-joined back onto all calls in the panel via `["gvkey", "fyearq_int"]` (line 144). This means every call within the same firm-year receives the same lag/lead values.

|-----------------|-------------|------|--------|----------|-----------------|
| `shift_intensity_mkvalt_ff48_lag` | CCCL Exposure_{t-1} | Constructed | t-1 (prior consecutive fiscal year) | 86,189 (76.3%) | **UNUSED** -- constructed but never loaded by `run_h6_cccl.py` (grep returns zero matches) |

7. **Winsorization.** Per-fyearq 1%/99% (applied with all other Compustat controls at lines 1194-1201).

## 11. Identification & Inference Assessment

### 11.1 Research Design

The H6 suite estimates a reduced-form regression:

```
Uncertainty_{i,t} = alpha + beta * CCCL_exposure_{j(i),t} + gamma * Controls_{i,t} + mu_i + tau_t + epsilon_{i,t}
```

where `j(i)` is firm `i`'s FF48 industry, `mu_i` are firm fixed effects, and `tau_t` are calendar-year fixed effects. The CCCL shift-intensity variable is a Bartik-style shift-share instrument: it interacts industry peers' lagged market-cap shares with the number of SEC comment letters received in the industry. It is designed to capture exogenous variation in SEC scrutiny at the industry level.

### 11.2 Instrument Validity (Conceptual)

For the CCCL shift-intensity to serve as a valid instrument (or to support reduced-form causal interpretation), two conditions must hold:

1. **Relevance:** Industry-level SEC comment-letter activity must predict firm-level SEC scrutiny awareness. This is plausible but not tested within this suite (no first stage is estimated).

2. **Exclusion restriction:** Industry-level CCCL exposure must affect speech vagueness ONLY through the scrutiny channel, not through other industry-level channels. This is a strong assumption -- industry-level regulatory activity could affect uncertainty language through information environments, peer effects, or industry-level economic conditions independent of SEC scrutiny per se.

### 11.3 Identification Threats

| Threat | Evidence | Severity | Assessment |
|--------|----------|----------|------------|
| **Contemporaneous treatment** | `iv_cols = ["shift_intensity_mkvalt_ff48"]` at time t; lag exists but unused | Critical | No temporal separation. Treatment and outcome are simultaneously determined. Causal interpretation indefensible. |
| **Look-ahead bias** | CCCL matched at fiscal year level. A Q1 call receives full-year CCCL including Q2-Q4 activity | High | Early-year calls assigned CCCL exposure that includes future months' SEC activity. This is a mechanical confound. |
| **Pre-trends violation** | Lead1 beta=-0.0727, p=0.038 (two-tailed) in Main MgrQA | High (with caveat) | Future CCCL predicts current uncertainty, violating the parallel-trends assumption. Caveat: pre-trends regression uses 15.2% fewer observations (57,136 vs 67,393); marginal significance may partly reflect sample composition. |
| **Omitted industry-time confounders** | Only firm + year FE; no industry-year FE | Medium | Industry-year shocks (sector downturns, regulatory waves) could simultaneously drive SEC scrutiny and speech uncertainty. Year FE absorb only common time effects. |
| **Potential bad controls** | `earnings_volatility`, `RD_Intensity`, `Volatility` may be affected by SEC scrutiny | Medium | If CCCL exposure affects firm volatility or R&D, controlling for them attenuates the coefficient toward zero. No specification without these controls is provided. |
| **Own-firm contamination in CCCL** | CCCL uses `share_lag_mkvalt_ff48` which is firm-specific market cap weight | Medium | Larger firms mechanically have higher exposure. The instrument is not purely industry-level. Whether leave-one-out exclusion is applied in the external CCCL construction is not verified from the codebase. |
| **Extreme CCCL skewness** | Mean=0.008, p75~0, max=1.0 in Main sample | Medium | OLS coefficient driven by a small number of high-exposure observations. Results may be outlier-sensitive. |

### 11.4 Inference Concerns

| Concern | Details |
|---------|---------|
| **Non-unique panel index** | PanelOLS index is `(gvkey, year)` but the panel is call-level with ~3.9 calls per firm-year. PanelOLS handles this correctly for estimation (entity effects demean within gvkey, time effects within year), and firm-level clustering accounts for within-firm correlation. However, within-firm-year correlation across calls is not explicitly modeled. Severity: Low-Medium. |
| **One-tailed test convention** | One-tailed tests are appropriate for the directional hypothesis (beta < 0). However, the LaTeX table applies standard star thresholds (0.01/0.05/0.10) to one-tailed p-values (line 303: `fmt_coef(r_1['beta1'], r_1['beta1_p_one'])`). This inflates apparent significance: Finance MgrQA has two-tailed p=0.066 but receives ** stars because one-tailed p=0.033 < 0.05. The table note states "(one-tailed)" but readers accustomed to standard two-tailed conventions may overinterpret the stars. |
| **Multiple testing** | 8 tests (4 DVs x 2 samples) with no correction. 2 significant at p<0.05 (one-tailed) out of 8. The probability of at least 1 false positive at p<0.05 under the null with 8 independent tests is ~34%. After Bonferroni correction (threshold 0.00625), neither result survives. |
| **Economic magnitude** | A one-SD increase in CCCL (~0.038 in Main sample) implies only ~0.004 pp decrease in Manager QA Uncertainty (0.038 * 0.1125). Given the mean of ~0.82%, this is a 0.5% relative change -- economically trivial. Within-R2 of 0.38% confirms negligible explanatory power. |

### 11.5 Supported Interpretation

The suite supports only a descriptive association: "Contemporaneous industry-level SEC comment-letter exposure is weakly negatively correlated with manager Q&A uncertainty language, conditional on firm FE, year FE, and financial controls." Causal language is not supported due to contemporaneous treatment, pre-trends violation, and look-ahead bias. Even the descriptive finding is fragile (2/8 significant, economically trivial, no MHC).

## 14. Design Decisions & Rationale

| Decision | Rationale | Trade-off |
|----------|-----------|-----------|
| **Reduced-form OLS rather than IV/2SLS** | The suite implements reduced-form regression: the CCCL shift-intensity variable enters directly as the RHS treatment variable. No endogenous regressor or first stage is specified. | Loses ability to estimate LATE; gains simplicity and avoids weak-instrument concerns. The reduced-form coefficient has a valid interpretation as "the effect of the instrument on the outcome" but requires careful framing. |
| **Contemporaneous CCCL (t) rather than lagged (t-1)** | The runner uses `shift_intensity_mkvalt_ff48` at time t. The lag variable is constructed in the panel builder but not used. | Eliminates temporal separation between treatment and outcome. This appears to be a design oversight rather than a deliberate choice, as the lag variable was explicitly constructed. |
| **Call-level panel with firm-year index** | Each call is a separate observation. The PanelOLS index `(gvkey, year)` is non-unique (~3.9 calls per firm-year). Entity FE demean within gvkey and time FE within year. | Preserves call-level variation (multiple observations per firm-year capture within-year changes). Trade-off: within-firm-year correlation across calls is not explicitly modeled, though firm-level clustering partially addresses this. |
| **Firm + year FE (not industry-year)** | Firm FE absorb all time-invariant firm characteristics. Year FE absorb common annual shocks. | Does not control for industry-specific time trends, which is the key omitted variable concern for an industry-level instrument. Industry-year FE would be more appropriate but are not implemented. |
| **One-tailed tests** | The directional hypothesis (beta < 0: scrutiny reduces uncertainty) is pre-specified. One-tailed tests are appropriate for pre-specified directional hypotheses. | Reduces p-values by half relative to two-tailed. The combination with standard star thresholds may mislead readers accustomed to two-tailed conventions. |
| **8 financial controls** | The control set includes Size, BookLev, ROA, TobinsQ, CashHoldings, earnings_volatility, RD_Intensity, and Volatility. | Three controls (earnings_volatility, RD_Intensity, Volatility) may be post-treatment if SEC scrutiny affects firm risk and R&D behavior. No specification without these is provided. |
| **Minimum 5 calls filter** | Firms with fewer than 5 calls with non-null CCCL are excluded (lines 474-479). | Removes firms with very limited panel variation. Trade-off: selects toward larger/more-frequently-covered firms. |
| **Per-year winsorization (heterogeneous thresholds by source)** | Linguistic DVs: 0%/99% upper-only. Compustat controls: 1%/99% by fyearq. CRSP Volatility: 1%/99% by year. CCCL: not winsorized. | Linguistic variables have a natural lower bound of 0%, making lower winsorization unnecessary. The table note inaccurately claims uniform 1%/99%. |
| **Warning suppression** | Line 75-77: `warnings.filterwarnings("ignore", message="covariance of constraints does not have full rank")`. | Suppresses a potentially informative diagnostic about covariance rank. May be related to the Utility rank-deficiency failures. Logging would be preferable. |
| **`drop_absorbed=True`** | PanelOLS parameter (line 196) that silently drops regressors collinear with the FE structure. | Prevents estimation failures from absorbed variables. Trade-off: if any controls are dropped, there is no diagnostic output indicating which ones. |
