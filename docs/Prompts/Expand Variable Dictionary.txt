Compustat Financial Variable Construction Patterns
========================================================

Most Compustat variables follow one of these patterns. Trace the ACTUAL code for each:

**Pattern A: Simple quarterly ratio** (e.g., CashHoldings, BookLev, Size, BM, RD_Intensity, CurrentRatio)
- Raw fields loaded from Compustat quarterly parquet
- Formula applied directly to quarterly observations
- Matched to calls via `merge_asof` backward on (gvkey, datadate <= start_date)
- Winsorized at 1%/99% by fiscal year at engine level

**Pattern B: Q4 annual variable** (e.g., ROA, CapexAt, DividendPayer, OCF_Volatility)
- Uses `_compute_annual_q4_variable()` which filters to `fqtr == 4` observations
- These are YTD cumulative values at fiscal year-end
- May involve lagged assets (`_compute_annual_q4_variable_lag` shifts fyearq +1)
- Average assets computed as `(atq_annual + atq_annual_lag1) / 2`
- Winsorized at 1%/99% by fiscal year

**Pattern C: Composite with guards** (e.g., TobinsQ)
- Multiple raw fields combined
- `.clip(lower=0)` on debt components
- NaN guards when all components missing
- Division guards when denominator <= 0

**Pattern D: Rolling window** (e.g., OCF_Volatility)
- Computed on annual panel (one obs per firm-year)
- Rolling window ("1826D" = ~5 years), min_periods
- Standard deviation within window

**Pattern E: CRSP-based** (e.g., StockRet, MarketRet, Volatility, pre_call_amihud)
- CRSP daily data loaded for inter-call windows
- Date-bounded CCM linkage (gvkey → PERMNO)
- Minimum trading days requirement
- Compound return or annualized volatility formula

**Pattern F: IBES-based** (e.g., SurpDec, PostCallDispersion, PreCallDispersion)
- IBES detail data with filter chain (MEASURE=EPS, FPI, stale, numest, meanest)
- Earnings surprise = actual - meanest
- Decile ranking within calendar quarter

Trace the actual code path for each variable — do NOT assume it follows a standard pattern.
