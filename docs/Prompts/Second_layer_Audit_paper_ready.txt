PHASE 2 — Verify claims manually (no bulk checks)
===========================================================
For EACH claim in Claim Inventory:

2.1 Decide the minimal verification method:
- Existence claim → ls/stat/file on that exact path
- Content claim → head/tail/pdftotext or open the exact generator code
- Numeric claim → python -c load ONE file (csv/parquet/json) and print the specific number
- Model-family claim → open the Stage 4 runner code and confirm imports/classes + output naming
- Linkage claim → inspect Stage 4 manifest/log and match the exact Stage 3 panel path + run_id
- “Quality” claim → verify the defined quality test actually matches the artifact contents (e.g., table includes required fields; hazard HR includes CI; notes match code)
- Root-cause claim → open generator code to see whether it exists / is invoked / writes to that location

2.2 Execute targeted command(s), capture evidence, issue verdict:
- VERDICT: PASS / FAIL / UNVERIFIED
- Evidence: command(s) + short output excerpt

IMPORTANT: Do not “verify by intuition.” Every verdict must have evidence.
