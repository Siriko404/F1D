# 1) Which premiums should be used for **Net Gain**?

## What the handout says

Page 1 defines the strategy:

> “The Split Strike Conversion Collar Strategy … includes
>
> 1. Owning a stock,
> 2. Selling a call option on the stock (**at 105% moneyness** in our case) and using the proceeds to
> 3. Purchasing put options on the stock (**at 95% moneyness** in our case).”

Page 3 then defines Net Gain:

> “**Net Gain (on option premiums(@t)) = Call option Premium(@t) − Put option premium(@t)**”

## What the spreadsheet structure says

In `template foe case study.xlsx`, `Modelling` sheet:

* `N11 = Buy Put Option (K = 0.95 S&P 500)`
* `V11 = P`
* `X11 = Sell Call Option (K = 1.05 S&P 500)`
* `AC11 = C`
* `AH11 = Net Gain`

So the workbook structure itself points to:

* **95% put premium** in column **V**
* **105% call premium** in column **AC**
* **Net Gain** in column **AH**

## Audit-safe answer

The only strategy-consistent interpretation is:

[
\text{Net Gain}_t = \text{105% call premium}_t - \text{95% put premium}_t
]

So in this workbook layout, that means:

```excel
AH_t = AC_t - V_t
```

### Example

For row 23, structurally it should be:

```excel
=AC23 - V23
```

## Red-team finding

The filled workbook currently has:

```excel
AH23 = AG23 - W23
```

That is **not** the premium difference.
That is **call payoff minus put payoff**, which conflicts with the handout wording.

So my earlier answer on this point was directionally right, but here is the stricter version:

### Final answer for Q1

**Use the 105% call premium minus the 95% put premium.**
In this sheet layout that is **`AC - V`**, not `AG - W`.

## Premium vs payoff: exact distinction

This is also directly supported by the handout.

### Premium

* `C = ...`

# 6) Are the formulas in columns M, W, and AG correct?

This is the strongest part of the audit.

## What the handout says

Page 4 gives the payoff formulas using:

> “**value of $1 backtested collar (@t − 1)**”

for both:

* **Put option payoff(@t)**
* **Call option payoff**

That means the payoff multiplier must be the **previous-period collar value**, not the current row.

## What the filled workbook does

In `template foe case study.xlsx`:

* `W23 = (MAX($B$7-(1+G23),0))*M23`
* `AG23 = (-MAX(((1+G23)-$C$7),0))*M23`

Those formulas use **`M23`**, i.e. the **current-row** collar value.

But `M23` itself is:

```excel
=(M22*(1+G23))+(AH22*(1+I22))+W23+AG23
```

So:

* `M23` depends on `W23`
* `W23` depends on `M23`
* `AG23` depends on `M23`

That is a circular reference.

## Additional hard evidence

The cached values in the filled workbook show the model breaking immediately:

* `M22 = 1`
* `M23 = 0`
* `M24 = 0`
* `M25 = 0`
* ...

That is exactly what you would expect from a bad circular setup / broken cached workbook.

## Audit-safe answer for W and AG

If the handout is followed, the formulas should use **previous-row M**.

### Correct structure

For row 23:

```excel
W23 = MAX($B$7-(1+G23),0) * M22
AG23 = -MAX((1+G23)-$C$7,0) * M22
```

That matches the PDF wording:

* payoff at time `t`
* scaled by backtested collar value at `t-1`

## What about column M?

The handout’s printed formula for collar value on page 4 is typeset awkwardly, but the only unit-consistent interpretation is additive:

[
M_t = M_{t-1}(1+G_t) + AH_{t-1}(1+I_{t-1}) + W_t + AG_t
]

That is also how the filled workbook writes `M23`, except its `W23` and `AG23` are wrong because they use current-row M.

So the **structure** of `M23` is defensible **if**:

* `AH22` is truly net premium gain from the prior month
* `W23` and `AG23` use **previous-row M**
* the upstream premium blocks are correct

## Red-team catch I need to add

There is a second inconsistency in the filled workbook:

* `H11 = Value of $1 (S&P 500) - S`
* `N23 = H23*$B$7`
* `X23 = H23*$C$7`

But the PDF page 3 says:

> “**Consider the S, as the value of $1 (backtested collar) every month.**”

So the filled workbook’s option-premium blocks are also inconsistent with the handout, because they use **H (S&P)** as `S` instead of the **backtested collar**.

That means even if someone fixes `AH = AC - V`, the premium calculations themselves may still be upstream-wrong if they are based on column H.

### Final answer for Q6

* **W and AG are wrong if they use current-row M.**
* They should use **previous-row M**.
* `M` is only defensible in additive form if its inputs are corrected.
* The filled workbook also has an upstream issue: its premium blocks use **H** as `S`, which conflicts with the handout’s instruction to use the **backtested collar** as `S`.

# 7) What is a “backtested collar,” and how is it used here?

## What the handout says

Page 1 defines the collar strategy:

1. own the underlying
2. sell the 105% call
3. buy the 95% put

Page 2 says:

> “**We are starting with $1, as the initial value of the backtested collar strategy**”

Page 4 then shows how to update the **value of backtested collar(@t)** month by month.

## Audit-safe explanation

A **backtested collar** is:

> the simulated historical path of what **$1** would have grown to if, each month in the past, you had:
>
> * held the S&P 500,
> * sold the 105% call,
> * bought the 95% put,
> * carried the net premium,
> * and applied the realized option payoffs.

So in this assignment:

* **Column M** = historical value of **$1 in the collar strategy**
* **Column AI** = monthly return of that collar strategy

The point of the case is then to compare that synthetic collar series against:

* Fairfield Guard
* HFRI
* S&P 500
* T-bills

and discuss whether Fairfield’s reported returns look believable.

### Final answer for Q7

Your earlier intuition was right: “backtested collar” is just the **simulated historical value path of the collar strategy**.
