Add tooling configuration sections (TOOL-01 to TOOL-05):

**TOOL-01: pyproject.toml Structure**
- PEP 621 compliant [project] section
- Required fields: name, version, description, readme, license, requires-python, authors, classifiers, dependencies
- Optional dependencies: [project.optional-dependencies] with dev, docs groups
- [project.urls] for Homepage, Documentation, Repository
- [build-system] with setuptools backend
- [tool.setuptools.packages.find] for src-layout
- All tool configs consolidated in pyproject.toml (no separate pytest.ini, .flake8, etc.)
- Complete example pyproject.toml

**TOOL-02: Pre-commit Hooks Configuration**
- .pre-commit-config.yaml structure
- Required hooks:
  - trailing-whitespace, end-of-file-fixer (file quality)
  - check-yaml, check-toml, check-json (config validation)
  - check-added-large-files (prevent large commits)
  - check-merge-conflict, check-case-conflict
  - detect-private-key, debug-statements (security)
  - ruff (linter + formatter)
  - mypy (type checking with pydantic, types-PyYAML deps)
  - bandit (security linting)
- Hook version pinning
- CI integration section

**TOOL-03: GitHub Actions Workflow**
- Separate lint job from test job
- Lint job: ruff check, ruff format --diff, mypy
- Test job: Python matrix (3.9, 3.10, 3.11, 3.12, 3.13)
- Coverage reporting with codecov
- E2E tests on main branch only
- Workflow triggers: push to main/master, pull requests
- Caching strategy for pip
- Complete .github/workflows/test.yml example

**TOOL-04: .gitignore Patterns**
- Python bytecode: __pycache__/, *.py[cod]
- Distribution: build/, dist/, *.egg-info/
- Test/coverage: htmlcov/, .coverage, .pytest_cache/
- Environments: .env, .venv, venv/
- IDE: .vscode/, .idea/
- Data files: *.parquet, *.csv, *.xlsx (project-specific)
- Archives: ___Archive/, *.zip, *.tar.gz
- OS-specific: .DS_Store, Thumbs.db
- Secrets: *.pem, *.key, credentials.json, secrets.yaml
- Complete .gitignore example

**TOOL-05: Linting/Formatting Configuration**
- ruff rules: E, W, F, I, B, C4, UP, ARG, SIM (conservative baseline)
- ruff per-file-ignores: __init__.py, tests/**
- ruff format: quote-style double, indent-style space
- mypy configuration: python_version 3.9, warn_return_any, check_untyped_defs
- Tier-based mypy overrides: strict=true for Tier 1 modules
- Alignment between pre-commit hooks and CI
- Complete [tool.ruff] and [tool.mypy] examples

Each section should include:
- Requirement statement
- Rationale
- Complete configuration examples
- Common pitfalls
- Integration notes with other tools
