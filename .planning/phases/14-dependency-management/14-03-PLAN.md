           python-version: ['3.8', '3.9', '3.10', '3.11', '3.12', '3.13']

       ```yaml
         key: ${{ runner.os }}-pip-${{ matrix.python-version }}-${{ hashFiles('requirements.txt') }}
         restore-keys: |
           ${{ runner.os }}-pip-${{ matrix.python-version }}-
       ```

    4. Update artifact names to include Python version:
       ```yaml
         name: coverage-report-${{ matrix.python-version }}
         # and
         name: test-results-${{ matrix.python-version }}
       ```

    This ensures:
    1. All supported Python versions are tested on every push/PR
    2. Failures are caught early for any Python version
    3. Users have confidence in cross-version compatibility

    ## Python Compatibility

    **Supported Versions**: Python 3.8, 3.9, 3.10, 3.11, 3.12, 3.13

    **Minimum Version**: Python 3.8
    **Latest Tested**: Python 3.13

    **Rationale**:
    - Broad compatibility ensures pipeline works on various environments
    - Python 3.8 is oldest stable version still receiving security updates
    - All dependencies support Python 3.8-3.13 range

    **Dependency Constraints**:
    - PyArrow 21.0.0 supports Python 3.8+ (compatible)
    - PyArrow 23.0.0+ requires Python 3.10+ (not compatible with 3.8-3.9)
    - NumPy 2.x and Pandas 2.x support Python 3.9+

    **Testing**:
    - GitHub Actions tests all supported versions on every push/PR
    - Matrix: 3.8, 3.9, 3.10, 3.11, 3.12, 3.13
    - See `.github/workflows/test.yml` for CI/CD configuration

    **Version Pinning in requirements.txt**:
    - Python version range specified as comment: `# Python >= 3.8`
    - Actual version controlled by user environment

    **For Users**:
    - Recommended: Python 3.10 or 3.11 for best performance
    - Minimum: Python 3.8 (fully supported)
    - See UPGRADE_GUIDE.md for Python upgrade procedures

    Add new section:

    ### Python Upgrade Procedure

    Before upgrading to a new Python version, validate compatibility:

    1. **Check Dependency Support**:
       - Review DEPENDENCIES.md for Python version constraints
       - Check each dependency's release notes for Python support
       - Ensure all dependencies support target Python version

    2. **Local Testing**:
       - Install target Python version locally
       - Create virtual environment: `python3.X -m venv venv`
       - Activate and install: `pip install -r requirements.txt`
       - Run pytest: `pytest tests/ -v`
       - Fix any compatibility issues (usually dependency version conflicts)

    3. **GitHub Actions Validation**:
       - Update .github/workflows/test.yml matrix to include new version
       - Push changes and verify CI/CD passes
       - Review test results for any Python-specific failures

    4. **Full Pipeline Validation**:
       - Run full pipeline on new Python version
       - Compare outputs with baseline (from Python 3.10 or 3.11)
       - Verify identical results (accounting for floating-point tolerance)

    5. **Update Documentation**:
       - Add new version to DEPENDENCIES.md "Supported Versions" list
       - Update minimum/maximum version annotations
       - Update .github/workflows/test.yml matrix
       - Document any Python-specific workarounds

    6. **Deprecate Old Versions** (optional):
       - If dropping old versions, update requirements.txt comment
       - Update DEPENDENCIES.md compatibility section
       - Update GitHub Actions matrix
       - Announce deprecation in project README

    Include example workflow for upgrading from Python 3.8 to 3.14 (future upgrade).
  </action>
  <verify>
