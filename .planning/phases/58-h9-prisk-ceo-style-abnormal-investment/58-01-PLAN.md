Create new script 2_Scripts/5_Financial_V3/5.8_H9_StyleFrozen.py implementing CEO style assignment to firm-years.

**Input Files:**
1. CEO Clarity scores: 4_Outputs/4.1.1_CeoClarity_CEO_Only/latest/ceo_clarity_scores.parquet
   - Columns: ceo_id, gamma_i, ClarityCEO_raw, ClarityCEO, n_calls, ceo_name, first_call_date, last_call_date
   - ClarityCEO is standardized (mean=0, SD=1), time-invariant per CEO

2. Manifest: 4_Outputs/1.4_AssembleManifest/latest/master_sample_manifest.parquet
   - Columns: file_name, gvkey, start_date (call_date), ceo_id, ceo_name
   - Links each call to its firm (gvkey) and CEO (ceo_id)

3. Compustat: 1_Inputs/comp_na_daily_all/comp_na_daily_all.parquet
   - Columns: gvkey, datadate, fyear
   - Provides fiscal year-end dates for "frozen" constraint

**Key Implementation Requirements:**

1. **Fiscal Year Grid Construction:**
   - Read Compustat to get all valid (gvkey, fyear, datadate) combinations
   - datadate = fiscal year-end date for applying frozen constraint
   - Filter: datadate must be non-missing, 2002 <= fyear <= 2018 (sample period)

2. **CEO-Firm Assignment (Frozen Constraint):**
   - Join manifest calls to CEO Clarity scores via ceo_id
   - For each firm-year (gvkey, fyear) with fy_end = datadate:
     - Keep only calls where start_date <= fy_end (no future information)
     - Group by (gvkey, fyear, ceo_id), count calls
     - Select ceo_id with MAX calls for that firm-year (primary CEO)
     - If tie, select the CEO with earlier first_call_date (tenure tiebreaker)

3. **Handle Multiple CEOs per Firm-Year:**
   - A firm may have CEO turnover within a fiscal year
   - Strategy: Assign the CEO who participated in MOST calls in that fiscal year
   - This captures the "dominant" CEO for the year
   - Record: n_calls_fy (number of calls by selected CEO in that fiscal year)

4. **Handle CEO Moves Between Firms:**
   - Same ceo_id may appear with multiple gvkeys over time
   - Each firm-year gets the ClarityCEO score of whoever was CEO
   - CEO Clarity is a personal trait, not firm-specific

5. **Create style_frozen Variable:**
   - Map selected ceo_id for each (gvkey, fyear) to ClarityCEO score
   - ClarityCEO is the standardized CEO clarity (negative = more vague)
   - For H9, we want "vagueness" so could use -ClarityCEO (optional, document choice)
   - Output variable: style_frozen = ClarityCEO (or -ClarityCEO, document decision)

6. **Output Dataset:**
   - Columns: gvkey, fyear, datadate, style_frozen, ceo_id, ceo_name, n_calls_fy, n_calls_total
   - n_calls_fy: calls by this CEO in this fiscal year (frozen)
   - n_calls_total: total calls by this CEO in full sample (from Phase 56)
   - One row per (gvkey, fyear) where CEO assignment exists

7. **Filtering Rules:**
   - Require n_calls_fy >= 1 (at least one call by CEO in fiscal year)
   - Consider n_calls_total >= 5 filter (Phase 56 threshold) - make configurable
   - Keep firm-years where fyear is NOT the first year observed for that gvkey (for SalesGrowth lag)

**Script Structure:**
- Imports: sys, os, pathlib, pandas, numpy, argparse
- CLI: --dry-run, --min-calls-fy (default 1), --min-calls-total (default 5)
- load_ceo_clarity(): Load CEO scores from 4.1.1 output
- load_manifest_calls(): Load manifest, extract call-level CEO assignments
- load_compustat_dates(): Load fiscal year-end dates
- build_fy_grid(): Create all valid (gvkey, fyear, datadate) combinations
- assign_ceo_to_fy(): Implement frozen constraint + max calls selection
- create_style_frozen(): Merge CEO Clarity scores to firm-year assignments
- generate_report(): Coverage statistics, CEO turnover count, style distribution
- main(): Orchestrate execution

**Anti-Patterns to Avoid:**
- DO NOT use calls AFTER fiscal year-end (violates frozen constraint)
- DO NOT use simple ceo_id from Compustat (may not match earnings call CEOs)
- DO NOT assign CEO based on first or last call of year (use most calls)
- DO NOT forward-fill CEO assignments across years (recompute each year)
- DO NOT drop firm-years with CEO turnover (assign dominant CEO)
