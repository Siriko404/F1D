Create new script 2_Scripts/5_Financial_V3/5.8_H9_PRiskFY.py implementing fiscal-year PRisk aggregation.

**Input Files:**
1. Hassan PRisk data: 1_Inputs/FirmLevelRisk/firmquarter_2022q1.csv
   - Columns: gvkey, date (e.g., "2002q1"), PRisk, NPRisk, PSentiment, etc.
   - Quarterly format with string date like "2002q1", "2002q2"
   - gvkey needs zero-padding to 6 characters

2. Compustat: 1_Inputs/comp_na_daily_all/comp_na_daily_all.parquet
   - Columns: gvkey, datadate (fiscal year-end), fyear
   - Provides fiscal year-end dates for window construction

**Key Implementation Requirements:**

1. **Load and Clean PRisk Data:**
   - Read CSV with columns: gvkey, date, PRisk
   - Zero-pad gvkey to 6 characters: gvkey.str.zfill(6)
   - Convert date string "YYYYQq" to calendar quarter-end date:
     - Q1 -> March 31 (03-31)
     - Q2 -> June 30 (06-30)
     - Q3 -> September 30 (09-30)
     - Q4 -> December 31 (12-31)
   - Create cal_yearq string for uniqueness: "2002Q1", "2002Q2", etc.
   - Handle duplicates: If multiple rows for same (gvkey, cal_yearq), keep one with max PRisk
   - Print warning count for duplicates

2. **Load Compustat Fiscal Year Grid:**
   - Read Compustat parquet with columns: gvkey, datadate, fyear
   - Zero-pad gvkey to 6 characters
   - Convert datadate to datetime
   - Filter to sample period: 2002 <= fyear <= 2018 (or to PRisk availability)
   - Create unique (gvkey, fyear, datadate) combinations
   - datadate = fiscal year-end date (fy_end)

3. **Build PRiskFY (Fiscal-Year Aggregation):**

   For each firm-year (gvkey, fyear) with fy_end = datadate:

   a. **Define Averaging Window:**
      - Lower bound: fy_end - 366 days (approximately one year back)
      - Upper bound: fy_end (inclusive)
      - Select PRisk quarters where: lower_bound < cal_q_end <= fy_end

   b. **Minimum Quarters Rule (STRICT):**
      - Count quarters in window (n_quarters)
      - If n_quarters < 2: Set PRiskFY = missing (NaN), DROP from final output
      - If 2 <= n_quarters <= 4: Take mean of available quarters
      - DO NOT forward-fill: Missing quarters stay missing

   c. **Compute PRiskFY:**
      - PRiskFY = mean(PRisk of quarters in window)
      - Record n_quarters_used (for diagnostics)

4. **Handle Edge Cases:**
   - **Calendar quarter vs Fiscal quarter:** PRisk is on calendar quarters, firms have various fiscal year-ends. The 366-day window aligns them correctly regardless of fiscal period.
   - **Early years:** If firm has insufficient PRisk history (< 2 quarters), PRiskFY is missing
   - **Late years:** If PRisk data ends before fiscal year-end, PRiskFY becomes missing
   - **Gaps within window:** Missing quarters reduce n_quarters but don't disqualify if >= 2 remain

5. **Deduplication Rule (from H9 spec):**
   - If duplicate (gvkey, cal_yearq) rows exist:
     - Sort by PRisk descending
     - Keep first row (highest PRisk value)
     - Print count of duplicates removed

6. **Output Dataset:**
   - Columns: gvkey, fyear, datadate, PRiskFY, n_quarters_used
   - Only firm-years with n_quarters_used >= 2
   - PRiskFY may contain zeros (if mean of PRisk values is zero)
   - One row per (gvkey, fyear) with valid PRiskFY

7. **Optional Variables:**
   - If NPRisk exists in source, also compute NPRiskFY using same method
   - If PSentiment exists, compute PSentimentFY
   - Focus on PRiskFY as primary variable

**Script Structure:**
- Imports: sys, os, pathlib, pandas, numpy, argparse, datetime
- CLI: --dry-run, --min-quarters (default 2), --prisk-var (default "PRisk")
- load_prisk_data(): Read and clean Hassan PRisk CSV
- construct_cal_q_end(): Convert "2002q1" -> datetime(2002, 3, 31)
- deduplicate_prisk(): Handle duplicate (gvkey, cal_yearq) rows
- load_compustat_dates(): Read fiscal year-end dates
- compute_prisk_fy(): Main aggregation logic with 366-day window
- validate_min_quarters(): Apply >= 2 quarters rule
- generate_report(): Coverage stats, distribution, missingness
- main(): Orchestrate execution

**Anti-Patterns to Avoid:**
- DO NOT forward-fill missing quarters (violates spec)
- DO NOT use fiscal quarters instead of calendar quarters
- DO NOT set min_quarters < 2 (spec requires >= 2)
- DO NOT include quarters AFTER fy_end in the average
- DO NOT impute PRiskFY when n_quarters < 2
