  <files>2_Scripts/shared/iv_regression.py</files>
  <action>
Create `2_Scripts/shared/iv_regression.py` with:

1. **Contract header** per CLAUDE.md conventions:
   - ID: shared/iv_regression
   - Description: Two-stage least squares (2SLS) with instrument diagnostics
   - deterministic: true

2. **Custom exceptions**:
   ```python
   class WeakInstrumentError(Exception):
       """Raised when first-stage F-stat < 10 (weak instruments)."""
       pass
   ```

3. **`run_iv2sls()` function** (main entry point):
   ```python
   def run_iv2sls(
       df: pd.DataFrame,
       dependent: str,
       exog: List[str],           # Exogenous regressors (controls)
       endog: str,                # Endogenous regressor (e.g., vagueness)
       instruments: List[str],    # Instrumental variables
       add_constant: bool = True,
       cov_type: str = 'robust',  # 'robust', 'clustered', 'kernel'
       cluster_col: Optional[str] = None,  # For clustered SE
       f_threshold: float = 10.0,          # First-stage F threshold
       fail_on_weak: bool = True,          # Hard fail if F < threshold
       save_first_stage: bool = True       # Save first-stage predictions
   ) -> Dict[str, Any]:
   ```

4. **IV2SLS implementation** (ECON-04):
   - Use `linearmodels.iv.model.IV2SLS`
   - Build model: `IV2SLS(dependent=y, exog=X_exog, endog=X_endog, instruments=Z)`
   - Add constant if `add_constant=True`: prepend 'const' column to exog
   - Fit with appropriate cov_type

5. **First-stage F-stat validation** (ECON-05):
   - Extract from `result.first_stage.diagnostics`
   - Access F-stat: `diagnostics.loc[endog_var, 'f.stat']`
   - If F < f_threshold:
     - If `fail_on_weak=True`: raise `WeakInstrumentError` with message including actual F value
     - If `fail_on_weak=False`: add to warnings list, continue
   - Log F-stat to console with context: "First-stage F = {F:.2f} (threshold: {threshold})"

6. **Hansen J overidentification test** (ECON-05):
   - Only run if over-identified (len(instruments) > len(endog))
   - Access via `result.sargan` (Sargan test, equivalent to Hansen J for homoskedastic errors)
   - Report: stat, p-value, interpretation ("p > 0.05: instruments valid" or "p < 0.05: instruments may be invalid")
   - For heteroskedastic-robust: use `result.wooldridge_overid` if available

7. **Save first-stage predictions** (intermediate output):
   - If `save_first_stage=True`, store predicted values from first stage in return dict
   - Enables debugging instrument quality

8. **Return dictionary** with:
   - `model`: fitted IV2SLS model object
   - `coefficients`: DataFrame with beta, SE, t-stat, p-value per variable
   - `summary`: dict with R2, N, F-stat
   - `first_stage`: dict with F-stat, partial R2, predicted values
   - `overid_test`: dict with Hansen J stat and p-value (or None if just-identified)
   - `warnings`: list of any warnings triggered

9. **Console output**: Print coefficient table with significance stars, plus first-stage diagnostics block

<task type="auto">
  <name>Task 2: Create LaTeX Table Generation Module</name>
  <files>2_Scripts/shared/latex_tables.py</files>
  <action>
Create `2_Scripts/shared/latex_tables.py` with:

1. **Contract header** per CLAUDE.md conventions:
   - ID: shared/latex_tables
   - Description: Publication-ready LaTeX regression table generation
   - deterministic: true

2. **`format_coefficient()` helper function**:
   ```python
   def format_coefficient(
       beta: float,
       se: float,
       pvalue: float,
       decimals: int = 3,
       show_stars: bool = True
   ) -> Tuple[str, str]:
   ```
   - Return tuple of (formatted beta with stars, formatted SE in parentheses)
   - Stars: *** for p < 0.01, ** for p < 0.05, * for p < 0.10
   - Format: ("0.123***", "(0.045)")

3. **`make_regression_table()` function** (main entry point):
   ```python
   def make_regression_table(
       results: List[Dict[str, Any]],  # List of regression result dicts
       model_names: List[str],         # Column headers
       variable_order: Optional[List[str]] = None,  # Order of rows
       variable_labels: Optional[Dict[str, str]] = None,  # Pretty names
       include_stats: List[str] = ['N', 'R2', 'F'],
       caption: str = '',
       label: str = '',
       output_path: Optional[Path] = None,
       decimals: int = 3
   ) -> str:
   ```

4. **LaTeX table structure** (booktabs format):
   ```latex
   \begin{table}[htbp]
   \centering
   \caption{<caption>}
   \label{<label>}
   \begin{tabular}{l<c*N>}  % N = number of models
   \toprule
   & (1) & (2) & (3) \\
   & Model1 & Model2 & Model3 \\
   \midrule
   Variable1 & 0.123*** & 0.456** & 0.789 \\
             & (0.045)  & (0.078) & (0.890) \\
   Variable2 & ...
   \midrule
   Observations & 1,234 & 1,234 & 1,234 \\
   R-squared   & 0.45  & 0.52  & 0.48  \\
   Fixed Effects & Yes & Yes & No \\
   \bottomrule
   \end{tabular}
   \end{table}
   ```

5. **Variable ordering**:
   - If `variable_order` provided, use that order
   - Otherwise: main variables first, then controls, then fixed effects indicators

6. **Variable labels**:
   - Map internal names to publication names via `variable_labels`
   - E.g., `{'vagueness_c': 'Speech Uncertainty', 'leverage_c': 'Leverage'}`

7. **Statistics rows**:
   - 'N': Observations with comma formatting (e.g., "1,234")
   - 'R2': R-squared to 2 decimal places
   - 'adj_R2': Adjusted R-squared
   - 'F': F-statistic
   - 'FE_entity': "Yes" / "No" for entity fixed effects
   - 'FE_time': "Yes" / "No" for time fixed effects

8. **Output handling**:
   - If `output_path` provided, write to file
   - Always return LaTeX string

9. **`make_iv_table()` function** (specialized for IV results):
   ```python
   def make_iv_table(
       results: List[Dict[str, Any]],
       model_names: List[str],
       include_first_stage: bool = True,
       **kwargs
   ) -> str:
   ```
   - Same as make_regression_table but adds first-stage F-stat row
   - Optionally adds Hansen J test p-value row
