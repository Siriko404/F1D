Create `4.5_H5DispersionRegression.py` following the pattern of `4.1_H1CashHoldingsRegression.py` but with H5-specific models:

**Script Header:**
```
==============================================================================
STEP 4.5: H5 Analyst Dispersion Regression
==============================================================================
ID: 4.5_H5DispersionRegression
Description: Panel OLS regressions for H5 (Speech Uncertainty & Analyst Dispersion).
             Tests whether hedging language (weak modal verbs) predicts analyst
             forecast dispersion beyond what general uncertainty words predict.

Model Specification (Primary - H5-A):
    Dispersion_{t+1} = beta1*Weak_Modal_t + beta2*Uncertainty_t 
                     + beta3*Prior_Dispersion_t
                     + gamma*Controls + Firm FE + Year FE + epsilon

Model Specification (Secondary - H5-B Gap):
    Dispersion_{t+1} = beta1*Uncertainty_Gap_t + beta2*Pres_Uncertainty_t
                     + beta3*Prior_Dispersion_t
                     + gamma*Controls + Firm FE + Year FE + epsilon

Hypothesis Tests:
    H5-A: beta1 > 0 (Weak Modal predicts higher dispersion, controlling for Uncertainty)
    H5-B: beta1 > 0 (Gap predicts dispersion - spontaneous reveals hidden uncertainty)
```

**Key Implementation Details:**

1. **Primary Model (H5-A) - Novel Contribution:**
   - DV: `dispersion_lead` (analyst dispersion at t+1)
   - Primary IV: `Manager_QA_Weak_Modal_pct` (NOVEL - hedging verbs)
   - Control for established effect: `Manager_QA_Uncertainty_pct`
   - Controls: prior_dispersion, earnings_surprise, analyst_coverage (log NUMEST), 
              firm_size, earnings_volatility, loss_dummy, tobins_q, leverage
   - FE: Firm + Year
   - SE: Clustered at firm level
   - **Key Test:** Is beta1 (Weak_Modal) significant AFTER controlling for beta2 (Uncertainty)?

2. **Secondary Model (H5-B) - Gap Analysis:**
   - DV: `dispersion_lead`
   - Primary IV: `uncertainty_gap` = Manager_QA_Uncertainty - Manager_Pres_Uncertainty
   - Control for base level: `Manager_Pres_Uncertainty_pct`
   - Same controls as primary
   - **Key Test:** Is beta1 (Gap) significant? Gap > 0 means more spontaneous uncertainty.

3. **Uncertainty Measures to Test:**
   Primary (6 measures):
   - Manager_QA_Weak_Modal_pct (PRIMARY FOCUS)
   - CEO_QA_Weak_Modal_pct
   - Manager_QA_Uncertainty_pct (established - as comparison)
   - CEO_QA_Uncertainty_pct
   - Manager_Pres_Uncertainty_pct (scripted comparison)
   - CEO_Pres_Uncertainty_pct

4. **Specifications (4 variants):**
   ```python
   SPECS = {
       'primary': {'entity_effects': True, 'time_effects': True, 'double_cluster': False},
       'pooled': {'entity_effects': False, 'time_effects': False, 'double_cluster': False},
       'year_only': {'entity_effects': False, 'time_effects': True, 'double_cluster': False},
       'double_cluster': {'entity_effects': True, 'time_effects': True, 'double_cluster': True},
   }
   ```

5. **Robustness Checks (3 variants):**
   - `no_lagged_dv`: Exclude prior_dispersion (addresses Nickell bias concern)
   - `no_numest`: Exclude analyst_coverage (addresses bad control concern)
   - `ceo_only`: Use CEO-specific measures instead of Manager aggregates

6. **Control Variable Centering:**
   - Center continuous variables before regression (avoid multicollinearity)
   - Use `center_continuous()` from shared/centering.py

7. **One-Tailed Hypothesis Tests:**
   - H5-A: beta1 > 0 (more hedging → more dispersion)
   - Compute one-tailed p-value: p_one = p_two / 2 if beta > 0, else 1 - p_two/2
   - Significance threshold: p < 0.05

8. **VIF Check:**
   - Check multicollinearity between Weak_Modal and Uncertainty
   - If VIF > 5, report models with each measure separately
   - Document in stats.json
