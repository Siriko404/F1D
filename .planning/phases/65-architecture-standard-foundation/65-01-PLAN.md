2. **When to Archive**:
   - Code replaced by newer implementation
   - Experiments that didn't pan out
   - Deprecated functionality
   - Backup files older than 30 days

3. **Archive Conventions**:
   - Maintain original directory structure within archive
   - Add ARCHIVED.md explaining what and why
   - Include date archived and reason
   - Never delete from archive (git tracks history)

4. **Legacy Code Policy**:
   - Legacy code is read-only
   - No modifications without explicit approval
   - Reference legacy only for reproducibility verification
   - New work uses canonical V2

5. **Archive Manifest**:
   - manifest.json tracks all archived items
   - Include: original path, archive date, reason, related issues

6. **Rationale**: Why archive rather than delete
