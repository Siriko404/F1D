**Imports:** Same as H1 (sys, os, argparse, Path, datetime, pandas, numpy, yaml, hashlib, json, time, psutil, pyarrow.parquet)

**load_compustat_h3() function:**
- Required columns (verified AVAILABLE):
  - `gvkey`, `datadate`, `fyearq` (identifiers)
  - `dvpspq` (Dividends per Share - Quarterly) for stability/flexibility
  - `epspxq` (EPS Basic - Excl Extra Items - Quarterly) for earnings volatility
  - `req` (Retained Earnings - Quarterly) for firm maturity
  - `seqq` (Stockholders Equity - Quarterly) for firm maturity
  - `oancfy` (Operating Cash Flow - Annual) for FCF growth
  - `capxy` (Capital Expenditures - Annual) for FCF growth
  - `atq` (Total Assets - Quarterly) for ratios
- Use PyArrow schema inspection before reading (pattern from H1/H2)
- Normalize gvkey to 6-digit zero-padded string
- Convert Decimal types to float64

**annualize_quarterly_data() function:**
- Aggregate quarterly DPS/EPS to fiscal year level
- For DPS (`dvpspq`): SUM quarters to get annual dividend per share
- For EPS (`epspxq`): SUM quarters to get annual EPS
- Group by gvkey + fiscal_year

**compute_div_stability() function (H3-01):**
- Formula: `-StdDev(ΔDPS) / Mean(DPS)` over trailing 5 years
- Steps:
  1. Compute annual DPS from quarterly dvpspq (sum)
  2. Compute ΔDPS = DPS_t - DPS_{t-1} per firm
  3. 5-year rolling window: StdDev of ΔDPS, Mean of DPS
  4. Stability = -StdDev / |Mean| (negative CV makes higher = more stable)
- Guard against division by zero: require abs(mean_dps) > 0.001
- Minimum 2 years required in window (per CONTEXT.md)

**compute_payout_flexibility() function (H3-02):**
- Formula: `% of years with |ΔDPS| > 5% of prior DPS` over 5-year window
- Steps:
  1. Compute annual DPS from quarterly dvpspq
  2. Compute relative change = |DPS_t - DPS_{t-1}| / |DPS_{t-1}|
  3. Flag as "change" if relative_change > 0.05
  4. Rolling 5-year: count(changes) / count(years)
- Guard: require |DPS_{t-1}| > 0.001 to avoid division by zero
- Minimum 2 years required in window

**compute_earnings_volatility() function (H3-03):**
- Formula: `StdDev(annual EPS)` over trailing 5 years
- Steps:
  1. Aggregate quarterly epspxq to annual EPS (sum)
  2. 5-year rolling StdDev per firm
- Minimum 2 years required

**compute_fcf_growth() function (H3-03):**
- Formula: `(FCF_t - FCF_{t-1}) / |FCF_{t-1}|`
- Where FCF = (OANCF - CAPX) / AT
- Use absolute value in denominator to handle negative FCF
- Guard: require |FCF_{t-1}| > 0.001

**compute_firm_maturity() function (H3-03):**
- Formula: `RE / TE` (Retained Earnings / Total Equity)
- Use quarterly req and seqq, take end-of-year value per fiscal year
- Allow negative RE (valid immaturity signal)
- Require seqq > 0 to avoid division issues

**merge_standard_controls() function (H3-04):**
- Load H1_CashHoldings.parquet from latest output
- Extract: firm_size, roa, tobins_q, cash_holdings
- Merge on gvkey + fiscal_year
- If H1 output unavailable, recompute these from Compustat (fallback)

**flag_dividend_payers() function:**
- Filter to firms with DPS > 0 in at least one year of 5-year window
- This is required because stability/flexibility are undefined for never-payers

**winsorize_series() function:**
- Winsorize at 1%/99% percentiles
- Apply to: div_stability, payout_flexibility, earnings_volatility, fcf_growth, firm_maturity

**main() function:**
- CLI arguments: --config, --output-dir, --dry-run
- check_prerequisites() validates all inputs exist
- DualWriter for terminal + log mirroring
- Stats tracking (timing, memory, checksums)
- Full variable computation pipeline:
  1. Load manifest (filter to sample first)
  2. Load Compustat with H3 columns only
  3. Filter Compustat to sample gvkeys BEFORE heavy computation
  4. Annualize quarterly data
  5. Compute DVs and controls
  6. Merge all variables
  7. Flag/filter dividend payers
  8. Winsorize continuous variables
  9. Merge standard controls from H1
  10. Output H3_PayoutPolicy.parquet + stats.json
- Output columns: file_name, gvkey, fiscal_year, div_stability, payout_flexibility, earnings_volatility, fcf_growth, firm_maturity, firm_size, roa, tobins_q, cash_holdings, is_div_payer

**Critical pitfalls to AVOID (from RESEARCH.md):**
1. Do NOT compute rolling windows on quarterly data - annualize first
2. Do NOT divide by mean_dps without abs() > 0.001 guard
3. Do NOT include non-dividend payers in stability/flexibility (undefined)
4. DO use absolute value in FCF growth denominator (handles negative FCF)
5. DO allow negative RE/TE (valid immaturity signal)
