## Validation Schema

### Required Fields

Each stats.json file must contain the following top-level fields:

1. **`step_id`** (string, required)
   - Script identifier following naming convention: `2.<step>_<PascalCaseName>`
   - Example: `"4.1.1_EstimateCeoClarity_CeoSpecific"`

2. **`timestamp`** (string, required)
   - ISO 8601 format execution timestamp
   - Example: `"2026-01-22T23:00:17Z"`

3. **`output`** (object, required)
   - Contains output file information
   - Must include `files` array with file paths

### Optional Fields (highly recommended)

- **`input`** (object): Input file information with checksums
- **`processing`** (object): Duration, memory usage, start/end times
- **`stats`** (object): Descriptive statistics, missing values, distributions
- **`merge_diagnostics`** (object): Matched/unmatched counts for data joins
- **`execution`** (object): Git commit, config snapshot, environment details
