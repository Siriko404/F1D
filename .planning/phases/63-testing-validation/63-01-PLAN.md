# Research Context
From research content "Common Pitfalls > 3. Missing PYTHONPATH in Subprocess Tests":

Current Issue (from Phase 11 verification):
```python
# WRONG - subprocess cannot import shared modules
result = subprocess.run(["python", "script.py"], ...)
```

Solution (already implemented in test_pipeline_step3.py):
```python
# CORRECT - Set PYTHONPATH explicitly
SUBPROCESS_ENV = {
    "PYTHONPATH": str(REPO_ROOT / "2_Scripts"),
    **os.environ,
}
result = subprocess.run(["python", "script.py"], env=SUBPROCESS_ENV, ...)
```

The test_pipeline_step3.py already has correct PYTHONPATH handling (lines 33-36). Other integration tests need the same pattern.

Current conftest.py provides basic fixtures but no subprocess_env fixture.
</context>
