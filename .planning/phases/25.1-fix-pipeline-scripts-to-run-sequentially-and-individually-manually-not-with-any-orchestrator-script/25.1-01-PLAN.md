1. **validate_prerequisites(required_files, required_steps)**: Validates all required inputs and prerequisite step outputs

     - required_steps: Dict of {step_name: expected_output_file} for prerequisite steps

   Behavior:
     - Check each input file exists using shared.path_utils.validate_input_file()
     - Check each prerequisite step's latest/ directory exists and contains expected output file
     - Collect all errors (missing files, missing steps)
     - If errors exist: print formatted error messages with next steps and call sys.exit(1)
     - If no errors: print success message

2. **validate_prerequisite_step(step_name, expected_output_file, root)**: Validates a single prerequisite step

   Args:
     - step_name: Name of step (e.g., "1.1_CleanMetadata")
     - expected_output_file: Expected output filename in latest/ directory
     - root: Project root path

   Behavior:
     - Check 4_Outputs/{step_name}/latest/ exists
     - Check expected file exists in that directory
     - Print error message with script execution command if validation fails
     - Return True if valid, False otherwise

3. **print_prerequisite_errors(errors)**: Formats and prints prerequisite error messages

   Args:
     - errors: List of error messages

   Behavior:
     - Print separator line "ERROR: Prerequisites not met"
     - List all errors with numbering
     - Print "To fix these issues:" section with actionable steps
     - Show script execution order for missing prerequisites
     - Print "For help, run with --help flag"

4. **handle_missing_output(step_name, output_file, calling_script)**: Print error for missing prerequisite output

   Args:
     - step_name: Name of step that should have been run
     - output_file: Expected output file name
     - calling_script: Name of current script

   Behavior:
     - Print "ERROR: Missing prerequisite output" header
     - Show expected file location
     - Print script execution order
     - Exit with sys.exit(1)

- Use existing shared.path_utils.validate_input_file() for file validation
- Check latest/ symlinks for prerequisite step outputs
- Provide actionable error messages with script commands
- Print to stdout before sys.exit(1) to avoid silent failures
- Follow anti-patterns: Never exit without printing clear error message

## dependency_checker.py

Validates prerequisites (input files, previous step outputs) for pipeline scripts.

### Functions

- **validate_prerequisites(required_files, required_steps)**: Main entry point - validates all inputs and prerequisite steps, prints error messages and exits if validation fails
- **validate_prerequisite_step(step_name, expected_output_file, root)**: Validates a single prerequisite step completed
- **print_prerequisite_errors(errors)**: Formats and prints prerequisite error messages with next steps
- **handle_missing_output(step_name, output_file, calling_script)**: Prints error for missing prerequisite output

### Usage Example

```python
from shared.dependency_checker import validate_prerequisites
from pathlib import Path

root = Path(__file__).parent.parent.parent

# Define required inputs and prerequisite steps
required_files = {
    "Unified-info.parquet": root / "1_Inputs" / "Unified-info.parquet",
    "LM dictionary": root / "1_Inputs" / "Loughran-McDonald_MasterDictionary_1993-2024.csv",
}

required_steps = {
    "1.1_CleanMetadata": "metadata_cleaned.parquet",
    "1.2_LinkEntities": "metadata_linked.parquet",
}

# Validate prerequisites (exits with clear error if validation fails)
validate_prerequisites(required_files, required_steps)

# Continue with main script logic
```

### Design Notes

- Uses existing shared.path_utils for file validation
- Checks latest/ symlinks for prerequisite step outputs
- Provides actionable error messages with script execution commands
- Designed for manual script execution (not orchestrator-only workflows)
