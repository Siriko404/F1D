## Lessons Learned

### Directory Structure Pattern Mismatch
The Step 2 scripts (2.1, 2.2) output to subdirectories under `4_Outputs/2_Textual_Analysis/` (e.g., `2.2_Variables/latest/`), but dependency_checker's `validate_prerequisite_step()` expects `4_Outputs/{step_name}/latest/` pattern. However, the validation still works because dependency_checker is flexible - it checks for the step directory by name. The 2.2_ConstructVariables step should ideally create a `4_Outputs/2.2_ConstructVariables/latest/` symlink for consistency with other steps.

### Lazy Import Pattern for Windows Compatibility
Following the pattern from 4.1.2, 4.1.3, 4.1.4, the check_prerequisites() function uses lazy imports (`from shared.dependency_checker import validate_prerequisites`) to avoid import-time errors on Windows where sys.path might not be set up correctly until the script runs.
