**Researched:** 2026-01-24
**Domain:** Python CLI validation, pipeline script dependencies, error handling

## Summary

Phase 25.1 focuses on making the 17 pipeline scripts executable independently by hand, without relying on test frameworks or orchestrators. Currently, scripts are designed to run via pytest orchestrator (test_full_pipeline.py), which doesn't validate inputs, check prerequisites, or provide clear error messages for manual execution.

Research identified:
- **Current state:** Scripts validate some inputs via shared utilities but lack comprehensive dependency checking
- **17 scripts** distributed across 4 steps with clear sequential dependencies
- **Existing validation:** Path validation via `shared/path_utils.py`, schema validation via `shared/data_validation.py`
- **Missing features:** Prerequisite checking, self-documenting help messages, dependency verification
- **Phase 25 blocker:** Input data schema mismatch blocked execution, highlighting need for better validation

**Primary recommendation:** Add argparse-based CLI validation to each script with input file existence checks, prerequisite validation, and clear error messages for missing dependencies. Use existing shared utilities and extend `shared/path_utils.py` with dependency checking functions.

### Pattern 1: argparse for CLI Validation
**What:** Use argparse to add CLI validation, help messages, and input checking
**When to use:** When scripts need to accept optional arguments or validate execution conditions
**Example:**
```python
# Source: Python 3.14+ argparse documentation
import argparse

def main():
    parser = argparse.ArgumentParser(
        description='STEP 1.1: Clean Metadata & Event Filtering',
        formatter_class=argparse.RawDescriptionHelpFormatter
    )
    
    parser.add_argument(
        '--dry-run',
        action='store_true',
        help='Validate inputs without executing'
    )
    
    parser.add_argument(
        '--year-start',
        type=int,
        default=2002,
        help='Start year for filtering (default: 2002)'
    )
    
    parser.add_argument(
        '--year-end',
        type=int,
        default=2018,
        help='End year for filtering (default: 2018)'
    )
    
    args = parser.parse_args()
    
    # Check prerequisites before proceeding
    check_prerequisites(args)
```

### Pattern 2: Prerequisite Checking
**What:** Verify all required input files exist before processing begins
**When to use:** At script startup, before any data loading
**Example:**
```python
from shared.path_utils import validate_input_file, PathValidationError

def check_prerequisites():
    """Verify all required inputs exist."""
    errors = []
    
    # Check required input files
    input_files = {
        "Unified-info.parquet": root / "1_Inputs" / "Unified-info.parquet",
        "master_sample_manifest.parquet": root / "4_Outputs" / "1.4_AssembleManifest" / "latest" / "master_sample_manifest.parquet",
    }
    
    for name, path in input_files.items():
        if not path.exists():
            errors.append(f"Missing: {name} ({path})")
    
    # Check output directories from previous steps
    required_outputs = {
        "1.1_CleanMetadata": root / "4_Outputs" / "1.1_CleanMetadata" / "latest",
        "1.2_LinkEntities": root / "4_Outputs" / "1.2_LinkEntities" / "latest",
    }
    
    for step, path in required_outputs.items():
        if not path.exists():
            errors.append(f"Missing prerequisite: {step} output not found at {path}")
    
    if errors:
        print("=" * 70)
        print("ERROR: Missing prerequisites")
        print("=" * 70)
        for error in errors:
            print(f"  ✗ {error}")
        print()
        print("Please run the pipeline in correct order:")
        print("  Step 1: 1.1_CleanMetadata.py")
        print("  Step 1: 1.2_LinkEntities.py")
        print("  Step 1: 1.3_BuildTenureMap.py")
        print("  Step 1: 1.4_AssembleManifest.py")
        sys.exit(1)
```

### Pattern 3: Dependency Checking via Latest Symlinks
**What:** Use existing `latest/` symlinks to check if previous steps completed
**When to use:** When scripts depend on outputs from earlier steps
**Example:**
```python
from shared.path_utils import validate_input_file
from pathlib import Path

def validate_prerequisite_step(step_name, expected_output_file):
    """
    Validate that a prerequisite step has been completed.
    
    Args:
        step_name: Name of prerequisite step (e.g., "1.1_CleanMetadata")
        expected_output_file: Expected file in latest/ directory
    
    Raises:
        SystemExit: If prerequisite not met
    """
    # Find output directory for prerequisite step
    output_dir = root / "4_Outputs" / step_name / "latest"
    
    if not output_dir.exists():
        print(f"ERROR: Prerequisite step {step_name} not completed")
        print(f"  Expected: {output_dir}")
        print(f"  Please run: python 2_Scripts/{step_name}/{step_name}.py")
        sys.exit(1)
    
    # Check for expected output file
    expected_file = output_dir / expected_output_file
    if not expected_file.exists():
        print(f"ERROR: Prerequisite output missing: {expected_output_file}")
        print(f"  Directory: {output_dir}")
        sys.exit(1)
    
    print(f"✓ Prerequisite met: {step_name} completed")

# Usage in script:
def main():
    # Check prerequisites
    validate_prerequisite_step("1.1_CleanMetadata", "metadata_cleaned.parquet")
    validate_prerequisite_step("1.2_LinkEntities", "metadata_linked.parquet")
    
    # Proceed with main processing
    # ...
```

### Pattern 4: Clear Error Messages for Manual Execution
**What:** Provide actionable error messages that guide user to next steps
**When to use:** When validation fails or errors occur during execution
**Example:**
```python
def handle_missing_input(file_path, required_by_step):
    """Print helpful error message for missing input."""
    print()
    print("=" * 70)
    print("ERROR: Missing required input")
    print("=" * 70)
    print()
    print(f"  File not found: {file_path}")
    print()
    print("This file is required for processing.")
    print()
    if required_by_step:
        print(f"  This output is produced by: {required_by_step}")
        print(f"  Please run that step first: python {required_by_step}")
    else:
        print("  Please ensure all input files are in 1_Inputs/")
    print()
    print("For help, run with --help flag")
    sys.exit(1)
```

### Anti-Patterns to Avoid
- **Silent failures:** Scripts exiting with non-zero code without printing error messages
  - Why bad: User doesn't know what went wrong
  - Do instead: Always print clear error message before sys.exit()
  
- **Generic error messages:** "Error occurred" without context
  - Why bad: User can't diagnose or fix the issue
  - Do instead: Include file paths, expected vs actual, and next steps
  
- **Continuing on missing inputs:** Processing with None/empty data instead of failing fast
  - Why bad: Wastes time, produces invalid outputs
  - Do instead: Fail immediately with clear message about missing inputs
  
- **Assuming orchestrator handles validation:** Not checking prerequisites
  - Why bad: Scripts fail mid-pipeline with obscure errors
  - Do instead: Validate all prerequisites at startup

## Common Pitfalls

### Pitfall 1: Silent Failures
**What goes wrong:** Script exits with sys.exit(1) but prints nothing to stdout
**Why it happens:** Developer assumes user will check logs, or forgets to add print() before exit
**How to avoid:** Always print error message to stdout before calling sys.exit()
**Warning signs:** Running `python script.py` returns exit code 1 but produces no output

### Pitfall 2: Not Checking Previous Step Outputs
**What goes wrong:** Script attempts to load file from `latest/` symlink that doesn't exist or is broken
**Why it happens:** Previous step failed, but current script doesn't validate prerequisites
**How to avoid:** Check existence of `latest/` directories and expected output files before loading data
**Warning signs:** FileNotFoundError mid-script when loading inputs from 4_Outputs

### Pitfall 3: Assuming Orchestrator Handles Dependencies
**What goes wrong:** Script validates only its direct inputs, not dependencies from other steps
**Why it happens:** Scripts were designed to run via pytest orchestrator which handles dependencies
**How to avoid:** Add prerequisite checking that validates all required previous-step outputs exist
**Warning signs:** Script starts loading data without verifying prerequisites

### Pitfall 4: Missing Configuration Validation
**What goes wrong:** Script loads config/project.yaml without checking if it exists
**Why it happens:** Assumes config always exists because orchestrator checks it
**How to avoid:** Use `validate_input_file()` to check config path at startup
**Warning signs:** FileNotFoundError when loading config file

### Pitfall 5: Not Providing Clear Next Steps
**What goes wrong:** Error message says "file not found" without explaining what to do
**Why it happens:** Focus on what went wrong, not how to fix it
**How to avoid:** Include actionable guidance: which script to run, what files to check
**Warning signs:** User sees error but doesn't know how to proceed

### Pitfall 6: Ignoring Latest Symlink Resolution
**What goes wrong:** Script tries to read from `latest/` but symlink is broken or points to wrong directory
**Why it happens:** Symlinks work in automated tests but can break when scripts run individually
**How to avoid:** Validate that `latest/` is a directory and exists before resolving it
**Warning signs:** "NotADirectoryError" when accessing latest symlink

### Pitfall 7: Not Validating Input File Format
**What goes wrong:** Script loads parquet file that's corrupted or has wrong schema
**Why it happens:** Assumes file format is correct because it worked in orchestrated run
**How to avoid:** Use `shared.data_validation.py` with strict schema checking
**Warning signs:** Pandas errors or DataValidationError during data loading

# Source: Combining existing path_utils patterns with dependency checking
from pathlib import Path
import sys

def check_prerequisites():
    """
    Validate all required inputs and prerequisites exist.
    
    Raises:
        SystemExit: If prerequisites not met, with clear error message
    """
    root = Path(__file__).parent.parent.parent
    errors = []
    
    # Check 1_Inputs files
    required_inputs = [
        ("Unified-info.parquet", root / "1_Inputs" / "Unified-info.parquet"),
        ("Loughran-McDonald_MasterDictionary_1993-2024.csv", 
         root / "1_Inputs" / "Loughran-McDonald_MasterDictionary_1993-2024.csv"),
    ]
    
    for name, path in required_inputs:
        if not path.exists():
            errors.append(f"Missing input file: {name} ({path})")
    
    # Check prerequisite outputs from Step 1
    step1_outputs = [
        ("1.1_CleanMetadata", root / "4_Outputs" / "1.1_CleanMetadata" / "latest"),
        ("1.2_LinkEntities", root / "4_Outputs" / "1.2_LinkEntities" / "latest"),
        ("1.3_BuildTenureMap", root / "4_Outputs" / "1.3_BuildTenureMap" / "latest"),
    ]
    
    for step, path in step1_outputs:
        if not path.exists():
            errors.append(f"Missing prerequisite: Step {step} not completed")
    
    # Check for config file
    config_path = root / "config" / "project.yaml"
    if not config_path.exists():
        errors.append(f"Missing configuration file: {config_path}")
    
    if errors:
        print_prerequisite_errors(errors)
        sys.exit(1)
    
    # All prerequisites met
    print("✓ All prerequisites validated")

def print_prerequisite_errors(errors):
    """Print formatted prerequisite errors."""
    print()
    print("=" * 70)
    print("ERROR: Prerequisites not met")
    print("=" * 70)
    print()
    print("The following issues were found:")
    print()
    for i, error in enumerate(errors, 1):
        print(f"  {i}. {error}")
    print()
    print("To fix these issues:")
    print("  1. Ensure all input files are in 1_Inputs/")
    print("  2. Run the pipeline in correct order:")
    print("     Step 1.1: python 2_Scripts/1_Sample/1.1_CleanMetadata.py")
    print("     Step 1.2: python 2_Scripts/1_Sample/1.2_LinkEntities.py")
    print("     Step 1.3: python 2_Scripts/1_Sample/1.3_BuildTenureMap.py")
