# Research Context
From 62-RESEARCH.md Pattern 1 and Example 1:
- Lines 505-509 in 1.2_LinkEntities.py use inefficient df.loc[update_df.index, col] = update_df[col] pattern
- Each .loc assignment creates an intermediate copy, causing O(n) memory operations per column
- Fix uses pd.merge() with how='left' or df.update() for in-place modification
- Expected speedup: 2-5x for bulk updates with 5 columns

Current problematic code (lines 502-511):
```python
unique_df = unique_df.set_index("company_id")
update_df = update_df.set_index("company_id")

unique_df.loc[update_df.index, "gvkey"] = update_df["gvkey"]
unique_df.loc[update_df.index, "conm"] = update_df["conm"]
unique_df.loc[update_df.index, "sic"] = update_df["sic"]
unique_df.loc[update_df.index, "link_method"] = update_df["link_method"]
unique_df.loc[update_df.index, "link_quality"] = update_df["link_quality"]

unique_df = unique_df.reset_index()
```

This pattern repeats for Tiers 2, 3, and 4 with similar column updates.
