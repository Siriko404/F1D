Create 2_Scripts/shared/data_validation.py with:

1. INPUT_SCHEMAS dictionary defining schemas for:
   - Unified-info.parquet:
     * required_columns: event_type, file_name, date, speakers
     * column_types: event_type (int), date (object), speakers (object)
     * value_ranges: event_type min=0 max=10
   - Loughran-McDonald_MasterDictionary_1993-2024.csv:
     * required_columns: word, negative, positive, uncertainty, litigious
     * column_types: word (object), negative (int), positive (int), etc.
   - Add schemas for other input files as needed (can be minimal for now)

2. DataValidationError exception class:
   - Inherits from Exception
   - Used for validation failures

3. validate_dataframe_schema(df, schema_name, file_path, strict) function:
   - Check required columns exist (add to errors list if missing)
   - Check column types match expected (add to errors if mismatch)
   - Check value ranges (add to errors for values outside min/max)
   - If errors exist:
     * strict=True: raise DataValidationError
     * strict=False: print warning to stderr
   - Print success message if no errors

4. load_validated_parquet(file_path, schema_name, strict) function:
   - Load file with pd.read_parquet(file_path)
   - Call validate_dataframe_schema() if schema_name provided
   - Return validated DataFrame

Follow Pattern 3 from 09-RESEARCH.md exactly.

Use Python standard library + pandas (already in project).

Add docstrings explaining validation is opt-in per script.
