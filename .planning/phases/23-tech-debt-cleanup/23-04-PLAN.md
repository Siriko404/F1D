<task type="auto">
  <name>Task 2: Fix error handling in identified scripts</name>
  <files>2_Scripts/4_Econometric/*.py</files>
  <action>
For each script that has bare except blocks (from Task 1), replace with specific exception handling.

For each bare except block:
1. Identify the risky operation (file load, data processing, model fitting, etc.)
2. Determine likely exception types (FileNotFoundError, PermissionError, OSError, ValueError, KeyError, etc.)
3. Replace with specific exceptions following Phase 7 pattern

Example replacement:
```python
# BEFORE (bad):
try:
    df = pd.read_parquet(filepath)
except:
    pass  # Silent failure

# AFTER (good):
import sys
try:
    df = pd.read_parquet(filepath)
except FileNotFoundError as e:
    print(f"ERROR: File not found: {e}", file=sys.stderr)
    print(f"  File: {filepath}", file=sys.stderr)
    sys.exit(1)
except PermissionError as e:
    print(f"ERROR: Permission denied: {e}", file=sys.stderr)
    print(f"  File: {filepath}", file=sys.stderr)
    sys.exit(1)
except OSError as e:
    print(f"ERROR: OS error reading file: {e}", file=sys.stderr)
    print(f"  File: {filepath}", file=sys.stderr)
    sys.exit(1)
```

Common exception types for econometric scripts:
- **FileNotFoundError** - Input files not found
- **PermissionError** - No write permissions for output
- **OSError** - General file system errors
- **ValueError** - Invalid parameter values, empty data
- **KeyError** - Missing column names in DataFrame operations

Key principles (from Phase 7):
1. Import sys if not already imported
2. Catch specific exceptions
3. Log error to stderr with context (file path, operation name)
4. Exit with sys.exit(1) on critical failures
5. Never use bare except: with pass (silent failures)

Apply this to all scripts identified in Task 1.
