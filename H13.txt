## H13 suite — final specification

### Suite ID

**H13_real_outcomes_uncertainty**

### Research question

Does managerial language uncertainty in earnings calls predict weaker subsequent real activity?

### Core hypothesis

Higher management uncertainty in the current call is associated with:

1. **lower future capital expenditure intensity**, and
2. **lower future employment growth**.

---

## Shared design rules for the full H13 suite

This suite should **inherit the existing pipeline conventions** wherever those conventions already exist.

That means Claude Code should use the **same panel structure, merge keys, sample restrictions, fixed effects, clustering, lag/lead logic, and outlier treatment** already used in the current lead-outcome regressions and H11 regressions, unless this H13 spec explicitly overrides something. Your existing models are already estimated with **PanelOLS**, **Entity and Time fixed effects**, and **clustered covariance**, so H13 should use the same estimator setup.    

### Primary explanatory variable for H13

Use **`CEO_Pres_Uncertainty_pct`** as the **primary RHS variable** in both H13 models.

Reason: this is the cleanest management-side operating-outlook uncertainty measure, and it avoids making H13 depend primarily on Q&A dynamics.

### Optional robustness variables

Only run these if you want robustness after the main H13 suite is complete:

* `Manager_Pres_Uncertainty_pct`
* `CEO_QA_Uncertainty_pct`

These are **not required** for the base H13 suite.

### Objective

Test whether higher current management uncertainty predicts lower future investment intensity.

### Estimation equation

[

\alpha_i + \delta_t

* \beta_1 CEO_Pres_Uncertainty_pct_{i,t}
* \beta_2 Size_{i,t}

### Dependent variable

* **`CapexAt_lead`** = one-period-ahead lead of the existing pipeline variable `CapexAt`, computed within firm using the pipeline’s existing period convention.

## Model H13.2 — Future employment growth

### Objective

Test whether higher current management uncertainty predicts lower subsequent hiring / employment growth.

### Estimation equation

[
EmploymentGrowth_lead_{i,t}
===========================

\alpha_i + \delta_t

* \beta_1 CEO_Pres_Uncertainty_pct_{i,t}
* \beta_2 Size_{i,t}
* \beta_3 TobinsQ_{i,t}
* \beta_4 ROA_{i,t}
* \beta_5 Lev_{i,t}
* \beta_6 CashHoldings_{i,t}
* \beta_7 DividendPayer_{i,t}
* \beta_8 OCF_Volatility_{i,t}
* \varepsilon_{i,t}
  ]

### Dependent variable

* **`EmploymentGrowth_lead`** = one-period-ahead employment growth constructed from **Compustat `EMP`**.

### Exact construction of the new variable

This variable is **not yet in the pipeline**, so define it exactly as follows:

[
EmploymentGrowth_lead_{i,t} = \ln(EMP_{i,t+1}) - \ln(EMP_{i,t})
]

### Construction rules

1. Source `EMP` from **Compustat**.
2. Use the **same firm identifier and time alignment convention** already used in the pipeline.
3. Sort by firm and time, then construct the one-period-ahead log difference.
4. Compute the variable only when both `EMP_{i,t}` and `EMP_{i,t+1}` are non-missing and strictly positive.
5. If the pipeline already uses a standard winsorization rule for continuous outcomes, apply the same rule to `EmploymentGrowth_lead`.
6. If `EMP` coverage at the current pipeline frequency is too sparse, run this model at the **highest-frequency Compustat panel with reliable EMP coverage** while keeping all other conventions unchanged.

### Variables to use

For the following variables, **do not redefine**; use the exact existing pipeline definitions:

* `CEO_Pres_Uncertainty_pct`
* `Size`
* `TobinsQ`
* `ROA`
* `Lev`
* `CashHoldings`
* `DividendPayer`
* `OCF_Volatility`

### Interpretation target

* Main coefficient of interest: **`CEO_Pres_Uncertainty_pct`**
* Expected sign: **negative**
* Interpretation: firms with more uncertain management language today show lower employment growth next period.

## Why this H13 suite is aligned with the current pipeline

The current H11 regressions already use the uncertainty-side control block built around variables such as `Size`, `TobinsQ`, `ROA`, `CashHoldings`, `DividendPayer`, `firm_maturity`, and `earnings_volatility`. 
The current downstream lead-outcome regressions already use the lead-outcome style control block with `Lev`, `Size`, `TobinsQ`, `ROA`, `CapexAt`, `DividendPayer`, and `OCF_Volatility`, under Entity and Time fixed effects with clustered standard errors.  

H13 is designed to sit naturally after those suites:

* it keeps the **same estimator architecture**,
* it uses the **same core uncertainty variable family**,
* and it extends the paper into **real outcomes** rather than payout-only outcomes.
